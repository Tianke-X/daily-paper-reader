## 摘要
多跳RAG提出了一个超越段落匹配的数据工程问题：在固定检索预算下，系统必须将检索到的文本组织成能够揭示答案链的证据单元。密集检索器对段落进行独立评分，而基于图的记忆使关联显式化，但通常依赖成对或实体中心的键，这会将多跳证据碎片化。我们提出了HKVM-RAG，一个键值分离的证据组织层。它从缓存的段落级LLM证据元组中组装答案路径超边，并将其用作检索键，同时保留段落文本作为答案值。为了隔离键空间设计，我们的固定子协议在成对图和超图变体之间保持元组缓存、候选段落、阅读器和评估预算不变。加权超图键值检索在2WikiMultiHopQA上比KG-PPR提高了+3.426 F1，在MuSiQue上提高了+3.592 F1；HotpotQA显示，更高的结构化支持覆盖率并不一定会带来独立的答案F1增益。因此，我们将WHG-KV视为一种证据控制信号，而非密集检索的替代。Oracle和训练到开发分析表明，支持选择是可修复的，一个密集感知控制器结合了冻结的ColBERTv2和基于HKVM排名/分数的特征，使用折叠外的HKVM预测。它在三个基准上分别达到了88.846、65.073和85.810的F1值，相比ColBERTv2分别提高了+11.084、+6.763和+5.966 F1。源级消融实验表明，匹配的非WHG结构化信号无法达到WHG-KV的增益。这些结果提供了有界证据，表明键值分离的超图组织可以作为多跳RAG的可重用证据控制机制。

## Abstract
Multi-hop RAG poses a data-engineering problem beyond passage matching: under fixed retrieval budgets, a system must organize retrieved text into evidence units that expose answer chains. Dense retrievers score passages independently, while graph-based memories make associations explicit but often rely on pairwise or entity-centered keys that fragment multi-hop evidence. We present HKVM-RAG, a key-value-separated evidence-organization layer. It assembles answer-path hyperedges from cached passage-level LLM evidence tuples and uses them as retrieval keys, while retaining passage text as answer values. To isolate key-space design, our fixed-substrate protocol holds the tuple cache, candidate passages, reader, and evaluation budget constant across pairwise graph and hypergraph variants. Weighted hypergraph key-value retrieval improves over KG-PPR by +3.426 F1 on 2WikiMultiHopQA and +3.592 F1 on MuSiQue; HotpotQA shows that higher structured support coverage need not yield standalone answer-F1 gains. We therefore study WHG-KV as an evidence-control signal rather than a dense-retrieval replacement. Oracle and train-to-dev analyses identify support selection as repairable, and a dense-aware controller combines frozen ColBERTv2 and HKVM rank/score features using out-of-fold HKVM predictions. It reaches 88.846, 65.073, and 85.810 F1 on the three benchmarks, improving over ColBERTv2 by +11.084, +6.763, and +5.966 F1. Source-level ablations show that matched non-WHG structured signals do not match the WHG-KV gains. These results provide bounded evidence that key-value-separated hypergraph organization can serve as a reusable evidence-control mechanism for multi-hop RAG.

## 1. 论文的核心问题与整体含义（研究动机和背景）

  - **基于图的记忆**（如知识图谱RAG）虽然使关联显式，但检索键仍然是成对或实体中心的二元边，导致多跳证据碎片化。
- **整体含义**：本文试图回答一个更底层的机制问题——在**相同的证据提取子层（cached evidence tuple cache）、候选段落、阅读器和评估预算**下，将检索键空间从成对图边改为答案路径超边（hyperedge），是否能够改善证据组织，从而提升多跳问答性能？论文还将超图键值信号定位为一种“可复用的证据控制机制”，与密集检索互补而非替代。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：**键值分离（Key-Value Separation）**。将检索键空间与值空间明确分离：
  - **键空间**：由LLM提取的实体、成对关系、以及更高阶的答案路径超边构成，包含置信度权重。
  - **值空间**：保留原始段落文本，仅用于最终答案提取。
- **关键技术步骤**：
  1. **共享LLM证据元组提取**：对每个段落，LLM（DeepSeek V4 Flash，温度0.0）提取局部关系三元组 `(h, r, t, p_i, c_f, c_s, c_b)`，其中`c_f`为事实置信度、`c_s`为语义显著性、`c_b`为桥接潜力。提取一次并缓存，三个评估种子复用。
  2. **答案路径超边组装**：通过“桥接顶点”（bridge vertex）将共享同一实体的多个元组分组，形成超边 `e_v = (V_v, P_v)`，`V_v`为超边包含的实体集合，`P_v`为可接受分数传播的段落ID集合。
  3. **加权超图键遍历（Weighted-HG-KV）**：
     - 超边权重由置信度决定：`a(e) = (λ_f c_f + λ_b c_b) / (λ_f + λ_b)`，默认`λ_f=0.5, λ_b=0.5`，忽略`c_s`；最终权重 `w(e) = 1 + (β-1)a(e), β=3.0`。
     - 查询种子由查询与实体匹配和token重叠得到。
     - 超图扩散：交替更新顶点分数和超边分数，公式：
       ```
       x(t+1)_v = ρ·x(0)_v + (1-ρ)·∑_{e∋v} [w(e) / (d(v)·|e|)] · ∑_{u∈e} x(t)_u
       ```
       其中`ρ=0.35`，遍历深度`T=1`（扩展为2作为消融）。
     - 扩散后，通过超边到段落的映射将键空间分数传回段落值空间，得到HKVM段落排名和分数。
  4. **成对图参考（KG-PPR）**：使用相同元组缓存，每对实体构成无向边，运行Personalized PageRank（α=0.5, 40迭代）。用于隔离成对键与超图键的对比。
  5. **密集感知证据控制器**：将冻结的ColBERTv2和冻结的HKVM预测按段落ID对齐，构建每个段落4维特征（ColBERTv2 rank/score, HKVM rank/score）及交互项，训练一个逻辑回归段落排序器（80 epochs, lr=0.01, L2=1e-4, 负采样比8, top-10检索, top-5答案上下文）。训练时使用“折叠外（out-of-fold）”HKVM预测以避免数据泄露。

## 6. 论文的主要结论与发现

1. **固定基质下，超图键值检索优于成对图**：
   - 2WikiMultiHopQA：Weighted-HG-KV F1 79.299 vs KG-PPR 75.873，Δ=+3.426（95% CI [2.877, 3.984]）。
   - MuSiQue：Weighted-HG-KV F1 39.925 vs KG-PPR 36.332，Δ=+3.592（95% CI [1.917, 5.155]）。
   - HotpotQA：Δ=-0.689（CI跨越零），表明结构化覆盖提升不一定带来答案F1增益。
2. **支持选择是主要瓶颈，可通过训练部分恢复**：
   - Oracle答案路径选择将2Wiki F1从79.299提升至90.884，MuSiQue从39.925提升至74.330，说明元组缓存足够好，但选择错误片段。
   - 训练到开发支持评分器恢复68.3%（2Wiki）、41.6%（MuSiQue）、47.2%（HotpotQA）的Oracle差距。
3. **密集感知控制器有效，WHG-KV信号不可替代**：
   - 最终控制器：2Wiki 88.846 F1、MuSiQue 65.073 F1、HotpotQA 85.810 F1，分别比ColBERTv2高+11.084、+6.763、+5.966。
   - 源级消融证实：用Co-occurrence-HG、Static-HG或Weighted-KG-PPR替代WHG-KV时，增益消失或远低于WHG-KV。第一阶段源（BM25/Contriever）替换后WHG-KV仍保持最大增益。
4. **HotpotQA角色不同**：独立结构检索未胜过词级/密集检索，但WHG-KV排名/分数仍可作为控制信号帮助重新排序。
