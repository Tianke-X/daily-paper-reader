## 摘要
我们提出了一种基于智能体的大语言模型（LLM）框架，用于主动容错控制（FTC），该框架将故障检测输出转化为基于工厂特定知识的约束感知恢复行动。该方法结合了：(i) 多智能体工作流，将操作员职责分解为监控、规划、行动合成、仿真、验证和重新提示；(ii) 数字过程工厂孪生体（DPPT），提供工厂数据、模型和用于执行前测试的仿真服务；(iii) 基于CPSMod本体的图检索增强生成（Graph RAG）层，该本体将工厂知识（结构、功能、混合动力学、控制上下文和故障语义）组织成图，支持面向关系的多跳检索，为智能体提供支持。纠正行动生成为最小风险的状态机恢复路径及相应的离散指令或连续设定点调整，然后在任何执行之前，通过互锁、包络和动态可行性进行确定性验证。如果在有限时间窗口内未找到可接受的计划，则控制权移交给安全后备方案。该框架在仿真中针对两个代表性基准进行了评估：离散批处理混合模块和闭环PID调节下的连续搅拌釜反应器（CSTR）。使用轻量级LLM（GPT-4o-mini和GPT-4.1-mini）的结果表明，语义驱动的智能体能够在符合相应过程动态的延迟预算内推导出有效的恢复决策，从而展示了从检测到经过验证的纠正行动在离散和连续FTC任务中的实用路径。

## Abstract
We propose an agentic Large Language Model (LLM) framework for active Fault-Tolerant Control (FTC) that transforms fault detection outputs into constraint-aware recovery actions grounded in plant-specific knowledge. The approach couples (i) a multi-agent workflow that decomposes operator duties into monitoring, planning, action synthesis, simulation, validation, and reprompting; (ii) a Digital Process Plant Twin (DPPT) that exposes plant data, models, and a simulation service for pre-execution testing; and (iii) a Graph Retrieval-Augmented Generation (Graph RAG) layer built on the CPSMod ontology, which organizes plant knowledge (structure, function, hybrid dynamics, control context, and fault semantics) into a graph that supports relation-aware, multi-hop retrieval for the agents. Corrective actions are generated as minimal-risk state-machine recovery paths and corresponding discrete commands or continuous setpoint adaptations, then validated deterministically against interlocks, envelopes, and dynamic feasibility before any actuation. If no acceptable plan is found within a bounded time window, control is handed to a safety fallback. The framework is evaluated in simulation on two representative benchmarks: a discrete batch Mixing Module and a Continuous Stirred-Tank Reactor (CSTR) under closed-loop PID regulation. Results with lightweight LLMs (GPT-4o-mini and GPT-4.1-mini) show that semantically grounded agents can derive valid recovery decisions within latency budgets compatible with the respective process dynamics, demonstrating a practical pathway from detection to validated corrective action across both discrete and continuous FTC tasks.

## 6. 主要结论与发现

- GPT-4.1-mini在混合模块上实现**100%成功率**（60/60），在CSTR上达到**93.3%成功率**（28/30）；GPT-4o-mini分别为85%和76.7%，表明更先进的轻量模型在结构化控制任务中表现更优。
- **图RAG知识检索有效保障了正确行动**：两个模型在所有测试中**行动召回率均为100%**（从未遗漏必要执行器），错误仅来自**多余的非许可操作**（GPT-4o-mini）或**错误规划路径**，证明知识图谱提供了可靠的执行器-状态关联。
- **重提示机制能纠正部分错误，但无法解决系统性推理失败**：在pump_degradation（混合模块）和sensor_fault场景中，GPT-4o-mini的低恢复率（44%和33%）表明其存在根本性误解，而其他场景恢复率可达100%。
- **延迟可接受**：单次决策延迟约2-3秒，总墙钟时间30-50秒，适合批处理和监控控制，但不适合毫秒级硬实时。
- **轻量级LLM足以执行此类结构化控制任务**，无需强大通用推理能力，因为关键领域知识已通过图RAG提供。

## 7. 优点

- **框架创新性**：首次将LLM智能体、数字孪生与本体驱动的图RAG相结合用于主动容错控制，将自然语言理解与工程约束集成。
- **安全优先设计**：所有行动在仿真中验证，并经过确定性规则检查（互锁、包络、动力学可行性），确保执行前安全性。
- **可解释性**：各智能体（特别是规划与行动智能体）输出中间推理和决策痕迹，便于事后审计与人类监督。
- **灵活性**：无需重新设计控制器即可适应新故障或设备变更，只需更新知识图谱即可扩展。
- **错误分析深入**：不仅报告总体指标，还区分了规划错误与行动错误，揭示了不同模型的能力差异和故障模式，为改进提供方向。
- **轻量级LLM的实用性验证**：表明GPT-4.1-mini等低成本模型即可胜任，降低了工业部署的门槛。

## 8. 不足与局限

- **知识图谱构建成本高**：当前实现依赖人工构建的CPSMod本体和实例数据，自动化提取方法（如利用LLM从非结构化文档生成）尚未纳入，限制了可扩展性。
- **无法处理未知故障**：数字孪生只能验证已建模的故障；对于全新或组合故障，仿真无法准确评估，存在失效风险。
- **延迟不适用于硬实时系统**：2-4秒的决策时间适用于慢速过程，但不适用于高速或安全关键应用（如毫秒级响应的紧急停堆）。
- **全局最优性未保证**：框架是“可行性优先”（feasibility-first）而非最优优先，可能输出保守但非最优的恢复方案。
- **样本量较小**：每个场景仅10次重复，缺乏统计显著性检验；可能因随机种子或API波动导致结果偏差。
- **未与基线方法对比**：缺少与模型预测控制（MPC）、强化学习、基于规则的FTC等成熟方法的性能对比，难以量化该方法的相对优势。
- **数字孪生理想化假设**：假设仿真模型与实际工厂匹配良好；实际中存在模型失配、传感器噪声、未建模动态，需要不确定性感知验证。
- **LLM API依赖**：实验采用云端API，存在延迟波动、数据隐私、网络分区等工业部署风险；未评估边缘部署或本地模型的可行性。
- **仅验证仿真环境**：未在真实物理设备上运行，系统鲁棒性和可靠性有待进一步测试。
