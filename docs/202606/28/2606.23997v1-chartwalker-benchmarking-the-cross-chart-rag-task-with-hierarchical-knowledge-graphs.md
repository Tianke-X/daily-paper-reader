## 摘要
跨图表检索增强生成（Cross-Chart RAG）对于科学、商业和政治领域的复杂多模态分析任务至关重要。然而，现有基准要么聚焦于结构化和文本化的表格，要么通过简单提取要点生成跨图表问题，这通常会导致查询与证据之间的词汇重叠以及逻辑不一致的推理链。为此，我们提出ChartWalker，一种用于构建具有挑战性的跨图表RAG任务的新框架。ChartWalker采用了一种针对图表的层次化知识图谱构建方法，该方法按粒度组织实体和关系以保留分析结构。随后，我们提出一种结构感知采样算法，用于合成语义连贯的多跳推理路径，从而实现对查询难度和粒度的显式控制以生成问答对。基于该框架，我们发布了ChartWalker-Bench，一个涵盖多个领域和跨图表查询类型的综合基准。在主要RAG范式上的广泛评估揭示了显著的性能差距，凸显了该基准的难度和实用性。此外，我们提供了ChartWalker-Agent，一个基于智能体的基线，以促进分析并启发未来系统设计。

## Abstract
Cross-Chart Retrieval-Augmented Generation (RAG) is critical for complex multi-modal analytical tasks in scientific, business, and political domains. However, existing benchmarks either focus on tables, which are well-structured and textualized, or generate cross-chart questions by simply extracting key points, which often induces lexical overlap between queries and evidence and yields logically inconsistent reasoning chains. To address this, we introduce ChartWalker, a novel framework for constructing challenging cross-chart RAG tasks. ChartWalker features a hierarchical knowledge graph construction method tailored to charts, which organizes entities and relations by granularity to preserve analytical structure. We then propose a structure-aware sampling algorithm that synthesizes semantically coherent, multi-hop reasoning paths, enabling explicit control over query difficulty and granularity for QA generation. Built with this framework, we release ChartWalker-Bench, a comprehensive benchmark spanning diverse domains and cross-chart query types. Extensive evaluations across major RAG paradigms reveal significant performance gaps, underscoring the benchmark's difficulty and utility. Furthermore, we provide ChartWalker-Agent, an agentic baseline to facilitate analysis and inspire future system design.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究问题**：跨图表检索增强生成（Cross-Chart Retrieval-Augmented Generation, Cross-Chart RAG）要求系统综合多个图表中的信息进行多步推理。现有基准存在两大缺陷：
  - **词汇重叠与逻辑不一致**：大多数工作仅通过提取要点（key points）生成跨图表问题，导致查询与证据之间存在严重的词汇重复，且推理链常发生主体漂移（subject mismatch），如从一个群体的人口统计推断到另一个不兼容的子集（见图1）。
  - **缺乏结构化信息**：现有基准主要针对表格（textual tables），而非实际多模态图表（如折线图、柱状图等），忽略了图表中视觉编码（颜色、形状、空间布局）带来的复杂性和弱结构性。
- **研究意义**：构建一个能真实反映多模态推理难度、逻辑连贯且可控粒度的跨图表RAG基准，对推动图表理解和检索增强生成系统的发展至关重要。

## 2. 论文提出的方法论

### 核心思想
- 通过构建**层次化知识图谱（Hierarchical Knowledge Graph, HKG）** 显式地捕获图表内的实体及其关系，并组织成按信息粒度分层的结构；然后利用**结构感知的路径采样（Structure-aware Path Sampling）** 生成语义连贯、逻辑严密的多跳推理路径，最后基于这些路径合成高质量问答对。

### 关键技术细节

1. **层次化图表知识图谱构建**
   - **图表内图谱构建**：使用VLM（如Qwen3-VL）提取图表中的实体（如标题、图例、数据点等），并为每个实体分配一个粒度层级 \( l_v \in \{0,1,\dots,L\} \)，层级越小表示信息越宏观（如标题），越大表示越微观（如具体数据点）。
   - **边定义**：同层实体之间建立“关联关系”（intra-level edges），不同层实体之间建立“语义递进关系”（inter-level edges），构成多图（multigraph）。每一条边是一个关系三元组 \((v, r, u)\)。
   - **全局整合**：对所有图表子图取并集，并按层级合并相同实体（但不对跨层实体做统一化处理，因为同一实体在不同图表中可能具有不同信息粒度）。

2. **结构感知路径采样与问答生成**
   - **锚点选择**：使用改进的PageRank算法计算实体重要性，其中转移概率同时考虑边的密度（connectivity）和源图表多样性（source diversity），避免仅依赖单图表中高连接度实体。最终按稳定分布 \(\pi\) 采样起始实体 \(v_1\)。
   - **下一跳策略**：每一步扩展时，采样概率由三个因子共同决定：
     - 锚点重要性 \(\pi(v_{t+1})\)
     - 语义主题连贯性 \(\phi_{\text{sem}}\)（当前子路径与候选边的余弦相似度）
     - 粒度控制函数 \(\phi_{\text{gran}}\)（可动态调节，例如倾向向上/向下层级移动，从而控制查询粒度）
   - **终止条件**：达到最大跳数 \(T\) 或无出边时停止。
   - **QA合成**：对同一起始实体采样多个路径，打包后利用VLM生成问题、答案、解释及显式证据引用。问题要求上下文独立、无指代（如“这个”、“那个”），并需对原始文本进行改写以减少词汇重叠。最后经自动后验证（检查答案是否被证据支持且有意义）过滤，保留通过率为0.77的样本。

## 3. 实验设计

### 数据集 / 场景
- **来源**：从ChartMRAG (Yang et al., 2025) 和 ChartQA-Pro (Masry et al., 2025) 收集原始图表，经VLM过滤视觉清晰度、语义丰富度及去重后，得到 **806张图表** 的语料库。
- **构建的HKG规模**：8802个实体，21436条关系，分为4个层级。
- **最终Benchmark**：ChartWalker-Bench，包含 **564个问答对**，分为4种查询类型：
  - Fact Check（事实核查）
  - Manipulation（算术运算：求和、平均、排名、比较）
  - Analysis（模式分析、趋势、关联）
  - Complex Reasoning（复杂多跳推理）
- **统计特性**：平均跨2.30个图表，2.81个推理分支，Complex Reasoning子集难度最高（平均3.12个图表、5.14跳、主观难度3.0）。

### 对比方法
- **检索器（Retriever）**：
  - **Plain RAG**：BM25（稀疏检索）、文本嵌入（Qwen3-Embedding-8B）、视觉-语言嵌入（Qwen3-VL-Embedding-8B）
  - **图增强 RAG**：RagAnything（local/mix）、HippoRAG（基于个性化PageRank的图检索器）
- **生成器（Generator）**：
  - Qwen3-VL-8B、Qwen3-VL-32B、GPT-4o
- **Agent基线（ChartWalker-Agent）**：基于Qwen2.5-VL-3B，使用PPO训练，在重新构建的标准KG（不含层次信息）上进行图导航。

### 评估指标
- **召回率 Recall@k**：检索到的top-k图表中是否包含标注的证据图表。
- **正确率 Correctness@k**：生成器基于top-k检索结果输出的答案是否正确（使用LLM-as-a-Judge范式）。
- 主要报告 k=5 和 k=10。

## 6. 论文的主要结论与发现

- **现有RAG系统在跨图表任务上表现欠佳**：最佳配置（GPT-4o + VL-Embedding）总正确率仅约64%，而复杂推理子集正确率低于30%。
- **多模态表示优于复杂检索启发式**：VL-Embedding在大多数情况下超越BM25和纯文本嵌入，表明直接将查询与图表图像对齐更有效。
- **图增强检索带来显著增益**：HippoRAG在图谱上传播相关性能有效召回分散的证据，尤其是分析类和复杂推理类查询（最佳R@10分别达70.66%和51.38%）。

- **静态检索难以满足多跳需求**：即使在k=10时，仍然存在大量漏检，说明需要动态、自适应的检索策略。

## 7. 优点（方法与实验设计亮点）

1. **层次化知识图谱设计新颖**：按信息粒度组织实体，能够显式控制查询的抽象层次，避免采样到语义不连贯的路径，解决了现有方法中主体漂移的问题。
2. **结构感知路径采样算法**：综合考虑实体重要性（改进PageRank）、语义连贯性（余弦相似度）和粒度控制（可调函数），生成的推理路径逻辑严密、可解释性强。
3. **构建了首个逻辑严谨的跨图表RAG基准**：覆盖多种查询类型，并附有显式推理链和证据引用，便于诊断系统在哪个环节失败。
4. **提供了VLM-based Agent基线**：采用PPO训练，在部分可观察马尔可夫决策过程（POMDP）下进行图探索，避免了长上下文带来的梯度方差问题，为未来Agent设计提供了参考。
5. **实验对比充分**：涵盖了从稀疏检索、稠密检索、图增强检索到Agent的多种范式，以及不同规模的生成器（3B~大型闭源），结论具有较高可靠性。
