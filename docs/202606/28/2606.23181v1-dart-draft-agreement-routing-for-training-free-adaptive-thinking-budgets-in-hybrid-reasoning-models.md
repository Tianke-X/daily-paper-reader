## 摘要
混合推理模型可以直接回答，也可以花费额外的token进行扩展思考。一个实用的路由器应该为每个查询在这些模式之间做出选择，以便简单问题避免不必要的推理，复杂问题获得足够的预算来完成回答。现有路由器正向这个方向发展，但它们通常需要标注的训练数据或预先固定思考预算，忽略了来自模型本身的答案级证据。我们提出了DART，一个无需训练的路由框架，它采样两个廉价的“无思考”草稿，当草稿一致时接受直接回答，当不一致时根据草稿熵预测思考预算。在主要比较中，DART在大多数设置下保持或提高了始终思考的准确性，同时减少了思考token的使用。在数学推理中，奥林匹克级别问题的准确性提高了最多+9.0个点，同时思考token减少了15-69%。在基于执行等价性的代码推理中，准确性提高了最多+22.5个点，同时思考token减少了51-63%。Stage~1信号跨越模型规模（0.6B-32B）、模型系列和仅API托管的设置，无需标注数据，也无需梯度更新。

## Abstract
Hybrid reasoning models can answer directly or spend extra tokens on extended thinking. A practical router should choose between these modes for each query, so easy problems avoid unnecessary reasoning and hard problems receive enough budget to finish the answer. Existing routers move in this direction, but they typically require labeled training data or fix thinking budgets up front, ignoring answer-level evidence from the model itself. We introduce DART, a training-free routing framework that samples two cheap no-think drafts, accepts direct answering when the drafts agree, and predicts a thinking budget from draft entropy when they disagree. Across the main comparisons, DART preserves or improves always-thinking accuracy in most settings while reducing thinking-token use. On math reasoning, accuracy improves by up to $+$9.0 points on Olympiad-level problems while thinking tokens drop 15-69%. On code reasoning under execution-based equivalence, accuracy improves by up to +22.5 points while thinking tokens drop 51-63%. The Stage~1 signal extends across model scales (0.6B-32B), model families, and API-only hosted settings, with no labeled data and no gradient updates required.

# DART: 用于混合推理模型的免训练自适应思考预算路由框架 — 详细中文总结

## 1. 核心问题与整体含义（研究动机与背景）

混合推理模型（如 Qwen3、DeepSeek-V3.2）支持两种推理模式：**直接回答（No-Think）** 和 **扩展思考（Think）**。实际部署中，固定使用“始终思考”模式会导致简单问题浪费大量 tokens，且在多查询场景下无法承受高额计算成本。理想路由器应能根据每个查询的难度自适应地选择模式或分配思考预算，在保证准确率的同时最小化计算开销。现有路由器存在两个主要问题：① 需要标注训练数据或针对性微调，无法适应模型更换或 API-only 场景；② 预先固定思考预算，缺乏基于模型输出的动态调整。本文提出 **DART（Draft-Agreement Routing for Thinking）**，一种完全免训练的路由框架，利用廉价 No-Think 草稿之间的“一致性”作为难度代理，动态决定是否跳转到 Think 模式，并在需要时根据草稿熵预测个性化思考预算。

## 2. 方法论

### 核心思想
DART 分为两阶段：
- **Stage 1 (SC-Route)**：采样 K 个（默认 K=2）No-Think 草稿，若它们的提取答案在给定等价函数下完全一致，则认为查询足够简单，直接接受其中一个草稿答案，不产生任何思考 token。
- **Stage 2 (Entropy-Budget Allocation)**：仅当 Stage 1 判定不一致时才触发，此时根据所有草稿的平均 token 级熵值，通过无标签保序回归（isotonic regression）预测一个查询特定的思考预算，并以安全因子 γ=1.5 放大，在 Think 模式下生成答案，且推理轨迹与最终答案分离生成，避免截断伪影。

### 关键技术细节
- **等价函数 (eq)**：唯一领域特定组件。对数学使用字符串归一化后比较，对代码使用沙盒执行结果匹配。
- **熵计算**：所有 K 个草稿上每个 token 的模型输出分布负平均。
- **预算映射**：对保序回归拟合的函数加上安全因子，保证预测预算通常大于实际需求。
- **超参数**：K、温度 T、预算上限等均为默认设置（K=2, T=0.6-0.8 按模型默认，预算由熵预测动态设定）。

### 算法流程（文字描述）
1. 给定查询 q，从模型中以 No-Think 模式采样 K 个草稿。
2. 提取每个草稿的答案对象。
3. 若所有提取答案两两等价（Stage 1 同意），则直接返回第一个草稿答案（零思考 token）。
4. 否则，计算草稿熵，通过预拟合的保序回归映射得到基础预算，乘以 1.5 得最终预算。
5. 使用 Think 模式生成响应，预算到达时注入 `</think>` 标签，分离出答案轨迹。
6. 返回 Think 答案。

- **自提示路由（MC-Binary, MC-Conf）**：由模型自我判断是否需要思考

## 6. 主要结论与发现

1. **性能提升**：在 14 个模型-基准对中，DART 在 13 个上达到或超过始终思考（AT）的准确率，最高提升：数学 +9.0 分（OlympiadBench Qwen3-14B），代码 +22.5 分（HumanEval Qwen3-32B）。
2. **token 大幅节省**：思考 token 减少 15%-69%；总量（含草稿）平均节省 35%-55%。
3. **跨模型通用性**：至少 0.6B~32B 的 Qwen3 及 DeepSeek-V3.2 均有效，无需重新训练。
4. **一致性信号优势**：草稿一致性与 AT 正确率在开放式任务上强相关（rpb ≈ 0.5-0.8），且比熵或长度等间接代理更有效。
5. **无标签、无梯度**：完全免训练，适合 API-only 部署。
6. **错误主要来源**：错误接受（草稿一致但答案错误）占错误的大部分（61%），是未来改进方向。

## 7. 优点

- **彻底免训练**：不依赖标注数据、模型微调或梯度更新，可无缝应用于任何暴露 No-Think/Think 切换的模型，包括仅文本 API 的封闭模型。
- **两阶段设计合理**：Stage 1 低成本筛选简单查询，Stage 2 自适应分配预算，分离推理与答案避免截断伪影（解决以往路由比较中的测量偏差）。
- **信号鲁棒性强**：草稿一致性作为难度代理在多种模型、规模、难度水平上保持高精度，且监督路由器分析证实其显著优于熵或长度。
- **实践友好**：只有两个可调超参数（K, 温度）且默认值鲁棒；等价函数唯一领域特定部分，且已有现成归一化或执行工具。
- **透明度高**：提供 oracle 分析、错误分类、测量伪影量化等深入诊断，结论可信。
- **代码和数学任务均有效**，且代码任务下提升尤为明显（AT 在短代码上反而可能不如 NT，一致性路由自动保留高置信 No-Think 答案）。

## 8. 不足与局限

- **任务范围有限**：方法专门针对开放式生成任务（数学推理、代码生成），不适用于多项选择题（如 ARC, MMLU-Pro, GPQA），因为选项空间小导致随机一致会抬高错误接受率。论文明确将此界外，不打算加入概率阈值。
- **等价函数依赖**：路由效果依赖任务恰当的答案提取器和等价函数。数学需要 \boxed{} 归一化，代码需要沙盒执行与测试划分。对于无代表性测试集的新领域，可能难以设计可靠等价函数。
- **Stage 2 有额外假设**：完整 DART 需要 token 级概率和思考预算/停止控制，部分托管 API 可能不支持这些接口，此时只能使用 Stage 1 路由（仍能获得部分节省，但可能不如完整版本）。
- **错误接受问题**：当两个 No-Think 草稿一致但答案都错时（约 61% 错误），DART 直接接受而不进入 Think，导致准确性损失。需要更精细的置信校准或额外抽样来缓解。
- **评估广度依然有限**：主要实验集中在 Qwen3 系列和 DeepSeek-V3.2 的一个版本，未在更多模型家族（如 Llama 系列、Gemma 系列）上验证。代码基准 HumanEval/MBPP 仅使用固定的 50/50 测试划分，随机划分可能引起变化。
- **部署成本计算抽象**：思考 token 节省未必直接对应云服务的经济成本；草稿产生、并行控制、等价检查等 overhead 未细致量化（仅给出了单卡 A100 上的延迟数据，且未提及其他硬件或服务化部署情形）。
- **路由参考集需求**：HumanEval 需要预留 4 题作路由测试，实际应用中可能需要用户自行提供，可能增加部署工作量。
- **无统计显著性报告**：所有结果都为点估计，未提供置信区间或多次运行的标准差（除随机路由外），且部分消融实验仅单次运行。
