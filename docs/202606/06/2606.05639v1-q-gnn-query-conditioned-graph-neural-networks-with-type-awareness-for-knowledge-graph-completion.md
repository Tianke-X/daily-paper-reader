## 摘要
知识图谱补全（KGC）旨在从不完全的知识图谱中预测缺失的三元组，这对于下游应用至关重要。近年来，基于图神经网络（GNN）的方法通过在查询中心的局部子图上进行消息传递取得了显著成功。然而，在实践中，查询由实体和关系共同定义，两者都携带推理不可或缺的信息，但这些方法仅依赖查询关系作为引导信号，而查询实体中固有的信息并未被利用来指导推理——实体仅作为子图提取的结构锚点。为此，我们从两个角度将查询实体信息融入推理过程：第一是结构上下文，即实体周围的邻域结构和关系模式，由专门的上下文编码器编码并用于调节消息；第二是实体的语义类型，由大语言模型推断，并纳入注意力计算和最终评分，以提供类型级的先验约束。这两类信息共同使得推理过程能够同时由查询关系和查询实体引导。在标准基准上的实验结果表明了所提出的Q-GNN的有效性。

## Abstract
Knowledge Graph Completion (KGC) aims at predicting missing triplets from incomplete knowledge graphs, which is crucial for downstream applications. Recently, Graph Neural Network (GNN)-based methods have achieved remarkable success by performing message passing over query-centered local subgraphs. However, in practice, a query is jointly defined by both the entity and the relation, with both carrying information indispensable for reasoning, yet these methods rely solely on the query relation as the guiding signal, while the information inherent in the query entity is not leveraged to guide inference - the entity serves merely as a structural anchor for subgraph extraction. To this end, we incorporate query entity information into the reasoning process from two perspectives: the first is structural context, i.e., the neighboring structure and relation patterns around the entity, which is encoded by a dedicated context encoder and used to modulate messages; the second is semantic type of the entity, inferred by a large language model, which is incorporated into attention computation and final scoring to provide type-level prior constraints. Together, these two sources of information enable the reasoning process to be guided by both the query relation and the query entity. Experimental results on standard benchmarks demonstrate the effectiveness of the proposed Q-GNN.

## 2. 论文提出的方法论

### 2.1 核心思想
从两个层面引入查询实体信息：
- **结构上下文**：通过上下文编码器提取查询实体周围的邻域关系和模式，生成查询上下文表示，用于调制候选实体编码过程。
- **语义类型**：利用大语言模型（如DeepSeek-V3）为每个实体推断类型标签，通过类型感知注意力机制和类型特定解码器融入推理。

### 2.2 关键技术细节

**a) 类型引导上下文过滤器（Type-Guided Context Filter）**
- 构建类型图（以类型为节点，单关系连接），在类型图上搜索从查询头类型到其他类型的所有路径（长度≤K）。
- 引入三个度量指标筛选有价值类型路径：
  - **置信度（Conf）**：路径实例中连接目标尾类型的比例。
  - **焦点（Focus）**：路径实例中连接特定尾类型的集中程度。
  - **语义相关性（Semantic）**：路径上各类型语义嵌入（由Qwen3-Embedding-8B生成）与尾类型嵌入的几何平均余弦相似度。
- 保留满足阈值的路径对应的三元组，形成降噪后的上下文图 \( \mathcal{G}_{\text{context}} \)。

**b) 查询上下文编码器（Query Context Encoder）**
- 在 \( \mathcal{G}_{\text{context}} \) 上沿尾→头方向进行 \( L_c \) 层消息传递，逐步扩展子图。
- 每层消息：\( m_{e_t \to e_h}^{(l)} = h_{e_t}^{(l-1)} + e_r^{(l)} \)。
- 类型感知注意力：将尾实体类型嵌入 \( e_{\tau(e_t)} \) 和查询关系嵌入 \( e_{r_q} \) 融入 logits 计算，调节不同上下文片段的重要性。
- 最终得到查询实体表示 \( q_h \)。

**c) 查询导向编码器（Query-Oriented Encoder）**
- 在全知识图谱上从头→尾方向进行 \( L_1 + L_2 \) 层消息传递。
- 采用**特征线性调制（FiLM）**：由查询上下文表示 \( q_h \) 和查询关系嵌入 \( e_{r_q} \) 通过 MLP 生成调制参数 \( \gamma_q^{(l)} \) 和 \( \beta_q^{(l)} \)，对每个消息进行仿射变换：
  \[
  \tilde{m}'_{e_h \to e_t}^{(l)} = \text{LayerNorm}\big((1+\tanh(\gamma_q^{(l)})) \odot m'_{e_h \to e_t}^{(l)} + \beta_q^{(l)}\big)
  \]
- 类型感知注意力（此时用头实体类型）进行消息聚合，最终得到每个候选实体的表示 \( h_e \)。

**d) 类型特定解码器（Type-Specific Decoder）**
- 为每种类型 \( t \) 学习一个评分权重向量 \( w_t \)，候选实体得分 \( s(e|e_q, r_q) = w_{\tau(e_q)}^\top h_e \)。
- 先通过类型-关系-类型掩码过滤掉训练数据中未出现过的类型组合。

**e) 训练目标**
- 多类 log 损失（multi-class log loss）：
  \[
  \mathcal{L} = \sum_{(e_q, r_q, e_t)} \left( -s(e_t|e_q,r_q) + \log\sum_{e\in\mathcal{E}} \exp(s(e|e_q,r_q)) \right)
  \]
