## 摘要
多模态空间推理通常依赖于长链的中间文本和视觉思维，其中累积的视觉标记和密集的跨模态注意力会带来巨大的计算和内存开销。为了解决这一挑战，我们提出了频谱渐进式思维流（SpecFlow），一种新颖的轻量级多模态空间推理框架，该框架在固定大小的离散余弦空间中表示中间视觉思维。通过利用强大的能量压缩，SpecFlow保留了全局布局和关系结构，仅在需要更高空间精度时引入高频细节。为了使视觉状态演化与语言意图对齐，无分类器引导使自回归文本思维能够指导基于流的视觉工作空间/状态更新，而无需扩展上下文。因此，SpecFlow保持了一个有界的视觉工作空间，其更新仅依赖于当前视觉状态和累积的文本轨迹，从而实现了稳定延迟和内存使用与推理深度无关的长时推理。实验结果表明，SpecFlow在实现具有竞争力或更优推理性能的同时，将计算和KV缓存成本降低了最多2.1倍。

## Abstract
Multimodal spatial reasoning often relies on long chains of intermediate textual and visual thoughts, where accumulating visual tokens and dense cross-modal attention incur substantial computation and memory overhead. To address this challenge, we propose Spectral-Progressive Thought Flow (SpecFlow), a novel lightweight multimodal spatial reasoning framework that represents intermediate visual thoughts in a fixed-size discrete cosine space. By exploiting strong energy compaction, SpecFlow preserves global layout and relational structure while introducing high-frequency details only when increased spatial precision is required. To align visual state evolution with linguistic intent, classifier-free guidance enables autoregressive textual thoughts to steer flow-based updates of the visual workspace/state without expanding the context. As a result, SpecFlow maintains a bounded visual workspace whose updates depend only on the current visual state and accumulated textual trace, enabling long-horizon inference with stable latency and memory usage independent of reasoning depth. Empirical results show that SpecFlow achieves competitive or superior reasoning performance while reducing computation and KV cache costs by up to 2.1 times.

### 2. 论文提出的方法论：核心思想与关键技术

*   **整体含义**：论文提出了 **频谱渐进式思维流 (SpecFlow)**，一种全新的多模态空间推理范式。其核心思想是将中间视觉思维表示为一个**固定大小的连续视觉工作空间**，该工作空间在离散余弦谱域中通过流匹配（Flow Matching）进行更新，不再向自回归上下文追加视觉Token，从而解耦内存消耗与推理深度。

*   **核心技术细节**：
    1.  **频谱-渐进式频率分配 (Spectral-Progressive Frequency Allocation)**：
        *   **思想**：利用块余弦变换（Blockwise Cosine Transform）强大的能量压缩特性。图像的能量主要集中在中低频系数中，保留了全局布局和结构；高频分量代表细节。
        *   **机制**：定义一个随时间变化的二进制掩码 `M(t)`，开始时只激活少量低频系数，随着推理步数 `t` 增加，逐步解锁中高频带。
        *   **好处**：早期推理关注全局结构，效率高；后期需要高空间精度时，再引入高频细节，实现从粗到细的推理过程。

    2.  **余弦空间流匹配 (Cosine-Space Flow Matching)**：
        *   **思想**：将视觉工作空间的演化建模为余弦谱域中的确定性连续时间流。
        *   **公式**：定义一个速度场 `uθ`：`dX(t)/dt = uθ(ẼX(t), t, c)`。其中 `X(t)` 是余弦系数，`ẼX(t)` 是经过频谱掩码 `M(t)` 过滤后的状态，`c` 是文本条件的信号。
        *   **训练目标**：采用标准的流匹配损失 `L_FM`，旨在让模型预测的流速接近数据点到噪声点的线性插值产生的恒定目标流速 `Ẋ_t = X_0 - X_1`。
        *   **推理过程**：从上一个视觉状态对应的余弦系数开始，使用固定步长的ODE求解器（如欧拉法）迭代更新，完成后通过逆余弦变换得到新的视觉工作空间。**整个更新过程只依赖当前状态和累积的文本轨迹，不依赖历史视觉状态，因此早期视觉状态可以被安全丢弃。**

    3.  **基于分类器无引导的文本条件控制**：
        *   **思想**：需要让视觉工作空间（状态）的演化与语言意图对齐，即由文本思维指导视觉更新。
        *   **机制**：采用分类器无引导（Classifier-Free Guidance, CFG）。在速度场层面，结合有条件的和无条件的速度预测，形成一个指导性的速度 `u_guided`。
        *   **公式**：`u_guided = u_θ(·, t, ∅) + w * [u_θ(·, t, c_i) - u_θ(·, t, ∅)]`。其中 `c_i` 是第 `i` 步的文本条件（当前文本思维），`w` 是引导强度。
        *   **推理流程**：
            *   自回归生成文本思维 `ˆt_{i+1}` 的条件是累积的文本历史 `ˆt≤i` 和当前的视觉状态 `ˆv_{i+1}`。
            *   流匹配模型生成视觉状态 `ˆv_{i+1}` 的条件是当前文本上下文 `c_i` 和上一个视觉状态 `ˆv_i`。
            *   这种交替机制（AR(Text) -> FM(Visual) -> AR(Text)...）使得推理可通过**并行化的视觉状态更新**取代**序列化的视觉Token生成**。
