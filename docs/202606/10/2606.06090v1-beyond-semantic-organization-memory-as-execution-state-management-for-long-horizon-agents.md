## 摘要
基于LLM的智能体越来越多地处理具有相互依赖决策的长周期任务，其中每个动作都会重塑未来的约束条件，而中间错误可能级联放大。现有的RAG和智能体记忆系统通过语义相似性组织历史记录，在决策时检索内容相关的条目。我们认为这种设计不匹配执行状态依赖关系：它碎片化决策轨迹，混合有效与错误轨迹，阻碍连贯的状态重建和错误隔离。我们提出MAGE（记忆作为智能体引导的探索），一种主动的执行状态管理器，将交互存储为层次化状态树。智能体从活动的根到当前路径推导其状态，结合子目标摘要、近期轨迹以及来自先前分支的提示。四种耦合操作维护这棵树：Grow记录新轨迹，Compress总结已完成子目标，Maintain验证摘要，Revise恢复目标边界并在新分支上继续。这种设计在限制上下文增长的同时，保持状态完整性，并将有缺陷的片段从活动路径中隔离。在MemoryArena上的实验表明，MAGE将平均任务成功率提高了7.8至20.4个百分点，同时将令牌消耗减少了55.1%。

## Abstract
LLM-based agents increasingly tackle long-horizon tasks with interdependent decisions, where each action reshapes future constraints and intermediate errors can cascade. Existing RAG and agent memory systems organize histories by semantic similarity, retrieving content-relevant entries at decision time. We argue that this design mismatches execution-state dependencies: it fragments decision trajectories and mixes valid and erroneous traces, hindering coherent state reconstruction and error isolation. We propose MAGE (Memory as Agent-Guided Exploration), an active execution-state manager that stores interactions in a hierarchical state tree. The agent derives its state from the active root-to-current path, combining subgoal summaries, recent traces, and hints from prior branches. Four coupled operations maintain the tree: Grow records new traces, Compress summarizes completed subgoals, Maintain validates summaries, and Revise restores a target boundary and resumes on a new branch. This design bounds context growth while preserving state integrity and isolating flawed segments from the active path. Experiments on MemoryArena show that MAGE improves the average task success rate by 7.8--20.4 pp over baselines, while reducing token consumption by 55.1%.

# 论文总结：MAGE——超越语义组织，将记忆作为长周期智能体的执行状态管理

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：现有LLM智能体在执行长周期、多步骤相互依赖的任务（如购物规划、旅行路线、渐进式搜索、形式推理）时，传统基于RAG或记忆系统的方法按语义相似性组织历史记录，在决策时检索相关条目。这种设计不匹配执行状态依赖关系，导致两个主要问题：
  - **状态碎片化**：决策轨迹被割裂，无法重建完整的执行状态；
  - **错误隔离困难**：正确与错误的轨迹混合在同一记忆池中，使得错误难以追溯和回滚。
- **背景**：近期基准测试（MemoryArena）显示，许多记忆系统性能甚至低于简单保留全部历史（长上下文）的方法，且消耗大量token。作者认为根本原因在于共享的设计哲学——依赖语义关系而非执行路径来组织和检索信息。
- **整体含义**：需要将记忆从被动的事实存储转变为主动的执行状态管理，以保持状态完整性和错误隔离能力。

## 2. 方法论的提出：核心思想、关键技术细节
- **核心思想**：MAGE将智能体的交互历史组织为一个两层层次状态树，智能体的当前执行状态从活动的根到当前路径派生，结合子目标摘要、最近原始轨迹和来自兄弟分支的执行提示。通过四个耦合操作维护树，实现上下文压缩、验证错误和选择性回滚。
- **关键技术细节**：
  - **两层层次执行状态树**：
    - 底层：记录每一步的原始动作-观察节点，保持执行顺序和状态依赖。
    - 顶层：为已完成的子目标片段存储摘要节点，实现有边界压缩，同时保留恢复所需的决策边界。
  - **执行状态S**由三部分组成：
    - 压缩状态C：顶层路径上的摘要，记录已完成子目标。
    - 原始状态R：最近未压缩的底层节点，提供精细上下文。
    - 执行提示H：暴露当前节点的兄弟分支（之前探索过的路径）和诊断反馈，帮助避免重复错误。
  - **四个状态转换操作（算法流程）**：
    - **Grow**：执行新动作后，自动在底层树中添加节点。若已有相同内容的兄弟节点则合并（避免重复），否则新建节点。更新原始状态R和执行提示H。
    - **Compress**：当子目标完成或原始状态R超过阈值时，将当前底层节点序列压缩为顶层摘要节点，清空R，将摘要追加到C，并更新H为子目标级别的提示。
    - **Maintain**：在每次压缩后立即执行，使用LLM验证新摘要及其覆盖的追踪是否准确、完整、无错误。若验证失败，记录诊断反馈并返回失败信号和修复目标步骤ID。
    - **Revise**：当Maintain失败或智能体主动检测到错误时，将指针回滚到目标步骤，恢复压缩状态C和原始状态R到该边界，注入诊断反馈，并从该点开始新分支（兄弟路径）。错误片段被排除在活动路径之外，有效隔离错误。
- **闭环机制**：Grow → Compress → Maintain → Revise（若需要） → 继续Grow，形成主动管理循环，模仿人类认知中的分块、监控和选择性修正过程。

## 3. 实验设计：数据集、基准、对比方法
- **数据集/基准**：MemoryArena（He et al., 2026），一个专门为相互依赖的长周期智能体任务设计的基准，包含四个领域：
  - Bundled Web Shopping（捆绑购物）：后续购买依赖于之前选择的商品约束。
  - Group Travel Planning（团体旅行规划）：多人行程需满足共享偏好和依赖。
  - Progressive Web Search（渐进式搜索）：多步搜索依赖于之前子问题的答案。
  - Formal Reasoning（形式推理）：数学/物理证明依赖于之前完成的步骤。
- **对比方法**：分为三类：
  - **长上下文**（Long Context）：保留完整历史。
  - **RAG系统**：HippoRAG2（基于知识图谱和PageRank多跳检索）、MemoRAG（轻量级模型生成检索线索）。
  - **记忆系统**：Mem0（提取事实构建图记忆）、ReasoningBank（提炼可复用推理策略）、MemoryOS（层次存储层动态更新）、SimpleMem（语义压缩和递归整合）。
- **所有方法使用相同的骨干模型**：Qwen3.6-27B（ReAct框架），嵌入模型使用Qwen3-8B-Embedding。
- **评估指标**：任务成功率（SR，%）、任务进度分数（PS，%）、总token消耗（包括提示和生成token）。
