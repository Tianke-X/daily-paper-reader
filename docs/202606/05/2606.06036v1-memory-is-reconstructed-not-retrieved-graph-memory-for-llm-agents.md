## 摘要
尽管近年来取得了进展，LLM智能体在处理长交互历史中的推理仍然面临挑战。当前基于记忆增强的智能体依赖于静态的“检索-然后-推理”范式，这种僵化的流水线设计阻碍了它们根据推理过程中发现的中间证据动态调整记忆访问。为填补这一空白，我们提出了MRAgent框架，该框架将关联记忆图与主动重建机制相结合。我们将记忆表示为线索-标签-内容图，其中关联标签作为语义桥梁连接细粒度线索与记忆内容。基于这一结构，我们的主动重建机制将LLM推理直接融入记忆访问，使智能体能够根据累积的证据迭代探索和剪枝检索路径。这确保了记忆检索能动态适应推理上下文，同时避免无约束扩展导致的组合爆炸。在LoCoMo基准和LongMemEval基准上的实验表明，相较于强基线方法，我们的方法取得了显著提升（最高提升23%），同时大幅降低了令牌和运行时间成本，突显了主动关联重建在长期记忆推理中的有效性。

## Abstract
Despite recent progress, LLM agents still struggle with reasoning over long interaction histories. While current memory-augmented agents rely on a static retrieve-then-reason paradigm, this rigid pipeline design prevents them from dynamically adapting memory access to intermediate evidence discovered during inference. To bridge this gap, we propose MRAgent, a framework that combines an associative memory graph with an active reconstruction mechanism. We represent memory as a Cue-Tag-Content graph, where associative tags serve as semantic bridges connecting fine-grained cues to memory contents. Operating on this structure, our active reconstruction mechanism integrates LLM reasoning directly into memory access, allowing the agent to iteratively explore and prune retrieval paths based on accumulated evidence. This ensures that memory retrieval is dynamically adapted to the reasoning context while avoiding combinatorial explosion caused by unconstrained expansion. Experiments on the LoCoMo benchmark and LongMemEval benchmark demonstrate significant improvements over strong baselines (up to 23%), while substantially reducing token and runtime cost, highlighting the effectiveness of active and associative reconstruction for long-horizon memory reasoning.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将记忆访问从静态的“检索-然后-推理”转变为动态的“推理中主动重建”。LLM在推理过程中，通过迭代地访问和评估一个结构化的记忆图，主动地、有选择地重建与当前问题相关的记忆证据链。
- **关键技术细节**：
    1.  **Cue-Tag-Content (CTC) 关联记忆图**:
        - 将记忆组织成一个异构图，包含三种节点：
            - **Cue (线索)**：细粒度的关键词，如实体、属性。
            - **Tag (标签)**：作为“语义桥梁”，连接Cue和具体的Content。它是一个中间表示，总结了从Cue到Content的关联关系（如“爱好”、“职业经历”）。
            - **Content (内容)**：具体的记忆单元，分为**Episodic (情景记忆)** 和 **Semantic (语义记忆)**。
        - 这种 **Cue -> Tag -> Content** 的两阶段结构是关键优势。LLM可以**先**通过Cue和Tag来判断检索方向是否相关，以此剪枝掉不相关的分支，避免直接加载大量细粒度Content所产生的高昂成本。
    2.  **主动重建机制 (Active Reconstruction Mechanism)**:
        - 定义了两个核心概念：
            - **重建状态 (Reconstruction State)**：$S^{(t)} = (Z^{(t)}, H^{(t)})$。其中$Z^{(t)}$是当前可用的候选节点集，$H^{(t)}$是当前已重建的证据上下文。
            - **遍历动作 (Traversal Actions)**：一组预定义的图遍历操作，如`Cue -> Tag`，`(Cue, Tag) -> Content`，`Content -> (Cue, Tag)`。这些动作允许智能体在图中向前（从线索到内容）或向后（从内容发现新线索）地探索。
        - **工作流程**：
            1.  **初始化**：从初始查询中提取Cue，定位到图上的起始节点。
            2.  **LLM推理与动作选择**：LLM根据当前的查询和已积累的上下文，选择最有希望的下一个遍历动作。
            3.  **受控遍历**：根据选定的动作，在图中执行遍历，获得新的候选节点$Z(t+1)$。
            4.  **LLM路由与状态更新**：LLM评估新候选节点，选择最相关的部分，更新上下文$H(t+1)$，并剪枝无关路径。这个过程会迭代进行，直到LLM判断证据足够回答问题（通过停止条件判断）。
    3.  **多粒度记忆层 (Multi-Granular Memory Layers)**:
        - 将记忆分为Episodic, Semantic和Abstract (Topic)三个层次，使得智能体可以从不同抽象级别访问信息，更加灵活。
    4.  **理论分析**：
        - 论文从近似理论的角度证明了，在相同信息获取预算下，主动检索策略在表达能力上严格优于被动检索策略。即，主动策略可以学习到的函数类严格包含被动策略所能学习的。
