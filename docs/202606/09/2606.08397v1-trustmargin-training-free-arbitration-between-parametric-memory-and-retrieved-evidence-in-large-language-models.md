## 摘要
大语言模型利用参数化记忆和检索证据回答知识密集型问题，但两种来源均非完全可靠。检索可以填补知识空白，但干扰性段落可能会覆盖正确的闭卷答案。我们将这种生成后的冲突视为答案层面的来源仲裁：给定来自同一固定模型的Direct和RAG答案，决定信任哪个来源。我们提出TRUSTMARGIN，一种无需训练、即插即用的仲裁层，它使用模型自身的似然度对两个现有候选答案进行评分。它结合了参数化先验边际（测试记忆是否接受检索到的答案）和证据绑定边际（折扣仅段落显著性并衡量问题特定支持）。TRUSTMARGIN无需微调、外部评判或额外生成即可在Direct和RAG之间进行选择。在2WIKIMQA和CWQA上，使用三种LLaMA规模，TRUSTMARGIN consistently优于Direct生成和BM25-RAG，弥补了Direct/RAG Oracle差距的一部分，并泛化到多种无训练RAG流水线。

## Abstract
Large language models answer knowledge-intensive questions using both parametric memory and retrieved evidence, but neither source is uniformly reliable. Retrieval can fill knowledge gaps, yet distracting passages may override correct closed-book answers. We study this post-generation conflict as answer-level source arbitration: given Direct and RAG answers from the same frozen model, decide which source to trust. We propose TRUSTMARGIN, a training-free, plug-and-play arbitration layer that scores the two existing candidates with the model's own likelihoods. It combines a parametric-prior margin, which tests whether memory accepts the retrieved answer, with an evidence-binding margin, which discounts passage-only salience and measures question-specific support. TRUSTMARGIN selects between Direct and RAG without fine-tuning, external judges, or additional generation. Across 2WIKIMQA and CWQA with three LLaMA scales, TRUSTMARGIN consistently improves over Direct generation and BM25-RAG, recovers part of the Direct/RAG oracle gap, and generalizes to multiple training-free RAG pipelines.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：基于同一冻结模型的三类似然度（闭书似然、证据条件似然、仅上下文似然），计算两个边际并组合成信任分数，决定是否用RAG答案替换Direct答案。
- **关键技术细节**：
  - **参数先验边际 \(M_{prior}\)**：在仅问题提示下计算RAG答案和Direct答案的闭书似然度之差。正值表示记忆接受RAG答案；强负值表示记忆偏好Direct答案。
    \[
    M_{prior} = \ell_D(y_R) - \ell_D(y_D)
    \]
  - **证据绑定边际 \(M_{bind}\)**：对每个候选答案，计算证据条件似然度减去仅上下文似然度，得到问题-证据绑定得分，然后比较两个候选的得分差。它消除仅段落显著性的混淆。
    \[
    \Delta(y) = \ell_R(y) - \ell_C(y), \quad M_{bind} = \Delta(y_R) - \Delta(y_D)
    \]
  - **最终信任分数**：
    \[
    M = M_{prior} + \lambda_{bind} M_{bind}
    \]
    选择规则：若 \(M > \tau\) 则采用RAG答案，否则保留Direct答案。默认 \(\lambda_{bind}=0.5\)，\(\tau=-1.5\)。
- **算法流程**（文字说明）：
  1. 给定问题 \(q\) 和检索段落集 \(P\)，用冻结模型生成Direct答案 \(y_D = G_\theta(q)\) 和RAG答案 \(y_R = G_\theta(q, P)\)。
  2. 对每个候选答案计算三种长度归一化似然度：\(\ell_D(y)\)、\(\ell_R(y)\)、\(\ell_C(y)\)。
  3. 计算 \(M_{prior}\) 和 \(M_{bind}\)，组合为 \(M\)。
  4. 若 \(M > \tau\) 则输出 \(y_R\)，否则输出 \(y_D\)。
- 该方法无需修改模型、重新检索、训练评判器或生成第三个答案，即插即用。

## 7. 优点

- **可解释性**：两个边际具有清晰语义（记忆检查、证据绑定），决策可审计。
- **鲁棒性**：对超参数不敏感（图4稳定区域宽），对检索噪声自动保守。
