## 摘要
检索增强生成（RAG）已成为通过利用外部知识来减轻大型语言模型（LLM）幻觉的重要方法。虽然对于简单查询有效，但传统RAG在处理信息高度碎片化的大规模非结构化语料库时存在困难。基于图的RAG（GraphRAG）利用知识图谱捕获结构关系，从而为复杂推理实现更全面的检索。然而，现有的GraphRAG方法依赖于孤立的、片段级别的提取来构建图，缺乏对整个语料库的全局视角。因此，这些方法经常导致图在主题上不一致、逻辑上冲突以及结构上碎片化，从而降低检索性能。在本文中，我们提出了MemGraphRAG，一种新颖的框架，引入了基于记忆的多智能体系统以确保高质量的图构建。具体来说，MemGraphRAG采用由共享记忆支持的智能体协作社会，在整个提取过程中提供统一的全局上下文。这种机制使智能体能够动态解决逻辑冲突，并维持整个语料库的结构连通性。此外，我们提出了一种针对所构建图定制的记忆感知分层检索算法。在多个基准上的广泛实验表明，MemGraphRAG以相当的性能优于最先进的基线模型。我们的代码可在https://github.com/XMUDeepLIT/MemGraphRAG获取。

## Abstract
Retrieval-Augmented Generation (RAG) has become an essential method for mitigating hallucinations in Large Language Models (LLMs) by leveraging external knowledge. Although effective for simple queries, traditional RAG struggles with large-scale, unstructured corpora where information is highly fragmented. Graph-based RAG (GraphRAG) incorporates knowledge graphs to capture structural relationships, enabling more comprehensive retrieval for complex reasoning. However, existing GraphRAG methods rely on isolated, fragment-level extraction for graph construction, lacking a global perspective on the whole corpus. As a result, these methods frequently lead to thematically inconsistent, logically conflicting, and structurally fragmented graphs that degrade retrieval performance. In this paper, we propose MemGraphRAG, a novel framework that introduces a memory-based multi-agent system to ensure high-quality graph construction. Specifically, MemGraphRAG employs a collaborative society of agents supported by shared memory, which provides a unified global context throughout the extraction process. This mechanism allows agents to dynamically resolve logical conflicts and maintain structural connectivity throughout the corpus. Furthermore, we propose a memory-aware hierarchical retrieval algorithm tailored for the constructed graph. Extensive experiments on multiple benchmarks demonstrate that MemGraphRAG outperforms the state-of-the-art baseline models with comparable efficiency. Our code is available at https://github.com/XMUDeepLIT/MemGraphRAG.

- **研究背景**：检索增强生成（RAG）利用外部知识缓解大语言模型（LLM）的幻觉问题，但传统RAG在处理信息碎片化严重的大规模非结构化语料时，检索到的上下文往往含有噪声、缺乏结构连贯性。图检索增强生成（GraphRAG）通过构建知识图谱捕获结构关系，对复杂推理更具优势，但现有GraphRAG方法存在关键缺陷。
- **核心问题**：现有GraphRAG的图构建过程采用**孤立的、片段级别的提取方式**，缺乏对整个语料库的**全局视角**。这导致三个关键缺陷：
  1. **主题不相关**：提取的三元组偏离核心主题，引入无意义事实。
  2. **逻辑不一致**：同一子图中出现矛盾事实（如互相排斥、时间冲突、粒度冲突）。
  3. **结构碎片化**：图连通性差，孤立节点和断开组件削弱了多跳推理能力。
- **整体意义**：提出MemGraphRAG，通过引入**记忆驱动的多智能体系统**实现全局一致的图构建，提升检索质量和生成准确性，为GraphRAG提供新范式。

## 二、方法论：核心思想、关键技术细节

### 2.1 核心思想
- 将知识图谱构建过程从静态、孤立提取转变为**动态共演化过程**，其中**全局记忆（Global Memory）** 与**层次化索引图（Hierarchical Indexing Graph）** 协同进化。
- 采用**多智能体系统**（提取、冲突检测、冲突解决）与共享记忆交互，实现全局一致性。

### 2.2 关键技术细节

#### （1）全局记忆（M）的三层结构
- **本体层（M_ont）**：存储模式（schema）及提取频率，提供语义结构和全局主题。
- **事实层（M_fac）**：存储具体事实三元组，是推理的主基面。
- **段落层（M_pas）**：保留原始文本段落作为证据支撑。
- 跨层双向索引：模式-实例对齐（Φ: M_fac → M_ont）和事实-证据连接（Ψ: M_fac × M_pas）。

#### （2）多智能体系统（A）
- **提取智能体（A_ext）**：处理文档块，同时生成候选模式、事实和出处段落，存入记忆。
- **冲突检测智能体（A_det）**：监控M_fac，通过语义相似度和结构匹配检测冗余、逻辑矛盾。
- **冲突解决智能体（A_res）**：利用M_ont的模式约束和M_pas的历史证据，进行基于证据的裁决（保留正确事实、添加时间属性、区分粒度）。

#### （3）记忆驱动的图构建流程
- **主题去噪（统一模式过滤）**：新提取的模式先为“候选”，频率超过阈值τ才提升为“稳定”，仅与其关联的事实被激活用于建图。
- **一致性维护（全局裁决）**：A_det检测冲突后，A_res检索出处段落，通过比较原始证据裁决（如丢弃错误事实、添加时间后缀、调整关系粒度）。
- **结构融合（记忆桥接）**：通过基于共享模式类型的边和实体间嵌入相似度边（超过阈值δ_b），连接断开子图。

#### （4）记忆感知的分层检索算法
- **多层记忆过滤**：并行从M_ont、M_fac、M_pas检索候选，仅保留与查询相似度>τ的模式和事实，若无则回退到标准RAG。
- **结构感知节点初始化**：
  - 实体节点：基于其关联的已检索事实平均相似度。
  - 类型节点：语义相似度 × log(度+1)的倒数（抑制高泛化类型）。
  - 段落节点：语义相似度 × 信息密度项（IDF加权，含阻尼因子α=0.05）。
- **个性化PageRank传播**：从初始化分布开始迭代，阻尼因子λ=0.5，收敛后选择Top-K段落和Top-M实体供LLM生成。
