## 摘要
GraphRAG通过将语料组织为显式知识图谱来扩展检索增强生成，从而实现基于图的检索以支持复杂问答。然而，现有框架仅在单个分块内抽取实体和关系，导致跨分块关系（即证据跨越多个段落的关系）在索引中系统性缺失。由于分块组合的指数爆炸，基于LLM的穷举式恢复此类关系不可行。我们提出CrossAug，一种GNN引导的跨分块图增强方法，在查询时检索之前作为离线步骤，用跨分块关系结构丰富GraphRAG索引。CrossAug通过自监督图损坏获得训练监督，使用拓扑感知的GNN对子图进行缺失性评分，并仅对选定的高分区域应用基于证据的LLM补全。在四个多跳和长文档QA基准上对三种基于LLM的GraphRAG框架进行的实验表明，CrossAug持续提升性能，证实了跨分块图增强对于基于检索的问答的益处。我们的代码可在https://github.com/DonFinliani/CrossAug获取。

## Abstract
GraphRAG extends retrieval-augmented generation by organizing corpora as explicit knowledge graphs, enabling graph-based retrieval for complex question answering. However, existing frameworks extract entities and relations within individual chunks, leaving cross-chunk relations -- those whose evidence spans multiple passages -- systematically absent from the index. Exhaustive LLM-based recovery of such relations is impractical due to the combinatorial explosion of chunk combinations. We present CrossAug, a GNN-guided CROSS-Chunk Graph AUGmentation method that enriches GraphRAG indices with cross-chunk relational structure as an offline step before query-time retrieval. CrossAug derives training supervision through self-supervised graph corruption, uses a topology-aware GNN to score subgraphs for missingness, and applies evidence-grounded LLM completion only to selected high-scoring regions. Experiments on three LLM-based GraphRAG frameworks across four multi-hop and long-document QA benchmarks demonstrate that CrossAug consistently improves performance, confirming the benefit of cross-chunk graph augmentation for retrieval-based question answering. Our code is available at https://github.com/DonFinliani/CrossAug.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有 GraphRAG 框架在构建知识图谱索引时，仅对单个文本分块（chunk）进行实体和关系的局部抽取，导致大量**跨分块关系**（证据分布在多个段落中的关系）在索引中系统性缺失。这类关系对于复杂问答（多跳推理、长文档问答）至关重要，例如别名解析、因果链、空间定位等。
- **背景**：经典 RAG 将长文档分块后检索，GraphRAG 进一步将语料组织为显式知识图谱，支持图算法和社区摘要。但主流框架（如 HippoRAG2、LightRAG、GFM-RAG）构建图时，每个分块内抽取三元组，缺少跨块联合推理，造成索引不完整。穷举式恢复所有跨块关系因分块组合爆炸而不可行。
- **研究意义**：本文首次系统识别并形式化 GraphRAG 的“分块局部抽取瓶颈”，提出一种离线图增强方法，在不改变查询时流程的前提下，通过 GNN 引导的跨块关系补全，显著提升检索质量和最终问答性能。

## 2. 论文提出的方法论

### 2.1 核心思想
**CrossAug** 是一个 GNN 引导的跨块图增强方法，作为离线步骤在基础 GraphRAG 索引构建之后、查询之前运行。它通过以下三个模块协作：
1. **自监督图损坏模块**：利用已有基础图自身生成训练数据——通过掩码事实边、删除实体节点模拟分块局部抽取的不完整性；
2. **拓扑感知 GNN 评分模块**：学习为局部子图预测“缺失性分数”，识别最可能缺失关系的区域；
3. **GNN 引导的 LLM 补全模块**：仅对高分且不重叠的子图调用 LLM，基于提供的证据段落进行证据驱动的三元组补全，并将验证后的三元组持久化回图索引。

### 2.2 关键技术细节
- **异构图表示**：节点包括实体、段落、辅助节点；边包括事实边（来自抽取三元组）、同义边、实体-段落边。每个节点有语义特征（来自嵌入模型），边有权重和类型。
- **加权随机游走采样**：从根实体出发，按边权重（事实边乘子更大）进行带偏见游走，采样局部实体子图，避免全图枚举。
- **自监督损坏**：
  - 掩码事实边（比例 ρe），模拟缺失关系；
  - 删除实体节点（比例 ρv），模拟缺失实体。
  - 完整子图作为负样例（y=0），损坏子图作为正样例（y=1）。
- **GNN 编码器**：采用关系图卷积网络（R-GCN），两层消息传递，边类型特定变换矩阵。节点初始表示 = 映射后的语义特征 + 节点类型嵌入。
- **图级缺失性预测**：经过消息传递后，对节点表示进行全局平均池化和全局最大池化并拼接，送入多层感知机（MLP），输出 sigmoid 缺失性分数 m(S)。训练损失为二元交叉熵。
- **推理选择**：对每个候选子图按 m(S) 排序，超过阈值 θ 且与已选子图 Jaccard 相似度低的候选进入 LLM 补全，直到达到预算 B。
- **LLM 补全**：将子图根实体、实体列表、已知三元组、证据段落组装成 prompt，要求 LLM 输出新的实体和三元组，并附上支持的 chunk ID 作为引用。无引用的三元组被丢弃。

### 2.3 算法流程（文字描述）
1. 基于文档分块和标准 GraphRAG 流程构建基础图 G。
2. 对每个根实体，进行加权随机游走采样局部子图。
3. 在训练阶段：对采样子图进行损坏（掩码边、删除节点），构建正负样本对，训练 GNN 评分器。
4. 在推理阶段：用训练好的 GNN 对所有候选子图评分，选择高分且多样性好的子图。
5. 对选中子图，构建上下文并调用 LLM 生成候选三元组。
6. 仅保留有 chunk ID 引用的三元组，插入到图索引中，得到增强图 G+。
7. 下游检索和问答在增强图上进行，无需重新调用 GNN 或 LLM。

## 6. 论文的主要结论与发现

1. **跨分块关系缺失是 GraphRAG 的固有盲点**：即使基础图包含许多局部正确三元组，仍可能缺少回答复杂问题所需的关键跨块关系。
2. **GNN 引导的缺失性检测有效替代穷举枚举**：通过自监督损坏训练，GNN 可以高精度识别不完整子图，将 LLM 补全的调用次数从指数级降为可控预算。
3. **CrossAug 一致提升性能**：在 4 个基准所有 3 个框架上，CrossAug 均提高 EM/F1（最大 F1 提升 +1.33）。
4. **效益来源多样化**：包括改善检索排序、引入新证据段落、提供隐藏事实（直接补充 prompt 等），且有害变化（True→False）远小于有益变化。
5. **增强三元组质量可靠**：自动评估显示>81% 的事实准确率和引用准确率，与人类判断高度一致。
6. **可控成本**：通过预算 B 和阈值 θ，可在性能与索引开销之间灵活权衡。

## 7. 优点

- **问题定义清晰**：首次系统形式化 GraphRAG 的跨块关系缺失问题，并给出验证案例。

## 8. 不足与局限

- **额外索引成本**：尽管受控，但 CrossAug 仍引入大量 prompt token（每子图表包含多个证据段落），在默认配置下 LiterayQA 额外耗时 5.7 小时、调用 1483 次 LLM（约基础流程的 8.2%）。对超大语料可能仍较昂贵。
- **实验覆盖有限**：仅在一个 LLM（Qwen3-32B）和一个嵌入模型（BGE-M3）上实验，未验证其他 LLM（如 GPT-4、Claude）或不同大小的 GNN。也未在非英语场景测试。
- **假设依赖**：方法假设基础图已经包含实体节点和事实边，且实体语义特征在同一向量空间。对于纯 OpenIE 或非 LLM 构建的稀疏图可能不适用。
- **未讨论错误传播**：LLM 补全的三元组仍有约 15% 错误率，这些错误可能被下游检索放大，论文未做鲁棒性分析（如对错误注入的容忍度）。
- **GNN 训练细节缺失**：未提供完整的训练配置（epoch、batch、学习率调度、收敛曲线），可复现性打折。
- **应用限制**：CROSSAUG 主要针对基于 LLM 抽取的 GraphRAG；对于 LinearRAG 等非 LLM 框架无法直接应用（论文也注明）。
