## 摘要
检索增强生成（RAG）已成为将大语言模型基于外部知识的标准方法，但许多系统仍将证据组织为平面片段并通过高度非结构化的搜索进行检索。这种薄弱的结构成为复杂检索的瓶颈：系统必须决定搜索位置、如何从粗略主题过渡到实体关系证据、哪些证据已被验证、以及哪些中间产物可以重用。我们将这些中间变量定义为检索状态，并将RAG视为结构化状态管理。EfficientGraph-RAG通过三种耦合机制使该状态显式化：TAM定义了证据上的类型化分层状态空间，MARS通过角色专用智能体更新和验证状态，SMP在层次感知访问控制下存储可重用状态。使用同一共享框架配置，EfficientGraph-RAG在三个评估的LongBench检索类子集上报告的平均答案质量指标中排名第一，在HotpotQA EM上匹配最强的智能体基线，同时将大模型令牌使用量减少3.51倍，并在检索组织型跨模态方法中为DocVQA提供了低令牌结果。组件分析显示了角色特定机制：MARS是主要答案质量驱动因素，TAM提供类型化遍历状态和自适应路由信号，SMP实现语料库依赖的重用，跨查询缓存命中率范围为3.77%至23.18%。

## Abstract
Retrieval-augmented generation (RAG) has become the standard way to ground large language models in external knowledge, but many systems still organize evidence as flat chunks and retrieve it through largely unstructured search. This weak structure becomes a bottleneck for complex retrieval: the system must decide where to search, how to move from coarse topics to entity-relation evidence, which evidence has been verified, and which intermediate artifacts can be reused. We define these intermediate variables as a retrieval state and study RAG as structured state management. EfficientGraph-RAG makes this state explicit through three coupled mechanisms: TAM defines a typed hierarchical state space over evidence, MARS updates and verifies the state through role-specialized agents, and SMP stores reusable state under hierarchy-aware access control. Using one shared framework configuration, EfficientGraph-RAG ranks first on the reported answer-quality metrics averaged over the three evaluated LongBench retrieval-style subsets, matches the strongest agentic baseline on HotpotQA EM while reducing large-model token usage by $3.51\times$, and provides a low-token DocVQA result among retrieval-organizing cross-modal methods. Component analysis shows role-specific mechanisms: MARS is the main answer-quality driver, TAM supplies the typed traversal state and Adaptive Routing signal, and SMP enables corpus-dependent reuse, with cross-query cache hit rates ranging from 3.77% to 23.18%.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有检索增强生成（RAG）系统通常将证据组织为扁平化的文本片段（flat chunks），并通过非结构化的相似性搜索进行检索。这种弱结构在复杂检索任务（如多跳问答、长上下文理解、文档视觉问答）中成为瓶颈，系统难以明确知道“在哪里搜索”“如何从粗粒度主题过渡到细粒度实体-关系证据”“哪些证据已被验证”“哪些中间结果可以重用”。
- **整体含义**：作者将上述中间变量定义为**检索状态**（retrieval state），并将RAG视为**结构化状态管理**问题，而非简单的“检索+生成”流水线。通过显式定义、更新、验证和重用检索状态，可以同时提升复杂任务的答案质量和计算效率。

## 2. 方法论

### 2.1 核心思想
提出 **EfficientGraph-RAG**，将检索状态显式建模为四元组：  
`s_q = (p_q, π_q, E_q, z_q, U_q)`  
其中：  
- `p_q`：查询计划（query plan）  
- `π_q`：在TAM中的遍历路径（traversal path）  
- `E_q`：候选或已验证的证据集  
- `z_q`：验证信号（verification signal）  
- `U_q`：可重用的人工制品（如缓存块、嵌入、中间路径）

### 2.2 三个耦合机制

#### 2.2.1 TAM（Tree-based Abstract Memory，树基抽象记忆）
- **作用**：定义类型化分层状态空间。
- **构造**：将语料库构建为类型化树。叶子节点存储原始证据（文本段落、图像-字幕对、表格片段）；中间节点编码局部区域的实体-关系抽象；高层节点编码主题级概念和可重用推理模式。
- **检索**：自顶向下遍历，每层将查询与候选子节点的抽象表示比较，保留 top-k 子节点（通常 k=1 或小值），直到叶子。复杂度为 O(r b h)，其中 b 为分支因子，h 为深度，r 为保留分支数。
- **自适应路由**：对叶子数少于阈值 τ 的 TAM 树，绕过 MARS 直接传给 Reasoner（在 DocVQA 上常用）。

#### 2.2.2 MARS（Multi-Agent Retrieval State，多智能体检索状态）
- **作用**：更新和验证状态。
- **智能体组成**：5个智能体——Planner（规划）、Navigator（导航）、Retriever（检索）、Verifier（验证）、Reasoner（推理）。**只有 Reasoner 使用大模型**（Qwen2.5-14B），其他使用小模型（Qwen2.5-7B）。
- **流程**：迭代循环。Planner 分解查询，Navigator 遍历 TAM，Retriever 选择证据，Verifier 检查相关性和充分性。若 Verifier 通过则传给 Reasoner 生成答案；否则触发重规划（rewrite query）或重新检索，最多迭代 2 次。

#### 2.2.3 SMP（Shared Memory Pool，共享记忆池）
- **作用**：存储和重用状态。
- **三层结构**：
  - `M_g`：全局内存，跨查询可重用资源（嵌入缓存、持久摘要）。
  - `M_t`：任务内存，当前查询的共享状态（规划结果、遍历路径、候选证据、验证输出）。
  - `M_a`：智能体私有内存，仅拥有智能体可访问。
- **层次感知访问控制**：根据智能体角色和内存层级进行读/写权限限制，防止状态干扰。

### 2.3 算法流程（文字说明）
1. 构建 TAM 树（离线，可复用）。
2. 对每个查询 q，第一步由 Planner 生成计划，Navigator 在 TAM 中自顶向下搜索，Retriever 从叶子中选出 top-K 证据。
3. Verifier 评估证据的 relevance、sufficiency、consistency，得到综合分数 α。
4. 若 α ≥ 阈值则通过，将证据传给 Reasoner（Qwen2.5-14B）生成答案；否则在预算内（最多 2 次）重写查询并重复步骤 2-3。
5. 中间状态被写入 SMP，允许后续代理在本次查询或跨查询中重用。

## 6. 主要结论与发现

1. **质量-效率权衡**：在 LongBench 检索子集上综合 EM/F1 排名第一（EM 0.362，F1 0.454），优于 ReAct（0.318/0.414）及其他基线；在 HotpotQA 上达到 EM 0.511（与 ReAct 的 0.516 在 95% CI 内无显著差异），但大模型令牌使用降低 3.51×（478 vs 1,678 token）。
2. **跨模态场景**：DocVQA 上 ANLS 0.755，令牌 607，远低于 ReAct-VL（1,652 token），但 ANLS 比 Vanilla-VL 低约 0.05 point，体现了质量-效率的权衡。
3. **组件角色特异性**：MARS 是主要质量驱动（移除后 EM 显著下降），TAM 提供类型化遍历路径和自适应路由信号，SMP 实现受控重用（跨查询缓存命中率 3.77%~23.18%）。
4. **自适应路由有效性**：DocVQA 上约 40.7% 的查询绕过 MARS 直接推理，若强制全流程则 ANLS 下降 1.87 point。
5. **验证器引导恢复**：在多跳 / 长文档场景中，约 9-10% 的查询由验证器触发第二次检索并成功纠正答案。

## 7. 优点

1. **创新性**：首次将复杂 RAG 问题形式化为检索状态管理，三个组件（TAM、MARS、SMP）分工明确，概念清晰。
2. **效率优化**：通过将检索侧决策分离至小模型（7B）、保留大模型（14B）仅用于最终推理，显著降低大模型令牌开销。
3. **自适应执行**：基于叶子数量自动判断是否需要完整多智能体循环，避免在不必要场景下浪费计算。
4. **细粒度状态管理**：显式的验证信号、重规划机制、层次化记忆池和访问控制，使系统可解释、可控制。
5. **实验覆盖全面**：跨三个任务族、10k+ 查询、6 种基线、组件消融、Token 分布分析、失败模式分类。

## 8. 不足与局限

1. **实验未报告方差**：除 EM 外，F1 和 ANLS 均为点估计，缺乏多次运行的标准差，统计可靠性有限。
2. **跨场景迁移未验证**：仅测试了文本多跳、长文档、文档 VQA 三种；未涉及对话状态检索、代码检索、跨语言检索等场景。
3. **TAM 构造依赖小模型摘要质量**：实体混淆、细节丢失、跨文档错误关联仍是失败主要原因（在 HotpotQA 手动检查 36 例中发现 44% 为实体混淆）。
4. **自适应路由阈值需校准**：DocVQA 的叶子数阈值 τ 在 200 题子集上调整，未证明在其他跨模态场景（如视频 QA、科学图表理解）中的通用性。
5. **跨用户全局内存可能改变行为**：实验中 Mg 按基准独立初始化且不存储答案；实际部署中若 Mg 在多用户间共享，模板化或重复文档较多时重用行为会变化，但未评估。
6. **算力报告不足**：未提供总 GPU 小时数或端到端运行时间，仅报告大模型侧延迟，无法复现完整成本。
7. **基线选择局限**：未对比最新基于图的检索方法（如 LightRAG）或更近的长期记忆系统（如 MemGPT、MemOS），仅对比较早期版本。
