## 摘要
会话数据越来越多地被用作长期运行的助手和AI代理的用户状态的持久来源。然而，查询这些数据仍然具有挑战性，因为对话会自然演变：计划被修改，偏好发生变化，后来的消息经常取代或与早期信息相矛盾。现有的长记忆流水线主要将记忆视为独立的文本或向量对象。这种方法通常检索语义相似但过时的证据，对状态感知推理的支持有限。为了解决这个问题，我们提出了TRACE，一个针对演化会话数据的时序证据图查询处理框架。TRACE将会话建模为一个跨越事件、会话和主题的分层图，并丰富了类型化的时序、因果、更新和矛盾关系。关键的是，该框架维护了有效性标注，使得过时的事实对于历史查询仍然可访问，但在当前状态答案中被折扣。在查询时，TRACE结合了基于向量的笔记检索和图引导的证据搜索，生成感知有效性的支持路径和用于答案生成的混合上下文。这种设计将词汇回忆与证据重建分离，使得能够在长对话历史上进行有界查询时推理。在长会话问答（QA）基准上的实验表明，TRACE提高了时序和多跳推理，消融研究突出了层次、更新感知播种和基于路径的证据的重要性。

## Abstract
Conversational data is increasingly used as a persistent source of user state for long-running assistants and AI agents. However, querying this data remains challenging because conversations naturally evolve: plans are revised, preferences change, and later messages frequently supersede or contradict earlier information. Existing long-memory pipelines largely treat memories as independent text or vector objects. This approach often retrieves semantically similar but stale evidence, offering limited support for state-aware reasoning. To address this problem, we present TRACE, a query processing framework over temporal evidence graphs for evolving conversational data. TRACE models conversations as a hierarchical graph spanning events, sessions, and topics, enriched with typed temporal, causal, update, and contradiction relations. Crucially, the framework maintains validity annotations so obsolete facts remain accessible for historical queries but are discounted for current-state answers. At query time, TRACE combines vector-based note retrieval with graph-guided evidence search, generating validity-aware support paths and a hybrid context for answer generation. This design separates lexical recall from evidence reconstruction, enabling bounded query-time reasoning over long conversational histories. Experiments on long-conversation query-answering (QA) benchmarks show that TRACE improves temporal and multi-hop reasoning, with ablations highlighting the importance of hierarchy, update-aware seeding, and path-grounded evidence.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：会话数据（如AI助手、客户服务、自主代理中的交互历史）会随时间自然演化：计划被修改、偏好变化、后续消息经常取代或否定早期信息。现有长期记忆系统（如LangMem、Mem0、A-Mem等）主要将记忆视为独立的文本或向量对象，检索时仅依赖语义相似性，忽略了事实的时序有效性和演化关系，导致经常返回语义相关但过时或矛盾的证据，无法准确反映用户当前状态，产生幻觉答案。
- **意义**：这本质上是一个数据管理挑战 – 需要一种能够追踪有效时间、处理更新和矛盾、支持状态重建的查询处理框架，类似于时态数据库和流处理中的概念，但应用于非结构化的会话文本。
- **目标**：设计一个系统，既能保留原始会话文本的精确性（词汇回忆），又能捕获事实之间的因果、时序和演化关系，从而在查询时重建当前可信的状态。

## 2. 方法：核心思想、关键技术细节

### 核心思想
TRACE维护两个互补的记忆结构：笔记记忆（`N`）用于词汇召回（原始对话文本的密集检索），以及时态证据图（`G⋆`）用于结构化、状态感知的推理。两者分离，确保生成回答既有精确文本支持又有正确的时态状态。

### 关键技术细节

#### 1）时态证据图构建（离线阶段）
- **三层顶点**：事件顶点（`Ve`，原子事实）、会话顶点（`Vs`，保留会话边界）、主题顶点（`Vt`，跨会话分组）。层次通过二分关联边（`Eh`）实现，支持多粒度遍历。
- **事件模式**：每个事件包含推理域（类型、参与者、时间锚点、描述）、依据域（源笔记集、来源）、有效性域（有效终止时间`u`、有效性分数`v`，初始`v=1`）。
- **类型化语义边**：支持语义关系（`causes`、`enables`、`prevents`、`before`）和演化关系（`updates`、`contradicts`）。演化关系触发状态变化。
- **图构建四步骤**：
  1. 会话级事件提取：从会话摘要中提取 `<Ve, Er>` 子图。
  2. 跨会话链接：通过候选门（参与者Jaccard重叠≥阈值θp + 主题约束）筛选候选事件对，避免O(|Ve|²)爆炸。
  3. 时序与演化边推理：基于可解析时间戳添加确定性 `before` 边；其他候选对通过LLM/规则推理标签。
  4. 层次注入：添加事件→会话、会话→主题的关联边（Inc(ES)∪Inc(ET)），以及会话间的时序边。

#### 2）有效性语义（Validity Semantics）
- **更新（updates）**：当 `en` 通过 `updates` 边连接 `eo`，则设置 `u(eo)=a(en)`，`v(eo)=0`（已过时）。对于通过 `causes/enables` 依赖已过时事件的事件，其 `v(e)=0.5`（部分可疑）。此外，更新会无效化从已过时事件出发的 `causes/prevents` 边（跳过该边）。
- **矛盾（contradicts）**：仅保留为冲突标志，不改变有效性分数，但在路径评分时惩罚。
- **三值有效性方案**：`1`（有效）、`0.5`（部分可疑）、`0`（已过时），规避了概率传播的复杂度。

#### 3）在线查询处理（Algorithm 1）
- **种子选择**：通过密集检索笔记记忆获得top-k笔记，映射到事件 `V0`；然后通过实体查找（`Vent`）、沿演化边1跳扩展（`Vupd`）、主题层次扩展（`Vhier`）增广得到最终种子集 `Vq`。
- **路径生成**：从 `Vq` 进行有界双向遍历，生成候选证据路径 `π = ⟨e1, ℓ1, e2, ..., ℓL-1, eL⟩`。应用三条硬剪枝规则：不跨越无效化的因果边；拒绝违反时序的路径；去除包含连续 `contradicts` 边的路径。
- **路径评分**：组合四个信号的加权和：
  - `NS(π)`：神经相关性（每个事件与查询的最大余弦相似度平均值）
  - `PC(π)`：路径置信度（边置信度乘积 × 关系折扣 × 矛盾惩罚）
  - `TC(π)`：时序一致性（检查所有 `before` 约束是否满足）
  - `UV(π)`：更新有效性（路径中事件的最小有效性分数）
  默认权重(0.4,0.3,0.15,0.15)。
- **上下文组装**：混合上下文 `Cnote`（顶部密集检索笔记）+ `Cpath`（顶部评分路径，标注关系标签和有效性标记），预算在两者之间可重分配（`B'path = Bpath + max(0, Bnote - |Cnote|)`）。
- **答案计划**：基于查询形状选择不同的推理计划（如时间查询→日期规范化指令；偏好查询→偏好聚合；多跳查询→链式证据提示等）。
- **事后证据追溯**（可选）：基于生成答案重新评分路径，用于审计/解释。

## 3. 实验设计

### 数据集
- **LoCoMo**：公开10会话子集，平均24K tokens/对话，1540个非对抗问题。
- **LongMemEval S**：500问题，平均115K tokens/对话，470个非对抗问题（排除假前提类型），覆盖6种问题类型。
- **Stress Test**：从LoCoMo中挖矿并手动重写59个更新密集型问题（四种类型：chain_modification, fact_override, contradiction_resolution, temporal_correction），包含当前有效答案、过期答案、证据路径。

共7种：**Full-Context**（全文输入）、**RAG**（每条消息独立索引，top-10检索）、**LangMem**、**Mem0**（top-30事实）、**Nemori**、**Zep**、**A-Mem**（TRACE继承其笔记摄入）。A-Mem被视为主要比较点。

## 6. 主要结论与发现

1. **整体有效性**：TRACE在两个基准上均超越所有记忆增强系统。在LoCoMo上LLM得分0.661（GPT-4o-mini），比最强基线Nemori高+5.7pp，比A-Mem高+12.8pp。LongMemEval S上LLM得分0.667，比Nemori高+4.5pp，比A-Mem高+13.8pp。提升主要集中在时序和多跳类别。
2. **跨骨干鲁棒性**：在DeepSeek-V4-Flash上排名一致，优势维持，说明性能来源于框架而非某一大模型。
3. **消融贡献**：查询形状计划（L3 CoT）和图证据是两大贡献源：去除L3使LongMemEval S LLM下降11.7pp；去除图证据使LoCoMo LLM下降6.2pp。层次提供额外1.7pp。更新感知种子对Stress Test严格净分贡献+10.17pp。
4. **更新跟踪能力**：Stress Test上TRACE的过时泄露率（7.6%）与Full-Context持平，严格净分+0.314，远超基线（Nemori +0.220，A-Mem +0.271）。显示有效标注和路径评分成功抑制了过时事实。
5. **效率与规模**：在线延迟<1.2秒，token消耗比Full-Context减少5-17倍，同时准确性更高。构建成本单次可接受，候选门有效控制了图复杂度。
6. **可解释性**：图边事实准确率94%、类型标签89%；预推理线索可浏览性良好（81.8%包含答案所需）。路径忠实性42.5%，但时序类别错误（22.5%忠实）主要来源于图层的时标解析误差（90.3%故障传播），表明改进时间分辨率可提升整体。

## 7. 优点

- **创新性建模**：将会话数据转化为带有有效性标注的时态证据图，统一了语义推理和状态演化追踪，解决了现有RAG和记忆系统的根本盲点。
- **分离设计**：词汇回忆（笔记）与状态推理（图）分离，避免混合污染，同时保留精确文本。
- **高效查询处理**：通过候选门控制离线构建复杂度，在线通过剪枝和多信号评分实现有界低延迟推理，无需喂入完整对话。
- **可解释性强**：类型化边、有效性标注、路径评分、事后追溯均支持透明答案生成；三层评估（图→提示→解释）直接量化问题所在。
- **系统性消融**：逐个组件分析贡献，证明每个设计（主题层次、更新种子、查询计划）均为必要。
- **跨骨干鲁棒**：在两种不同架构大模型上表现一致，证明通用性。

## 8. 不足与局限

- **构建质量依赖LLM**：事件提取和边推理依赖大模型，存在误差（如图层时序边错误率达11pp），且这些误差会传播到下游路径忠实性（时序类仅22.5%）。改善时标解析（如消息级排序）是未来方向。
- **超大规模简化**：LongMemEval S（199K会话）上不得不禁用更新检测、使用固定会话块代替LLM主题聚类，可能导致信息损失。这限制了对较少更新场景的通用性。
- **检索能力上限**：Stress Test上TRACE的valid_acc（0.390）仍显著低于Full-Context（0.542），说明某些更新链的证据分布太广，现有有界路径遍历无法完全覆盖。检索方法固有局限。
- **应用范围窄**：仅验证了单用户、单会话链的对话QA；未讨论多用户、多代理协作或流式增量场景。
- **计算资源未披露**：未报告GPU型号和能耗，不利于复现衡和其他系统对比。
- **路径忠实性不高**：整体可信路径率仅42.5%，尤其时序和开放域偏低。虽然有规则验证支持差异显著性，但仍有提升空间。
