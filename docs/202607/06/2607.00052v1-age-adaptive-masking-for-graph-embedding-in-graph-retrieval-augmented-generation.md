## 摘要
GraphRAG是检索增强生成（RAG）的扩展，通过引用图结构数据作为外部知识来支持大型语言模型（LLM）。虽然这种技术理想地捕捉了复杂关系，但由于基于图的潜在特征与基于文本的潜在特征之间的不对齐，它常常在处理LLM的图表示时遇到困难，尤其是对于冻结的LLM。我们通过引入自适应掩码图嵌入（AGE）来解决这个问题。AGE采用基于掩码的自监督学习（SSL）方法中的Transformer。我们设计了类似于文本嵌入编码器的架构，以解决潜在特征不对齐问题。与自然语言文本不同，图是简洁的表示，并且存在一些关键节点，它们包含主导性上下文信息，这些节点很难从其周围环境中预测出来。掩码这样的关键节点会导致SSL过程的低效。因此，AGE专注于预测除关键节点以外的节点，利用一个可学习的节点采样器。我们的实验结果表明，AGE在GraphQA任务中显著改进了使用非参数搜索组件的方法，在四个具有不同特征的基准数据集上实现了优越的准确性。

## Abstract
GraphRAG is an extension of retrieval-augmented generation (RAG) that supports large language models (LLMs) by referring to graph-structured data as external knowledge. While this technique ideally captures intricate relationships, it often struggles with graph representations for LLMs, particularly for frozen LLMs, due to the misalignment between graph-based and text-based latent features. We tackle this issue by introducing the {\it Adaptive-masking for Graph Embedding (AGE)}. AGE employs a Transformer in a mask-based self-supervised learning (SSL) approach. We designed the architecture similar to text embedding encoders, addressing the latent feature misalignment. In contrast to natural language texts, graphs are concise representations, and there exist {\it key nodes} that hold dominant contextual information, which are challenging to predict from their surroundings. Masking such key nodes leads to inefficiency in the SSL process. Therefore, AGE focuses on predicting nodes apart from key nodes, utilizing a learnable node sampler. Our experimental results indicate that AGE significantly improves approaches using non-parametric search component in GraphQA tasks, achieving superior accuracy across four benchmark datasets with distinct characteristics.

### 方法论：核心思想、关键技术细节

- **核心思想**：AGE基于**联合嵌入预测架构（JEPA）**，借用文本预训练（如BERT/MAE）的掩码重建思想，但针对图的特殊性进行改进。图是高度简洁的结构，存在一些上下文支配性的**关键节点（key nodes）**，随机掩码这些节点会导致SSL训练低效，因此引入**可学习节点采样器**，通过强化学习（REINFORCE）自适应地选择哪些节点应作为关键节点（不被掩码），哪些作为辅助节点（被掩码后预测）。
- **关键技术细节**：
  - **总体流程**：查询→非参数检索（Sentence-BERT + kNN + Prize-Collecting Steiner Tree）→子图构建→文本化拼接+图嵌入→输入冻结LLM生成答案。
  - **AGE模块**（训练阶段）：
    - **图编码器（GNN）**：从子图编码节点特征 `h_in`。
    - **节点采样器**：基于多头注意力和线性层生成概率 `p_NS`，通过REINFORCE采样 `N_key` 个关键节点，其余为辅助节点。
    - **概念编码器-解码器**：仅处理关键节点特征 `h_key`，输出 `h_out`（包括辅助节点的占位向量）。
    - **目标编码器**：处理所有节点，输出 `h_target`（作为预测目标）。
    - **JEPA损失**：对辅助节点计算 `||h_out - sg(h_target)||^2`（sg为停止梯度）。
    - **采样损失（L_NS）**：基于REINFORCE，奖励为重建误差，引导节点难预测的节点成为关键节点。
    - **提示调优损失（L_PT）**：训练目标编码器、图聚合器、投影层，直接优化下游任务准确率。
  - **损失分离**：三个损失分别优化互不重叠的参数（θ_GE,θ_CE,θ_CD 由 L_target 更新；θ_TE,θ_GSA,θ_Proj 由 L_PT 更新；θ_NS 由 L_NS 更新），无需权重调节。
  - **推理阶段**：使用概念编码器-解码器的输出 `h_out` 连接LLM，而非 `h_target`（学生超越教师）。
- **理论分析**：从贝叶斯角度论证了分离潜在识别和表示学习的合理性，并通过ELBO解释。

### 主要结论与发现

1. **AGE显著提升非参数检索组件性能**：在ExplaGraphs上，与G-Retriever相比，Llama3.2-1B提升26.72个百分点（从0.5595到0.8267）；在WebQSP上，Llama3.2-3B提升2.02个百分点（从71.3到73.5）。
2. **JEPA优于生成式架构（GA）**，且可学习节点采样器优于随机掩码。
3. **最优采样率ρ=0.3**，能够平衡关键节点识别与辅助节点预测。
4. **跨数据集迁移**：AGE在WebQSP上训练后迁移到ExplaGraphs，性能优于直接训练。
5. **与LLM检索器方法相比**：AGE+AMAR（非参数检索）在CWQ上超越ToG、ReKnoS、PoG等使用GPT-4的LLM检索器方法，在WebQSP上接近，展示了非参数检索的竞争力。
6. **计算成本可控**：AGE在较小LLM（1B/3B）上表现优异，且训练速度与基线相当，甚至更快。

### 优点

- **方法创新**：将JEPA引入图嵌入，并设计强化学习指导的节点采样器，解决了图的“关键节点”问题。
- **损失设计巧妙**：三个损失分别优化不同参数，无需调权重，避免了损失平衡难题。
- **实用性**：基于非参数检索和冻结LLM，训练和推理成本低，可在消费级GPU（2080Ti）上训练。
- **实验全面**：覆盖多个数据集、多种LLM规模、充分消融，且报告了训练速度和推理速度。
- **可视化分析**：t-SNE展示了节点嵌入重组和关键节点语义合理性，提供定性理解。

---

### 不足与局限

1. **固定采样率**：尽管ρ=0.3表现好，但不同图可能最优ρ不同，缺乏自适应调整。
2. **仅测试GraphRAG任务**：方法本身适用于其他模态（如图像、多模态），但未验证通用性。
3. **主要针对小规模LLM**：最大测试模型为Llama3.1-8B，更大模型（如70B）未验证，受限于算力。
4. **关注图嵌入表示**：未直接执行图学习任务（如节点分类、链接预测），应用范围局限于KGQA场景。
5. **对大规模知识图谱的性能提升较小**：在WebQSP等超大图上，AGE提升不如在小图上显著，可能受限于非参数检索器的噪声。
6. **理论保证不充分**：对节点和链接整合的益处缺乏严格理论证明，主要依赖实验验证。
