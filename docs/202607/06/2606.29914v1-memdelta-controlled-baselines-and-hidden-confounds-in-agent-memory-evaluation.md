## 摘要
代理记忆系统越来越多地基于RAG和全上下文基线进行评估，但报告的性能提升往往同时改变了记忆方法、语言模型、嵌入模型或检索管道，使得实际测量内容不明确。我们提出MemDelta，一种受控评估协议，在LongMemEval-S（500个问题，50+会话，三个模型系列）上每次只改变一个组件。四个发现：（1）逐字RAG匹配全上下文GPT-4o-mini（47.2% vs 49.8%，p=0.34），但排名因模型而异：Gemini从全上下文中获得+14pp提升，而Sonnet从RAG中获得+31pp提升，部分原因是它拒绝了63%的全上下文查询；（2）在相同管道中仅更换嵌入模型，n=500时准确率变化+6.2pp（p=0.004），Mem0比MiniLM-RAG高+11pp，但比云端RAG低1.2pp，因此一个变量即可颠覆结论；（3）代理自记忆（42%）表现不如基本检索（47%）；（4）在6种问题类型中的2种（n=88）上，Mem0以50倍的成本匹配云端RAG（72.7% vs 73.9%，p=1.0），表明增益狭窄而非通用。我们建议记忆评估在比较中固定嵌入模型，按模型系列分层，并在将增益归因于架构之前报告写入路径成本。

## Abstract
Agent memory systems are increasingly evaluated against RAG and full-context baselines, but reported gains often mix changes in the memory method with changes in the language model, embedding model, or retrieval pipeline, making it unclear what is actually being measured. We present MemDelta, a controlled evaluation protocol that varies one component at a time on LongMemEval-S (500 questions, 50+ sessions, three model families). Four findings emerge: (1) verbatim RAG matches full-context GPT-4o-mini (47.2% vs. 49.8%, p = 0.34), but the ranking reverses across models: Gemini gains +14pp from full context, while Sonnet gains +31pp from RAG, partly because it refuses 63% of full-context queries; (2) swapping only the embedding model in an identical pipeline shifts accuracy by +6.2pp at n = 500 (p = 0.004), and Mem0 beats MiniLM-RAG by +11pp but loses to cloud-RAG by 1.2pp, so one variable flips the conclusion; (3) agent self-memory (42%) underperforms basic retrieval (47%); (4) on 2 of 6 question types (n = 88), Mem0 matches cloud RAG (72.7% vs. 73.9%, p = 1.0) at 50x the cost, suggesting narrow rather than general gains. We recommend memory evaluations fix embedding models across comparisons, stratify by model family, and report write-path cost before attributing gains to architecture.

- **核心问题**：当前代理（Agent）记忆系统的评估存在严重的混淆变量问题。当某个记忆系统（如Mem0）在基准测试中表现优于基线（如RAG）时，通常的说法是“记忆架构带来了增益”，但实际上这一增益可能来自同时被更改的语言模型、嵌入模型、检索流水线配置等非记忆因素。评估者无法确定“到底什么真正起了作用”。
- **研究动机**：作者观察到两个典型反例：（1）当基线RAG使用MiniLM嵌入时，Mem0看起来领先11个百分点；但仅把嵌入模型换成OpenAI云端嵌入（text-embedding-3-small），结论反转——RAG反超1.2个百分点。一次嵌入更换就颠覆了架构“胜利”。（2）同样的问题，全上下文提示下不同模型行为差异巨大：Sonnet拒绝回答63%的问题，而GPT-4o-mini和Gemini则能正确回答。因此，模型自身的长上下文行为也在模糊结论。

## 6. 论文的主要结论与发现

1. **检索质量主导性能**：无记忆（2%）→基本RAG（47%），+45pp；全上下文（49.8%）与RAG（47.2%）无显著差异（p=0.34）。
2. **模型选择能反转结论**：GPT-4o-mini上S1≈S4；Gemini上S1领先+14pp；Sonnet上S4领先+31pp（因Sonnet拒绝63%全上下文问题）。一个模型得出一个结论。
3. **嵌入更换可产生6.2pp增益**（n=500，p=0.004），效果堪比架构级改进。Mem0用MiniLM时胜11pp，用云端时输1.2pp。
4. **代理自记忆（S2, 42%）不如基本检索（S4, 47%）**，且多会话仅3.3%，压缩策略频繁失效。
5. **Mem0 vs 云端RAG（匹配嵌入）**：在88题上（72.7% vs 73.9%，p=1.0），但写入成本高50倍。优势集中在SS-User和Multi-Session，且不显著。
6. **全程无单一策略主导**：不同类型表现有结构差异——检索在单会话强，全上下文在知识更新（72%）和多会话（37%）有优势；更好嵌入可缩小全上下文优势在多会话（24%→35%）。

## 7. 优点

- **方法学贡献显著**：首次系统性地隔离代理记忆评估中的四大混淆变量，提供了可复制的受控协议。
- **实验设计严谨**：成对比较+统计检验（McNemar）+ bootstrap CI，代码和数据开源。
- **揭示关键陷阱**：用具体案例说明同一变量（嵌入、模型）足以翻转结论，对领域有方法论警醒作用。
- **成本意识引入**：将写入路径成本作为一等指标（时间、调用次数、金钱），利于实际部署判断。
- **跨模型验证**：三种不同系列模型，暴露模型特定的长上下文行为差异。
- **推荐评估规范**：建议固定嵌入、分层报告模型、报告成本等六条实用指南。

## 8. 不足与局限

- **实验覆盖有限**：
  - Mem0比较仅2/6问题类型（88题），时序推理和知识更新未评估。
  - Sonnet和Gemini子集样本量小（100~300题）。
  - 所有结果基于单一合成基准（LongMemEval-S），真实用户数据缺失。
- **嵌入匹配不完美**：S4b vs S3使用不同OpenAI嵌入模型（ada-002 vs text-embedding-3-small），并非严格可控。
- **评判模型单一**：仅用GPT-4o-mini评判所有，虽效应量大，但潜在偏差未验证（无人工抽检）。
- **非正式因果分解**：论文明确声明是“控制探针”而非因果分解，变量间可能交互。
- **Mem0配置限制**：只用了默认配置；其他设置（如不同top-k、提取策略）可能改变结论。
- **随机检索（S_rand）仍含5K tokens**，不完全为零信息（但3.2%准确率很低，问题不大）。
- **应用局限性**：结果依赖于固定任务（LongMemEval-S），代理记忆在更复杂、更开放的任务中可能仍有价值，本论文未否定架构创新。
