## 摘要
提升视觉-语言模型（VLM）在视觉推理上的性能通常需要重新训练或手工设计的提示与工具。我们提出Dynamo，一种无需训练的框架，能够适配冻结的VLM而无需任何权重更新。在一个小的标注训练子集上，代理检查其自身的正确与错误尝试，并演化出两种互补能力：用于认知瓶颈的可复用推理技能，以及用于感知瓶颈的可执行视觉工具。每个生成的工具都与一个指定何时调用它的技能配对，两种能力类型都积累在一个持久库中。在四个视觉推理基准测试和五个VLM主干上，Dynamo在所有20个模型-基准设置上提升了直接推理性能（平均+5.6准确率）。当工具集预先给定后，该框架学习何时调用每个工具，且每一步的工具选择在所有测试主干上均有所提升。与任务特定的强化学习方法（VTool-R1、DeepEyes）相比，Dynamo以极少的计算量缩小了65-99%的强化学习差距，并在可用情况下与强化学习加性结合。

## Abstract
Improving vision-language models (VLMs) on visual reasoning typically requires retraining or hand-designed prompts and tools. We present Dynamo, a training-free framework that adapts a frozen VLM without any weight updates. On a small labeled training subset, the agent inspects its own correct and incorrect attempts and evolves two complementary capabilities: reusable reasoning skills for cognitive bottlenecks, and executable visual tools for perceptual ones. Each generated tool is paired with a skill that specifies when to invoke it, and both capability types accumulate in a persistent library. Across four visual reasoning benchmarks and five VLM backbones, Dynamo improves direct inference on all 20 model--benchmark settings (avg. +5.6 acc). When the tool set is given in advance, the framework learns when to call each tool, and per-step tool choice improves on every tested backbone. Against task-specific RL (VTool-R1, DeepEyes), Dynamo closes 65--99% of the RL gap at a fraction of the compute, and combines additively with RL when available.

- **核心问题**：视觉-语言模型（VLM）在适应新的视觉推理任务族时，通常需要手工制作的监督微调（SFT）数据或专门的强化学习（RL）流程，这需要大量的人力、计算资源和时间。论文核心问题是：能否让一个冻结的VLM代理通过自我检查在小标注子集上的行为，自动构建任务特定的能力集，而无需任何权重更新？
- **研究动机**：现有的VLM适应方法（如SFT、RL）是任务特定且资源密集的；同时，自我改进代理和工具创建方法（如Reflexion、Voyager、CREATOR）要么缺乏持久库，要么仅适用于文本域。视觉推理任务同时需要认知推理（reasoning）和感知处理（perceptual），现有方法难以兼顾。
- **整体含义**：Dynamo提出了一种全新的范式——能力演化（capability evolution）：冻结的VLM通过分析自身正确/错误尝试，自动演化出两种互补能力（推理技能和视觉工具），并积累于持久库中，从而无需梯度更新即可提升视觉推理性能。

## 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

### 核心思想
- **训练-免费适应**：冻结VLM的全部权重，不进行梯度更新。
- **能力演化**：在小标注训练子集上，代理通过诊断、探索、验证的迭代过程，自动生成可复用的推理技能（结构化Markdown SOP）和可执行的视觉工具（Python程序），并配以适用条件。
- **双能力类型**：
  - **技能（Skills）**：用于认知瓶颈（visual evidence available but reasoning weak），如分解步骤、数值计算规则。
  - **工具（Tools）**：用于感知瓶颈（need transformed view of input），如裁剪、缩放、对比调整、颜色提取等。
- **持久库**：演化的能力和工具积累在库中，推理时可检索调用。
- **掌握（Mastery）**：每个工具配有一个技能作为其掌握SOP，指定何时调用、如何使用、常见陷阱，实现选择性调用而非盲目使用。

### 关键技术细节
1. **问题形式化**：给定小训练集 Dtrain，冻结VLM π_θ，能力集 C=(S,T)，代理通过检索相关技能/工具后调用π_θ进行推理。目标是学习C最大化在验证集上的准确率。
2. **演化循环（每个迭代三步）**：
   - **诊断（Diagnose）**：用当前C在Dtrain上运行，采样包含正确和错误的子集Dsub。分析器-决策器（AnalyzerDecider）检查问题、答案、图像、推理轨迹，进行根因分析，确定瓶颈类型（认知/感知）和动作（仅技能/仅工具/两者）。
   - **探索（Explore）**：生成器（Generator）根据诊断，提出M个候选技能+工具组合，每个候选采用不同策略。
   - **验证与促进（Validate and Promote）**：在Dtrain上评估每个候选，选择准确率最高的候选，若严格优于当前C则加入库中，保证单调性。
3. **技能格式**：Markdown文档，含When-to-Use、Strategy、Common Pitfalls、Worked Example四个字段。
4. **工具格式**：Python函数（≤150行），使用OpenCV/PIL/NumPy，输入图像路径，输出处理后的图像或数据。
5. **掌握机制**：当动作是“两者”时，生成器同时输出技能和工具，该技能即为工具的掌握SOP，用于门控调用。
6. **在线适应分布漂移**：使用滚动窗口检测当前查询分布，当主导任务族变化时触发新演化迭代，更新库。

### 算法流程（文字描述）
```
输入：训练集Dtrain，冻结VLM π_θ，迭代次数N，候选数M
输出：能力集C=(S,T)
1. 初始化C为空
2. for n=1 to N:
   a. 用C在Dtrain上求解，收集尝试
   b. 从Dtrain中采样子集Dsub
   c. 调用AnalyzerDecider得到动作a和根因r
   d. 调用Generator根据(a,r)生成M个候选(s_m,t_m)
   e. 计算每个候选在Dtrain上的准确率，选最优m*
   f. 若准确率提升，则C = C ∪ {(s_m*, t_m*)}
3. 返回C
```

## 6. 论文的主要结论与发现

1. **Dynamo在自主演化中有效**：在所有20个模型-基准设置上，Full Dynamo相比零样本直接推理平均提升+5.6准确率。最大提升出现在o4-mini/V\* (+14.1)、o4-mini/HRBench4K (+12.7)等。
2. **认知/感知瓶颈分离有效**：Skill Only在ChartQA、MathVista（认知瓶颈）表现好；Tool Only和Full在HRBench4K、V\*（感知瓶颈）占优，验证了瓶颈分类的有效性。
3. **掌握机制（工具有条件调用）提升工具使用**：在GTA基准上，Dynamo Mode B（为预提供工具学习掌握技能）在所有骨干上提升工具选择、参数预测、指令遵循准确率，显著优于Base Agent和XSkill-style通用技能注入。
4. **相比RL，Dynamo缩小大部分差距且可加性组合**：在ChartQA/TableQA上，Dynamo恢复65-99%的RL差距；在高分辨率感知（V\*）上甚至超过RL；Dynamo+RL达到最强结果，说明两者可加性组合。
5. **在线适应有效**：在分布漂移模拟中，Dynamo的Online adapt策略接近Oracle上界，远优于Static和Direct。
6. **机制消融证实核心设计**：通用提示不能解释增益；仅凭文本诊断无法生成有效视觉工具；诊断必须同时包含正确和错误案例。

## 7. 优点

- **训练-免费**：无需权重更新，显著降低计算成本，易于部署。
- **自动能力构建**：代理从自身行为学习，无需人工设计提示或工具。
- **双能力互补**：同时处理认知和感知瓶颈，覆盖更广的失败模式。
- **掌握机制**：学习工具何时调用，避免盲目使用导致性能下降。
- **在线适应**：能自动检测分布漂移并更新库，适用于动态生产环境。
- **通用性**：跨多个VLM骨干（从开源到闭源）和多个任务类型均有效。
- **与RL兼容**：可作为RL的补充，进一步提升性能。
- **实验充分、分析深入**：包括多骨干、多基准、消融、敏感性、定性分析，结论可信。

## 8. 不足与局限

- **依赖正确性信号**：演化循环和在线适应需要每个案例的正确性标签。实验中使用基准的ground-truth，但在实际部署中需人工审核、自动QA或LLM验证器，未覆盖完全无监督场景。论文明确承认这一局限。
- **诊断受限于VLM自身能力**：分析器-决策器和生成器由同一冻结VLM驱动，若骨干自我反思能力弱，可能导致误诊，进而生成次优技能/工具。小骨干可能收敛更慢或产出较低质量库。
- **基准范围有限**：仅评估基于图像的VLM视觉推理，未涵盖3D空间推理、医学影像等其他模态。认知/感知决策规则和可执行工具接口能否迁移到这些领域尚未验证。
- **多候选探索机制未充分消融**：论文指出多候选探索（M>1）需要更大子集实验，留待未来工作。
- **工具生成行数限制**：工具限定≤150行Python，可能无法处理复杂视觉处理需求。
- **计算成本虽低但非零**：尽管无需梯度，但仍需多次调用大模型API（数千次），对API成本敏感的应用可能存在限制。
