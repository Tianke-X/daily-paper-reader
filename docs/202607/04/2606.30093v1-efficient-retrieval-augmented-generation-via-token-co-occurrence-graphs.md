## 摘要
检索增强生成（RAG）通过将生成过程锚定在外部知识上来缓解大型语言模型（LLM）的幻觉问题。然而，标准的RAG方法在多跳推理方面存在困难。尽管近期基于图的RAG方法改进了相互关联的文本块的检索，但它们通常依赖于计算成本高且易出错的基于LLM的提取管道。为了解决这些问题，我们提出了TIGRAG（Token-Induced GraphRAG），一种基于Token共现知识图谱的高效图增强RAG框架。TIGRAG使用滑动窗口共现统计直接建模Token之间的拓扑关系，从而实现了可扩展的图构建。在推理过程中，它结合了基于图的语义扩展和神经重排序，以检索用于多跳推理的相互关联的证据。具体来说，它引入了一种迭代的实体驱动检索策略，该策略使用从先前检索到的上下文中提取的桥接实体逐步扩展查询。我们在三个广泛采用的多跳问答（QA）基准上对TIGRAG进行了评估。实验结果表明，我们的框架在检索和下游QA任务中持续优于密集检索和基于图的RAG方法，同时大幅减少了索引时间、推理延迟和提示足迹。

## Abstract
Retrieval-Augmented Generation (RAG) mitigates hallucinations in Large Language Models (LLMs) by grounding the generation process on external knowledge. However, standard RAG approaches struggle with multi-hop reasoning. While recent graph-based RAG methods improve the retrieval of interconnected chunks, they often rely on computationally expensive and error-prone LLM-based extraction pipelines. To address these issues, we propose TIGRAG (Token-Induced GraphRAG), an efficient graph-augmented RAG framework based on a token co-occurrence Knowledge Graph. TIGRAG directly models topological relationships between tokens using sliding-window co-occurrence statistics, thus enabling scalable graph construction. During inference, it combines graph-based semantic expansion and neural reranking to retrieve interconnected evidence for multi-hop reasoning. Specifically, it introduces an iterative entity-driven retrieval strategy that progressively expands the query using bridging entities extracted from previously retrieved contexts. We evaluated TIGRAG on three widely adopted multi-hop Question Answering (QA) benchmarks. Experimental results demonstrated that our framework consistently outperforms dense retrieval and graph-based RAG methods in both retrieval and downstream QA tasks, while substantially reducing indexing time, inference latency, and prompt footprint.

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**: TIGRAG的核心创新在于**直接利用文本中Token的共现统计信息来构建知识图谱**，从而完全避免了传统GraphRAG中昂贵的LLM驱动提取过程。
- **关键技术细节（算法流程）**:
    1.  **离线图构建**:
        - **文本分块**: 使用滑动窗口策略将文档分割成重叠的文本块。
        - **Token提取与净化**: 对每个文本块进行分词，并去除停用词和短/无意义的Token。
        - **共现图创建**: 在一个滑动窗口内，统计任意两个Token共同出现的频率。以Token为节点，以其共现频率为边的权重，构建一个**加权无向知识图谱**。
    2.  **在线推理（单跳）**:
        - **查询种子节点识别**: 对用户查询进行相同的预处理，识别出查询中在图G中出现的Token，作为种子节点。
        - **语义扩展 (PPR)**: 在种子节点上初始化一个个性化向量，在图G上运行**个性化PageRank (PPR)** 算法。PPR的传播机制使得语义上相关的Token被激活，从而扩展查询的语义范围。
        - **候选块检索**: 检索所有包含扩展后Token集的文本块。
        - **图加权评分**: 结合BM25和PPR分数，对候选块进行评分。公式为：`ρ(c, μ) = Σ_{v} p̂(v) · BM25(v, c)`。这里PPR分数`p̂(v)`作为拓扑权重，强调来自图表征的语义概念。
        - **动态候选池**: 不采用固定数量的Top-k截断，而是动态地选择候选块，直到其累积分数达到总分数的一个阈值 (th_ρ=0.8)。
        - **神经重排序**: 对动态池中的候选块，使用嵌入模型计算其与原始查询的余弦相似度，进行最终的精排。
    3.  **多跳推理（迭代扩展）**:
        - **实体提取**: 对第一轮检索到的最高分的Top-ι个文本块进行命名实体识别（NER）。
        - **查询扩展**: 将提取的、**尚未出现过**的新实体拼接回原始查询，形成扩增查询。
        - **迭代检索**: 使用扩增查询重复上述单跳检索过程（PPR、评分、重排序）。重复此过程至多h次（论文中h=2）。
        - **结果合并**: 将所有轮次检索到的候选块，按它们在任一**轮次中达到的最高重排序分数**进行全局排序，最终选取Top-β个块作为最终的上下文。

### 8. 不足与局限

- **对NER质量的依赖**: 多跳推理中使用的迭代实体驱动检索策略高度依赖于命名实体识别（NER）的准确性。不准确的实体提取可能导致语义漂移，这是其一个明显的脆弱点。
- **共现统计的局限性**: 共现图主要捕获的是词汇的**表层共现关系**，无法建模更深层的**语义关系**（如同义词、反义词等）。虽然这保证了效率，但可能限制了对复杂语义联系的挖掘。
- **数据集限制**: 所有实验仅在**英文文本**的多跳问答数据集上进行。其方法在跨语言、多模态或其他类型的NLP任务中的泛化能力尚未验证。
- **仅限于文本**: 论文的评估仅限于文本，未能探索其在多模态（如同时处理图像和文本）场景下的有效性。
- **未讨论长尾/稀有Token**: 论文在Token提取阶段设置了最低长度限制。对于领域内非常关键但拼写较短或罕见的Token（例如特定代码或缩写），可能会被过滤掉，从而丢失信息。
