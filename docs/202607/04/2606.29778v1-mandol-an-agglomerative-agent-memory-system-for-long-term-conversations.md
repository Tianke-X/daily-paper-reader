## 摘要
长期对话智能体需要记住并查询跨会话、多类型且具有复杂关联的信息。现有智能体记忆系统依赖异构的向量和图数据库，导致记忆信息碎片化，并产生高跨数据库I/O延迟。在检索方面，常见的RAG风格方法容易引入噪声、遗漏相关线索，且缺乏令牌预算控制，降低了LLM的准确性和效率。我们提出Mandol，一种聚合式记忆系统，将碎片化的记忆表示和存储整合为统一的记忆原生架构。其核心组件包括：(1) 分层记忆模型，将记忆组织为表示原始记忆信息的基本层和将基本记忆聚合为可追溯的抽象记忆的高级抽象层，两者统一表示为结构化语义图；(2) 聚合语义数据结构，结合了SemanticMap和SemanticGraph，原生融合了键值、向量和图结构，并提供统一的混合检索算子以消除跨数据库I/O；(3) 定量查询机制，包括查询自适应路由、定量去噪与冲突解决、以及令牌约束的上下文生成，在检索过程中无需LLM参与。在两个广泛使用的长期对话基准LoCoMo和LongMemEval上的实验表明，Mandol在代表性智能体记忆系统中取得了最佳总体准确率。在性能比较中，Mandol在10 QPS并发负载下获得了5.4倍的检索加速和4.8倍的插入加速，同时在消费级硬件上仍保持低延迟。

## Abstract
Long-term conversational agents need to remember and query cross-session, multi-typed information with complex correlations. Existing agent memory systems rely on heterogeneous vector and graph databases, which fragment memory information and cause high cross-database I/O latency. For retrieval, common RAG-style methods tend to introduce noise, miss correlated clues, and lack token budget control, degrading LLM accuracy and efficiency.   We propose Mandol, an agglomerative memory system that consolidates fragmented memory representations and storage into a unified memory-native architecture. Its core components include: (1) a hierarchical memory model that organizes memory into a basic layer representing raw memory information and a high-level abstract layer that agglomerates basic memories into traceable abstract memories, both uniformly represented as structured semantic graphs; (2) an agglomerative semantic data structure combining SemanticMap and SemanticGraph, which natively fuses key-value, vector, and graph structures and provides unified hybrid retrieval operators to eliminate cross-database I/O; and (3) a quantitative query mechanism with query-adaptive routing, quantitative denoising and conflict resolution, and token-constrained context generation, all without involving LLMs during retrieval. Experiments on two widely used long-term conversation benchmarks, LoCoMo and LongMemEval, show that Mandol achieves the best overall accuracy among representative agent memory systems. For performance comparison, Mandol also obtains a 5.4x retrieval speedup and a 4.8x insertion speedup under 10 QPS concurrent load, while still maintaining low latency on consumer-grade hardware.

## 1. 核心问题与整体含义（研究动机与背景）

长期对话智能体（如客服、个人助理、医疗诊断）需要跨会话、多类型信息（对话、意图、事件、实体状态）的记忆与查询。现有记忆系统存在三大痛点：

- **表示碎片化**：常用异构数据库（向量库+图库）组合，缺乏统一表示，导致跨库I/O开销高、查询延迟大。
- **存储与查询效率低**：混合查询需跨数据库编排，序列化与数据传输成为瓶颈。
- **检索质量差**：RAG式检索易引入噪声、遗漏关联线索，且无法控制token预算，简单查询浪费上下文，复杂查询可能丢失关键证据。

Mandol 旨在通过统一的记忆原生架构，解决上述碎片化与检索低效问题，实现高效、准确的长时对话记忆管理。

## 2. 提出的方法论

### 核心思想
Mandol 将所有记忆（原始与抽象）统一表示为结构化语义图，并内置聚合数据结构与定量检索机制，消除异构存储依赖与LLM介入的检索噪声。

### 关键技术细节

#### 2.1 层次记忆模型
- **基本记忆层**：由记忆单元（包含语义向量、原始文本、元数据）、记忆空间（会话/主题隔离）和显式/隐式关系（时序、引用、相似性）构成统一语义图。
- **高级抽象记忆层**：通过LLM自动聚合基本记忆生成可追溯的抽象记忆，包括：
  - 情景记忆（事件链，时序+因果边）
  - 语义记忆（实体图，引用+属性边）
  - 情感记忆（用户偏好，状态更新边）
- 抽象节点保留 `source_unit_uids` 链接回原始基本单元，确保证据可追溯。

#### 2.2 聚合语义数据结构
- **SemanticMap**：融合键值存储、稠密/稀疏向量索引、倒排索引，存储记忆单元及其语义向量。
- **SemanticGraph**：轻量邻接表存储显式边；隐式边按需通过SemanticMap的索引计算。
- 两者共享同一内存单元ID，实现原子化混合检索（如：从向量检索→图扩展→空间过滤在统一数据结构内完成），消除跨库I/O。

#### 2.3 定量查询机制（无需LLM参与）
- **查询自适应路由**：轻量意图分类器根据查询特征（时间、实体提及）选择最相关记忆源（基本/情景/语义/情感），并分配候选预算。
  - 混合召回：同时执行BM25、SPLADE稀疏扩展、稠密向量搜索，通过倒数排名融合（RRF）合并候选，再用跨编码器重排序得到语义相关分数 \( S_{ce}(c_i) \)。
  - 对多跳查询进行图扩展（按边类型和预算剪枝）。

- **定量去噪与冲突解决**：
  - **源内去噪**：基于中位数绝对偏差（MAD）的动态阈值 \( \tau_{mad} = \text{median}(S_{ce}) - \kappa \cdot \text{MAD}(S_{ce}) \)，鲁棒过滤低分噪声。
  - **跨源冲突解决**：对冲突候选计算仲裁分数：
    \[
    S_{arb}(c_i) = w_{rel} \cdot S_{ce}(c_i) + w_{temp} \cdot f_{time}(c_i) + w_{source} \cdot f_{source}(c_i)
    \]
    其中 \( f_{time} \) 为时间衰减函数，\( f_{source} \) 按源置信度赋权（基本=1.0，情景=0.8，语义=0.6）。保留最高分记录，淘汰陈旧/冗余信息。

- **Token约束上下文生成**：采用最大边际相关性（MMR）贪心选择：
  \[
  \text{MMR}(c_i|S) = \lambda \cdot \text{Rel}(c_i) - (1-\lambda) \cdot \text{Rwd}(c_i, S)
  \]
  Rel为跨编码器相关分数，Rwd基于实体与源重叠度量冗余。迭代添加高MMR候选直至token预算耗尽，偏好多样性。
