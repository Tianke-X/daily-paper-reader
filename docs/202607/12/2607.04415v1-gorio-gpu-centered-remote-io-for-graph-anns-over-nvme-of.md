## 摘要
基于图的近似最近邻搜索（ANNS）越来越多地用于向量数据库和检索增强生成服务，但大型向量索引通常超出单个GPU服务器的内存容量。基于结构的NVMe over Fabrics（NVMe-oF）提供了一种有吸引力的存储解耦基板，然而现有的远程存储路径仍然主要是CPU中心的：CPU形成I/O请求、驱动传输进展，并决定GPU计算何时可以恢复。这种组织方式与图ANNS不匹配，因为下一个数据访问是在GPU图遍历内部发现的。
本文提出了GORIO，一项系统研究，将GPU中心的本地I/O扩展到远程存储，并专门针对NVMe-oF上的图ANNS定制生成的基板。GORIO将查询演进、页缺失生成、挂起查询状态和恢复决策保留在GPU上，而CPU仅作为NVMe-oF传输和完成代理。该设计包含两层：GPU直通远程I/O路径，将本地页缓存缺失转化为分阶段远程操作；以及针对ANNS的调度机制，将图遍历与远程页服务重叠。在基于RDMA NVMe-oF路径的SIFT1M DiskANN风格图工作负载上，GORIO比最先进的远程I/O参考路径快1.31倍，比直接远程页缓存路径快3.73倍。这些结果证明了用于图ANNS的具体GPU中心化远程I/O基板。

## Abstract
Graph-based approximate nearest neighbor search (ANNS) is increasingly used in vector databases and retrieval-augmented generation services, but large vector indexes often exceed the memory capacity of a single GPU server. NVMe over Fabrics (NVMe-oF) provides an attractive storage-disaggregation substrate, yet existing remote storage paths are still largely CPU-centered: the CPU forms I/O requests, drives transport progress, and determines when GPU computation can resume. This organization is poorly matched to graph ANNS, where the next data access is discovered inside GPU graph traversal.   This paper presents GORIO, a system study that extends GPU-centered local I/O to remote storage and specializes the resulting substrate for graph ANNS over NVMe-oF. GORIO keeps query evolution, page-miss generation, pending-query state, and resume decisions on the GPU, while the CPU acts only as an NVMe-oF transport and completion proxy. The design has two layers: a GPU-direct remote I/O path that turns local page-cache misses into split-phase remote operations, and ANNS-specific scheduling mechanisms that overlap graph traversal with remote page service. On a SIFT1M DiskANN-style graph workload over an RDMA NVMe-oF path, GORIO is 1.31$\times$ faster than the state-of-the-art remote-I/O reference path and 3.73$\times$ faster than the direct remote page-cache path. These results demonstrate a concrete GPU-centered remote I/O substrate for graph ANNS.
