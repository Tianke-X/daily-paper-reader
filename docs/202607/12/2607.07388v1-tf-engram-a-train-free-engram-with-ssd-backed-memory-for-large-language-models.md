## 摘要
大型语言模型（LLMs）将事实知识和领域特定模式隐式地存储在密集的Transformer参数中，这使得通过预训练、微调、检索增强或更长的上下文进行知识扩展成本高昂。Engram风格的内存提供了一种紧凑的隐藏状态注入途径，但现有的GPU端设计通常依赖于基于哈希的压缩，导致不相关的短语在共享槽中发生冲突，削弱了短语级别的语义保真度。我们提出TF-Engram，一种无训练的Engram系统，它从外部语料库离线构建短语特定的语义记忆，将大型记忆表存储在GPU-DRAM-SSD层次结构中，并使用早期退出引导的预测预取技术来隐藏自回归解码过程中的外部内存延迟。在Qwen3-0.6B上，TF-Engram将平均下游分数从57.6提升至59.4，优于冻结主干网络和参数匹配的LoRA基线。系统评估表明，大型TF-Engram表可以以适度的离线成本构建，SSD支持的存储大幅减少了GPU内存需求，而预测预取技术恢复了因外部内存访问导致的吞吐量损失。这些结果表明，静态短语记忆可以作为可扩展、无训练且低开销的系统组件集成到LLM推理中。

## Abstract
Large Language Models (LLMs) store factual knowledge and domain-specific patterns implicitly in dense Transformer parameters, making knowledge expansion costly through pretraining, fine-tuning, retrieval augmentation, or longer contexts. Engram-style memory offers a compact hidden-state injection pathway, but existing GPU-resident designs often rely on hash-based compression, causing unrelated phrases to collide in shared slots and weakening phrase-level semantic fidelity. We present TF-Engram, a train-free Engram system that constructs phrase-specific semantic memory offline from external corpora, stores large memory tables across a GPU--DRAM--SSD hierarchy, and uses Early-Exit Guided Predictive Prefetching to hide external-memory latency during autoregressive decoding. On Qwen3-0.6B, TF-Engram improves the average downstream score from 57.6 to 59.4, outperforming both the frozen backbone and a parameter-matched LoRA baseline. System evaluation shows that large TF-Engram tables can be built with moderate offline cost, SSD-backed storage substantially reduces GPU memory demand, and predictive prefetching recovers much of the throughput loss caused by external memory access. These results demonstrate that static phrase memory can be integrated into LLM inference as a scalable, train-free, and low-overhead system component.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）将事实知识和领域模式隐式编码在密集的Transformer参数中，导致知识扩展（如引入新事实、领域术语）需要高昂的预训练、微调或检索增强（RAG）等手段。现有的Engram风格记忆虽然提供了一种紧凑的隐藏状态注入途径，但依赖哈希压缩将大量短语映射到固定大小的GPU表，导致不同短语在共享槽中冲突，降低了短语级语义保真度。
- **整体含义**：本文旨在设计一种**无训练、可扩展、低开销**的静态短语记忆系统，将知识存储与模型参数解耦，使LLM能通过外部层次化存储访问大量短语级语义信息，而无需修改模型权重或增加训练成本。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将Engram记忆从GPU端哈希压缩的紧凑表格重构为**SSD支持的层次化外部记忆系统**，并利用**早期退出预测预取**隐藏外部存储访问延迟。
- **关键技术细节**：
  - **离线无训练语义记忆构建**：从多源语料（FineWeb-Edu、Wikipedia、C4、arXiv、PubMed）中挖掘候选短语（如双词、三词、命名实体），用目标LLM分词器分词，再用外部语义编码器（Qwen3-Embedding）生成短语嵌入，构建键-值存储。短语表不经LLM训练，独立于模型。
  - **SSD支持的层次记忆**：按热数据→GPU缓存、温数据→主机DRAM、冷数据→NVMe SSD三级存储。GPU仅保留高频条目，SSD存储完整短语表，通过两级索引（token前缀→短语ID→存储地址）进行查找。
  - **早期退出引导预测预取**：在Transformer中间层（如第L-r层）附加辅助预测头，提前输出下一token的近似分布。利用该分布预测未来可能访问的短语（Top-K），在剩余层计算期间异步将候选条目从SSD或DRAM预取至更高速缓存。预取优先级由预测概率、短语有用性和当前存储层级决定。公式（13）-（17）描述了该流程。

## 6. 论文的主要结论与发现

- TF-Engram在10个下游benchmark上平均得分从57.6（backbone）提升至59.4，优于参数匹配LoRA（58.7）；在8/10任务上超过backbone。

- 更大规模（100M条目）和适中早期退出层（第8层）提供最佳精度-效率权衡。
- 结论：静态短语记忆可作为无训练、低开销的可扩展系统组件集成到LLM推理中，实现知识存储与模型解耦。

## 7. 优点：方法或实验设计上的亮点

- **无训练**：完全避免模型参数更新，记忆表从语料库直接构建，可跨模型复用，易扩展至新领域。
- **层次化存储**：利用SSD大幅扩展记忆容量，同时通过GPU/DRAM缓存保持高频访问低延迟。
- **早期退出预取**：创新性地利用中间层预测作为调度信号，与剩余层计算重叠，有效隐藏SSD及数据传输延迟，无需增加最终输出延迟。
- **实验设计细致**：分别报告了离线构建成本、运行时资源、延迟分解、敏感度分析，涵盖多维度系统指标。
- **性能增益明确**：在知识密集型任务（ARC-Challenge、TruthfulQA-MC2）上提升显著，且平均优于可训练LoRA，证明无训练外部记忆在实际中可行。

## 8. 不足与局限

- **模型规模狭窄**：仅在单一小模型（Qwen3-0.6B）上评估，未验证在更大模型（如7B、70B）或多GPU部署下的表现。预取窗口和层次延迟缩放可能不同。
- **任务覆盖不全**：在HellaSwag和WinoGrande上性能略低于backbone，表明静态短语记忆不适用于所有任务类型（如需要上下文常识或句子级消歧的任务）。论文未深入分析失败原因。
- **消融实验不够系统**：未提供单独的移除“SSD层次”或“预取”的消融表，而是通过间接对比（如GPU-resident vs SSD-backed）及不同预取策略体现，不够直接。
- **早期退出头校准需求**：虽然论文声称可无训练，但实际采用轻量校准（训练投影层）以提高预测质量。严格无训练设置下的预取效果未单独评估。
- **更新与一致性**：记忆表离线构建后固定，未讨论在线更新或一致性维护（如词表变化、知识刷新），在实际动态场景中可能受限。
