## 摘要
长上下文语言模型推理日益受到存储键值缓存所需的内存带宽和容量的限制，然而现有的压缩方法通常在各层或各令牌之间应用统一的预算，当词法线索和语义状态需要不同的保留策略时，检索性能会下降。我们提出了DepthWeave-KV，一种令牌自适应缓存压缩方法，它利用共享的低秩通道基对相邻Transformer层的键和值状态进行分解，同时在注意力行为敏感的令牌处保留轻量级的令牌特定残差。DepthWeave-KV将跨深度残差分解与令牌条件深度路由器相结合，该路由器为承载指令和检索关键性的令牌分配更高的重建秩，并使用来自注意力输出探针的免校准在线误差跟踪，在生成过程中自适应压缩，无需重新训练基础模型。融合的CUDA实现联合执行基查找、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、大海捞针、L-Eval以及长文本问答和摘要基准测试中，DepthWeave-KV以显著更低的内存使用达到了接近全缓存的任务质量，在平均分数和检索准确率上超越了先前的压缩缓存，同时实现了8.3倍的KV内存缩减和64K上下文下每秒72.8个令牌的处理速度。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

- **核心问题**：长上下文大语言模型（LLM）推理时，存储注意力机制中的键值（Key-Value, KV）缓存成为了内存带宽和容量的主要瓶颈。现有的KV缓存压缩方法通常对所有Transformer层和所有令牌（Token）采用**均匀的压缩预算**，忽略了不同层和不同令牌在任务中的重要性差异。这导致在需要精确检索（如“大海捞针”）或复杂推理的任务中，关键信息容易丢失，性能严重下降。
- **研究动机**：论文观察到，相邻Transformer层的KV缓存存在冗余，可以共享，但直接共享又过于粗糙。同时，一些“关键令牌”（如指令、实体、答案句）需要比“填充令牌”（如流畅但无信息的连续词）更精确的缓存重建。因此，需要一种能够**同时感知深度（Depth-aware）和令牌（Token-adaptive）** 的压缩机制，自适应地将更高保真度分配给更重要的部分。
- **整体含义**：本文旨在解决长上下文LLM推理中内存效率与任务质量（尤其是检索质量）之间的矛盾，提出一种不牺牲检索精度、无需重新训练模型的高效KV缓存压缩方法。

### **2. 论文提出的方法论：核心思想、关键技术细节**

- **核心思想**：DepthWeave-KV采用“跨深度残差分解”策略。它不是为每一层独立存储完整的KV张量，而是在相邻的若干层之间，**共享一组低秩的通道基（Bases）**，用于表示缓存中通用的、共性的部分。同时，对于需要高保真度的关键令牌，则额外存储轻量级的**令牌特定残差（Token-specific Residuals）**，用于恢复丢失的细节信息。
- **关键技术细节**：
    1.  **跨深度残差分解**：
        - 将Transformer层划分为重叠的“深度窗口”（Depth Windows）。
        - 在每个窗口内，为每注意力头维护共享的键和值低秩基。
        - 每一层中每个令牌的键/值状态由两部分重建：**共享基的投影** + **门控的令牌特定残差**。
        - 共享基捕捉了窗口内各层的共同信息，而残差则保留了该令牌在该层的局部独特信息。
    2.  **令牌条件深度路由（Token-Conditional Depth Router）**：
        - 设计一个轻量级的路由器，为每个令牌的残差分配不同的“秩”（Rank），即保留多少残差细节。
        - 路由器的决策基于四个在线特征：累计注意力质量、最大注意力峰值、是否为指令/分隔符、以及上一次探测间隔的注意力输出误差。
        - 最终将令牌分为不同等级：**高重要性令牌**（如检索关键点）获得高秩残差，**低重要性令牌**（如填充词）零秩残差（仅由共享基表示），从而动态分配压缩预算。
    3.  **免校准在线误差跟踪（Calibration-Free Online Error Tracking）**：
        - 在生成过程中，通过“注意力输出探针”监测压缩引入的误差。
        - 每隔若干步，比较使用压缩缓存和临时使用高保真缓存计算出的注意力输出之间的差异。
        - 若误差超过阈值，路由器会自动调整，为窗口内更多令牌分配更高残差秩；反之则降低。此机制无需任何校准数据，可**在线自适应**调整压缩率。
    4.  **融合CUDA实现**：
        - 实现一个融合的内核（Fused Kernel），将基查找、残差反量化、系数乘法和注意力计算整合在一起。
        - 避免将重建后的完整KV张量写回全局内存，显著减少解码时的内存流量和延迟。
