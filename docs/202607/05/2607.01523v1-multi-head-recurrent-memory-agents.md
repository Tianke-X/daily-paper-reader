## 摘要
循环记忆代理通过将输入迭代整合到固定大小的记忆窗口中，将大语言模型扩展到任意长上下文。尽管具有可扩展性，但这些代理存在一个被充分记录的可信度问题：随着上下文长度增加，端到端性能会系统性下降。我们通过将性能分解为两个因素——记忆捕获和记忆保持——来诊断这一失败，并定量确认保持是主要的瓶颈。保持能力崩溃是因为现有设计将记忆维持为单一文本块，迫使每次更新都有覆盖先前保留内容的风险。基于这一诊断，我们提出多头循环记忆（MHM），一种通用的、无需训练的框架，将记忆划分为独立的多头，由分阶段先选后更新策略控制。在每一步，恰好有一个头被选中进行更新，而其余头在结构上被保护免受覆盖，从而将保持的负担从模型行为转移到架构设计上。作为一种轻量级实例化，我们引入了最近最少更新MHM（MHM-LRU），它保证了均匀的头利用率且零额外令牌开销。在长上下文基准上的大量实验表明，MHM-LRU在100K至1M令牌范围内显著提高了保持率和端到端准确性，而基线性能急剧下降。在896K令牌的RULER-HQA上，MHM-LRU将记忆保持率从低于30%提升至73.96%。这些增益跨模型家族、规模和任务类型通用，将架构优化定位为实现可靠长上下文循环记忆的实用且成本高效的路径。

## Abstract
Recurrent memory agents extend LLMs to arbitrarily long contexts by iteratively consolidating input into a fixed-size memory window. Despite their scalability, these agents exhibit a well-documented reliability problem: end-to-end performance degrades systematically as context length grows. We diagnose this failure by decomposing performance into two factors--memory capture and memory retention--and quantitatively confirm that retention is the dominant bottleneck. Retention collapses because existing designs maintain memory as a monolithic text block, forcing every update to risk overwriting previously retained content. Motivated by this diagnosis, we propose Multi-Head Recurrent Memory (MHM), a general, training-free framework that partitions memory into independent heads governed by a stage-wise select-then-update strategy. At each step, exactly one head is selected for update while the remaining heads are structurally shielded from overwriting, shifting the burden of retention from model behavior to architectural design. As a lightweight instantiation, we introduce Least-Recently-Updated MHM (MHM-LRU), which guarantees uniform head utilization with zero additional token overhead. Extensive experiments on long-context benchmarks show that MHM-LRU substantially improves both retention and end-to-end accuracy across the 100K--1M token range, where baselines degrade sharply. On RULER-HQA at 896K tokens, MHM-LRU improves the memory retention rate from less than 30% to 73.96%. These gains generalize across model families, scales, and task types, positioning architectural optimization as a practical and cost-efficient path toward reliable long-context recurrent memory.

**整体含义**

## 2. 论文提出的方法论

### 核心思想
提出 **Multi-Head Recurrent Memory (MHM)**，将记忆划分为多个独立头（heads），每步只更新一个头，其余头在结构上被保护免受覆盖。本质上将保持负担从模型行为转移到架构设计。

### 关键技术细节
- **记忆表示**：记忆状态 \( m_t = \{ m_t^1, m_t^2, \dots, m_t^H \} \)，每个头用 `<memory_h> ... </memory_h>` 标签分隔。
- **阶段式选择-更新策略**：
  1. **选择阶段**：根据准则函数 \( h^\star = \arg\max_{h \in [H]} r_{\text{select}}(h \mid m_{t-1}, q, c_t) \) 选出一个头。
  2. **更新阶段**：仅更新选中的头 \( m_t^{h^\star} = f_\theta(m_{t-1}, h^\star, q, c_t) \)，其他头不变。
- **轻量实例 MHM-LRU**：采用**最近最少更新**（Least-Recently-Updated）选择规则，每步选择间隔最久未被更新的头。这保证了均匀利用所有头，且无需额外LLM推理，零额外令牌开销。

### 公式与算法流程（文字说明）
论文用伪代码（Algorithm 1）描述了MHM流水线：
- 初始化H个空记忆头。
- 将上下文C等分为T个块。
- 对每个块t：
  - 根据rselect选择头h⋆。
  - 用LLM根据当前记忆状态、查询和块内容生成h⋆的新内容。
  - 覆盖记忆状态中的h⋆。
- 最后用最终记忆m_T和查询生成答案。

## 6. 论文的主要结论与发现

1. **保留失败是主导瓶颈**：MCR稳定，MRR随上下文长度急剧下降，且与端到端准确率强相关（ρ=0.98）。
2. **MHM-LRU显著改善记忆保持**：在896K令牌时，MRR从<30%提高到73.96%（RULER-HQA），在BABILong 1M时从42.96%提高到68.68%。
3. **保持改进直接转化为端到端准确率**：例如RULER-HQA 896K时，MHM-LRU准确率49.74%，远超MemAgent的21.62%和LLM基线的0.00%；且准确率在100K–1M范围内稳定（48.70%–50.26%），而基线持续下降。
4. **跨模型家族和规模泛化**：在Qwen2.5-32B和gpt-oss-120b上均获得类似增益。
5. **跨任务类型泛化**：在BABILong 10种任务中，MHM-LRU在大多数任务上优于MemAgent，尤其在多步关系推理任务上增益最大。
6. **计算效率优越**：在短中上下文长度下更快且输出更少令牌；位于VRAM-准确率的帕累托前沿。
7. **头数量增加持续改善MRR**：从1头到16头，MRR近似逻辑增长（896K时从26.53%提升至87.12%）。
8. **阶段式设计优于并发设计**：MHM-Concur虽然MCR更高，但MRR更低，导致准确率低于MHM-LRU。
9. **简单规则选择优于模型驱动选择**：MHM-Relevance因噪声和覆盖不足而表现更差。

## 7. 优点

- **无需训练**：MHM是通用框架，不要求修改模型权重或任何训练，直接与现有LLM兼容。

- **计算开销低**：LRU选择只需年龄比较，比模型驱动更高效，且实际输出令牌数在短中长度下更少。
- **记忆容量一致下的公平比较**：严格控制总令牌数，确保非参数增加带来的优势。
- **诊断驱动设计**：先分解性能瓶颈，再针对性地设计架构，逻辑链条清晰。
