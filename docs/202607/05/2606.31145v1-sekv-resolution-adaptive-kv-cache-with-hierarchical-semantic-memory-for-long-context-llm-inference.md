## 摘要
大型语言模型越来越多地处理长上下文场景，其中KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，且必须在整个解码过程中保留，这使得如果没有压缩，完整的GPU缓存将极其昂贵。现有的KV缓存压缩方法难以在效率与忠实上下文保留之间取得平衡。Token驱逐会丢失信息，而语义分组在预填充阶段固定压缩决策；一旦在生成过程中压缩段变得相关，这两种方法都无法恢复token级别的细节。作为解决方案，我们提出SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织成熵引导的语义段，并在GPU-CPU内存层次结构中存储它们，而不丢弃信息。每个段在GPU上保留一个轻量级摘要向量用于粗粒度路由，并在CPU上保留一个低秩SVD基用于按需的token级重建。一个训练出的“缩放”机制在解码过程中选择性地扩展与查询相关的段，从而无需在GPU上实例化完整的KV缓存即可实现精确检索。SeKV实现了自适应的token级重建，同时保持基础LLM完全冻结，并添加少于0.05%的可训练参数。在四个基准测试中，与最强的语义压缩基线相比，SeKV平均提升5.9%，同时与128K上下文下的完整KV缓存相比，GPU内存减少53.3%。代码可在https://github.com/AmirAbaskohi/SeKV获取。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：大语言模型（LLM）在进行长上下文推理时，KV缓存（Key-Value cache）大小随序列长度线性增长，成为严重的内存瓶颈。现有的KV压缩方法（如token eviction）永久丢弃信息，或（如语义分组）在预填充阶段固定压缩决策，都无法在解码过程中当某个压缩段变得相关时恢复其token级细节。
- **整体含义**：长上下文推理需要一种不丢弃信息、并能在解码时按需动态调整缓存分辨率的方案。本文提出SeKV，一种分辨率自适应的语义KV缓存，通过分层语义内存组织，实现高效且可恢复的缓存管理。

## 2. 方法论
- **核心思想**：将上下文组织成熵引导的语义跨度（span），每个跨度在GPU上保留轻量摘要向量用于粗略路由，在CPU上保留低秩SVD基用于按需的token级重建。训练一个zoom-in机制，在解码时动态选择查询最相关的跨度进行扩展和重建。
- **关键技术细节**：
  - **熵引导跨度分割**：利用预填充阶段token的surprisal（−log p(xt|x<t)）作为边界信号，当surprisal超过μ+ασ时标记为边界，边界token作为锚点保留全精度在GPU上，中间形成语义跨度。
  - **双分辨率表示**：每个跨度在GPU上存储一个投影后的摘要向量（通过可学习的每头每层投影W^(l,h)将key投影到d′=32维），以及原始空间中的加权平均key和value。在CPU上存储key和value矩阵的截断SVD分解（最大秩R=32，通过轻量预测器gϕ分配每个分量的软门控m_i），用于近似重建。
  - **训练的zoom-in机制**：每个解码步，将query投影到路由空间，与所有跨度摘要计算sigmoid门控得分（含log|S|长度偏置），超过学习到的阈值τ^(l,h)则触发扩展，异步从CPU取回SVD基重建token级KV，参与精细注意力。未扩展的跨度使用粗粒度均值参与注意力。
  - **训练目标**：四部分损失：蒸馏损失（对齐完整KV教师）、zoom监督（二分类交叉熵，目标来自教师注意力覆盖集）、重建损失（监督软门控）、预算损失（惩罚扩展跨度数和总秩系数）。基础LLM冻结，仅训练路由投影、阈值和秩预测器（共约4.3M参数，<0.05%）。
- **推理流程**：GPU常驻锚点token、跨度摘要和粗均值；CPU存储SVD基。解码时先进行Stage 1路由，确定哪些跨度zoom-in，异步取回基；Stage 2重建并计算混合分辨率注意力。

## 6. 主要结论与发现
- SeKV在所有压缩方法中表现最优，平均提升最强语义基线SentenceKV 5.9%，且仅使用10% KV预算时接近Full KV（LongBench差0.80，InfiniteBench差1.23）。

- 多示例推理（GSM8K）上SeKV也保持领先，表明不仅保留检索能力，也保留推理模式。
- GPU内存几乎不随上下文长度增长（8K→128K仅从31.2GB到34.9GB），Full KV从36.0GB到74.8GB。

## 7. 优点
- **不丢弃信息**：通过GPU-CPU层次和低秩SVD基，所有上下文均可恢复，无永久丢失。
- **自适应分辨率**：训练的zoom-in机制根据查询动态调整，与静态压缩方法相比更灵活。
- **轻量训练**：仅添加<0.05%参数，基础LLM冻结，训练成本低（2-6小时）。
- **广泛的验证**：在多种模型、基准、预算下均取得一致优势，消融和敏感性分析完备。
- **效率与精度平衡**：比Full KV快且内存少，比强压缩方法精度更高。
- **设计与分析透明**：提供了zoom-in行为热力图，解释模型内部如何工作。

## 8. 不足与局限
- **依赖语义跨度质量**：熵引导分割在高度碎片化文档、表格、代码或非常规格式中可能产生噪声边界，导致次优内存分配。
- **超参数敏感**：熵阈值α、锚点数量、秩、检索预算等需针对新模型或上下文长度进行调整，可能存在过拟合风险。
- **依赖CPU-GPU带宽**：在慢速传输或对抗性查询（重复激活大量跨度）场景下，延迟优势可能减小。
- **冻结基础LLM**：虽然有利部署，但可能限制模型完全适应压缩内存结构；未来可考虑联合训练更丰富的跨度表示或自适应重建策略。
- **实验涵盖模型还不够极大规模**：最大为14B，未测试更大模型（如70B+）的泛化。
