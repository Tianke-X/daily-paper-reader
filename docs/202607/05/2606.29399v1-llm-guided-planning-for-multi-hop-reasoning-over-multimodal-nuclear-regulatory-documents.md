## 摘要
审阅核监管文档需要在数万页文档中进行多跳推理，判断依据来自跨多个章节的证据。我们将此任务定义为规划问题：基于LLM的智能体观察已收集的证据，选择下一个要检查的文档片段，并在证据充分时停止。智能体通过浏览、阅读和搜索工具在无向量的文档树上操作，并维护一个动态知识图谱作为状态。在NuScale最终安全分析报告文档上的200题基准测试中，该系统达到81.5%的准确率，RAGAS忠实度为0.93。主导性能因素为规划：与使用相同文档树但无状态条件动作选择的PageIndex相比，差距为+38.0个百分点（43.5% vs 81.5%，p<0.001）。该系统还优于LightRAG（73.0%，p<0.05）、HippoRAG（70.5%，p<0.01）和GraphRAG（49.5%，p<0.001），且在没有离线索引的情况下与RAPTOR（75.5%，p=0.11）持平。边的推理增加了2.8倍成本而未提高准确率；我们将其保留作为可追溯性模块。在推断的7,391条边中，3条违反边（0.04%）标记了范围边界（Q058）和部分符合性（Q176）作为类型化注释，供人工审阅者审计。

## Abstract
Reviewing nuclear regulatory documents requires multi-hop reasoning across tens of thousands of pages, where judgments depend on evidence assembled across multiple chapters. We frame this task as planning: an LLM-based agent observes the evidence collected so far, picks the next document fragment to inspect, and stops when the evidence is sufficient. The agent operates over a vectorless document tree using browse, read, and search tools, and maintains a dynamic knowledge graph (KG) as state. On a 200-question benchmark over NuScale Final Safety Analysis Report (FSAR) documents, the system reaches 81.5% accuracy with a RAGAS Faithfulness of 0.93. The dominant performance factor is planning: against PageIndex, which uses the same document tree without state-conditioned action selection, the gap is +38.0pp (43.5% to 81.5%, p<0.001). The system also outperforms LightRAG (73.0%, p<0.05), HippoRAG (70.5%, p<0.01), and GraphRAG (49.5%, p<0.001), and matches RAPTOR (75.5%, p=0.11) without offline indexing. Edge inference adds 2.8x cost without raising accuracy; we retain it as a traceability module. Of 7,391 inferred edges, 3 Violates edges (0.04%) flag scope boundaries (Q058) and partial conformance (Q176) as typed annotations that a human reviewer can audit.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **整体含义**：将多跳核监管文档审查建模为规划问题，通过LLM智能体在无向量的文档树上使用浏览/阅读/搜索工具，并维护动态知识图谱作为状态，实现训练-free的、状态条件化的迭代规划，显著提升检索准确性并提供可审计的推理路径。

## 2. 论文提出的方法论

### 核心思想
- 将监管文档审查形式化为规划问题 ⟨S, A, f_tr, φ⟩，其中：
  - 状态s_t：动态子知识图谱G_t = (V_t, E_t)，记录经过t跳后收集的证据及其推断关系。
  - 动作a_t：从{browse(d,v), read(d,v), search(κ)}中选择，模拟文件系统操作。
  - 转移函数f_tr(s_t, a_t) → s_{t+1}：执行工具后整合新节点，可选进行边推断。
  - 目标测试φ(s_t, q) ∈ {0,1}：LLM判断当前KG是否包含足够证据回答问题，若充足则终止。

### 关键技术细节
1. **环境：无向量多模态文档树**
   - 将PDF解析为JSON分层树（章节→节→段落），不进行分块或嵌入。
   - 解析图表列表和文本引用，将图表元数据附加到对应节点，解决“图表在不同页”问题。
   - 检索使用BM25Okapi关键词搜索，章节标题权重3倍，文档长度归一化自然促进短节点排名。

2. **状态：动态子知识图谱与双层边本体**
   - 节点集V_t包含收集的文档章节及其多模态引用。
   - 边集E_t采用两层本体：Tier 1结构性边（REFERENCES, SPECIFIES）组织探索轨迹；Tier 2语义边（SATISFIES/VIOLATES, SUPPORTS/CONTRADICTS, LEADS TO, IS PREREQUISITE OF）用于监管合规判断。
   - 保留置信度≥0.4的边。

3. **动作规划：LLM-based工具选择**
   - 闭环规划：观察当前KG状态，决定动作，立即集成环境反馈后再选下一步动作。
   - 强制Hop 1浏览优先（注入目录），使智能体先获得全局地图再搜索（该干预将单证据Context Recall从0.45提升至0.89）。
   - 使用伪相关反馈（PRF/RM3）自动扩展查询，维护搜索历史避免重复关键词。
   - 动态终止：从Hop 2开始，每跳前检查是否已有足够证据回答，若充足则提前终止（33%查询在1-3跳终止，平均3.4跳，最大4跳）。

4. **事后边推断（可选组件）**
   - 分两阶段：阶段1用LLM生成两节点间关系的自然语言描述；阶段2将其映射到监管本体（如SATISFIES/VIOLATES）。不可映射时保留为SEMANTIC，不丢弃信息。
   - 在每跳证据收集后立即执行，使推断结果人类可审计。

5. **视觉增强最终答案生成**
   - 仅在最终答案生成步骤使用多模态处理（中间操作保持纯文本），收集所有图表引用，渲染PDF页面为JPEG，附带文本KG上下文一起送入GPT-4.1视觉语言模型。
   - 表格使用PyMuPDF的find_tables()直接提取结构化文本，无需VLM图像处理。

### 算法流程（文字说明）
1. 初始化空KG G_0，注入目录（Hop 0）。
2. 循环 t=0 到 T-1：
   - LLM根据查询q、当前KG G_t和目录规划动作a_t。
   - 执行动作（浏览/阅读/搜索），得到新证据。
   - 整合新节点并进行边推断（可选），更新G_{t+1}。
   - 如果 t≥1 且 LLM判断已足够证据，则提前终止。
3. 最终生成：将收集的KG上下文和渲染的图表图像送入VLM，得到视觉增强答案。

## 6. 论文的主要结论与发现

1. **状态条件化规划是多跳推理的主导因素**：与无规划基线PageIndex的+38.0pp准确率差距（p<0.001），以及优于LightRAG等方法的显著性，证明规划机制是性能关键。
2. **边缘推断不提升准确率但提供可追溯性**：200题尺度消融显示准确率无差异，但3条VIOLATES边（0.04%）标记了范围边界和部分符合性，满足了10 CFR 50 Appendix B的可追溯性要求。
3. **动态终止机制可靠**：提前终止查询准确率高于全额预算查询，防止在困难问题上过早停止的风险。
4. **视觉增强仅在最终步骤有效**：仅在答案生成时使用视觉处理，避免了中间迭代的高成本，在仅表格问题上比RAPTOR高+18pp（86.0% vs 68.0%）。
5. **双重评估框架的必要性**：RAGAS衡量接地质量，Judge衡量正确性，任何单一指标会遗漏重要案例（如29例接地好但Judge错属于措辞差异，38例接地差但Judge正确属于外部知识）。
6. **实际部署失败率低**：排除措辞差异后，真实失败率4.5%，幻觉仅1.0%，在安全关键领域可管理。

## 7. 优点

- **方法论创新**：将文档审查建模为规划问题，使用无向量文档树+动态知识图谱状态+闭环规划，训练-free、可审计。
- **环境设计合理**：保留文档原生层次结构，避免分块破坏，通过浏览/阅读/搜索工具类比文件系统操作，直观且灵活。
- **多模态集成高效**：仅在最终步骤使用视觉API，避免昂贵的中维视觉调用，同时有效处理表格和图表。
- **边本体具有领域特异性**：基于SysML、论证挖掘、因果KG等前沿工作，设计的双层本体（Tier1结构+Tier2语义）能捕捉监管审查核心关系（SATISFIES/VIOLATES等），并且保留语义关系不丢失信息。
- **双评估体系**：结合自动指标（RAGAS）和LLM裁判，互补评估接地质量和正确性，提供更全面的性能画像。
- **消融实验全面**：分别隔离规划、边缘推断、浏览优先、视觉处理的效果，并在10题和200题两种尺度验证关键发现。
- **可部署性考虑**：动态终止节省33%查询，每查询成本$0.076（仅规划）在监管背景下可接受，并提供了失败模式分类指导人工审查优先级。

  - 基准为自建，无外部验证，但通过三轴正交设计和5种方法一致比较部分缓解。

- **方法局限性**：
  - 在纯文本问题上（76.2% vs RAPTOR 80.0%，-3.8pp），RAPTOR的递归总结树可能有益，作者建议增加总结节点。
  - 每查询成本（$0.215含边缘推断，93秒）是RAPTOR的42倍，但对于监管任务中人类数小时每问题而言可接受。
  - 与RAPTOR的差距（+6.0pp）在n=200下未达统计显著性（McNemar p=0.11），需更大规模验证。
  - 缺少“follow reference”工具直接导航“see Table 5.1-1”类指针。
  - max hops=4的天花板：67%查询使用满4跳，增加预算是否能提升复杂问题准确率未探索。

- **边本体覆盖不足**：8种关系集针对FSAR符合性审查，不覆盖其他监管框架所需的跨管辖权适用性、时间有效性、条件适用性、工程先例等关系。部署到其他领域需要领域专家本体整理和人工监督。

- **实验范围有限**：
  - 仅使用GPT-4.1一种生成模型，未测试其他LLM的泛化能力。
  - 未进行跨文档、跨语言或真实用户评估。
  - 边缘推断的三种VIOLATES案例仅来自单个FSAR corpus，是否在其他监管文档中普遍存在未知。
  - 未评估系统对对抗性查询或非合规场景的鲁棒性。
