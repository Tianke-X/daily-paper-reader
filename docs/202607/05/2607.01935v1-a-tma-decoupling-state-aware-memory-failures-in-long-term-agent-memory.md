## 摘要
长期记忆使LLM智能体能够充当持久助手，但用户事实会发生变化。一个有用的记忆系统必须知道现在什么是真的，过去什么是真的，以及什么发生了变化。我们研究“幽灵记忆”——一种状态协调故障，其中旧的、当前的和过渡的事实共存于记忆库中，在检索时保持混合，并误导答案模型。我们认为，应从三个层面理解和优化记忆系统：库维护、检索和答案时间分辨率。我们提出ATMA，一种针对现有记忆系统的状态感知覆盖层。ATMA在库中保留被取代的记录和过渡记录，为查询的请求状态视图构建证据包，并向问答模型暴露当前、历史和过渡标签。我们进一步呼吁对库、检索和答案层面的故障进行解耦评估，因为最终问答准确性可能隐藏幽灵记忆的发生位置。为使这种故障可测量，我们构建了LTP（LoCoMo Temporal Plus），一个针对幽灵记忆的高冲突基准，并在LoCoMo上评估长对话泛化能力。在LTP上，Graphiti+ATMA相比Graphiti将冲突准确性绝对提升了0.240。在LoCoMo上，Graphiti+ATMA将时间F1从0.0295提升至0.1705。收益依赖于宿主，但它们表明显式状态角色可以减少被最终问答准确性隐藏的记忆故障。

## Abstract
Long term memory lets LLM agents act as persistent assistants, but user facts change. A useful memory system must know what is true now, what used to be true, and what changed. We study \emph{ghost memory}, a state coordination failure in which old, current, and transition facts coexist in the memory bank, remain mixed during retrieval, and mislead the answer model. We argue that memory systems should be understood and optimized from three levels: bank maintenance, retrieval, and answer time resolution. We propose ATMA, a state aware overlay for existing memory systems. ATMA keeps superseded and transition records in the bank, builds evidence packets for the query's requested state view, and exposes current, historical, and transition labels to QA. We further call for decoupled evaluation of bank, retrieval, and answer level failures, since final QA accuracy can hide where ghost memory occurs. To make this failure measurable, we build LTP (LoCoMo Temporal Plus), a conflict heavy benchmark for ghost memory, and evaluate on LoCoMo for long conversation generalization. On LTP, Graphiti+ATMA improves conflict accuracy by 0.240 absolute over Graphiti. On LoCoMo, Graphiti+ATMA raises temporal F1 from 0.0295 to 0.1705. The gains are host dependent, but they indicate that explicit state roles can reduce memory failures hidden by final QA accuracy.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：LLM 智能体从单轮对话走向持久辅助，需要长期记忆来记住用户偏好、项目决策、约束以及早期对话。然而用户事实是动态变化的（如地址变更、离职、计划修改）。
- **核心问题**：传统记忆系统将每个存储的事实视为永恒正确，导致**“幽灵记忆”（ghost memory）**——旧事实、当前事实和过渡性事实共存于记忆库，检索时混合在一起，误导答案模型。这是一种**状态协调故障**（state coordination failure），并非简单的记录存在与否。
- **关键洞察**：正确的响应不是删除旧事实，而是保留历史的同时，在检索和回答时显式暴露每个记忆的状态角色（当前、历史、过渡）。现有系统（基于时间戳、图结构、控制器等）仅部分满足这一需求，最终答案准确性可能隐藏了故障发生的位置。
- **研究目标**：提出一种方法，将长期记忆状态变化问题解耦为三个层次——库维护（bank）、检索（retrieval）和答案生成（QA），分别诊断和优化，从而减少幽灵记忆。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想：ATMA（Adaptive Truth Maintenance Auditing）

ATMA 是一种**状态感知覆盖层**（state-aware overlay），不替换宿主记忆系统，而是在宿主流水线周围添加状态语义，使得每个层次可被检查。

### 关键技术细节

#### 2.1 银行级状态维护（Bank Level）
- **目标**：避免历史事实被默认视为当前事实，同时不清除历史。
- **Sentry 候选提议**：新记忆到达时，通过轻量级模型（基于 nomic-embed-text-v1.5 的 SentenceTransformer + 两个投影头：主题头 z_top 和逻辑头 z_log）检测与新记忆可能共享同一状态槽的旧记录。仅当主题相似且逻辑可疑时，才送入 Judge 审计。
- **公式**：对文本 x，计算 h_x = φ_θ(x)，z_top = norm2(g_top(h_x))，z_log = norm2(g_log(h_x))。使用相似度 a_ij（主题一致性）、b_ij（逻辑兼容性）和差距 d_ij 决定是否审计。
- **Judge 审计与提交**：使用 Qwen2.5-3B-Instruct 模型（经 QLoRA SFT 和 GRPO 优化）对可疑对进行结构化判决：冲突状态、冲突类型、融合动作、转换元数据等。旧记录状态从 active 改为 superseded，新记录变为 active，并建立相互链接。
- **训练目标**：Sentry 使用监督成对损失 L_sentry；Judge 使用 GRPO 规则奖励（格式、正确性、旧状态保留、转换摘要等）。

#### 2.2 检索级证据构建（Retrieval Level）
- **目标**：使检索到的证据与查询请求的状态视图对齐。
- **查询状态视图**：使用轻量级基于规则的查询分析器（非 LLM），分配视图 v_q ∈ {current, historical, transition, neutral}。
- **种子与跳池**：宿主检索器提供种子集 A_q；通过状态链接（supersedes, superseded by, transition）扩展得到 R_q；通过弱邻居链接扩展得到 N_q。合并去重后得到候选池 C_q。
- **排序**：默认使用稳定预排序 π_0（保留原始桶顺序）。可选地，使用检索控制器（与 Judge 同一模型）重新排序，最多选择 3 个候选，剩余由预排序填充。控制器输出包含视图预测、置信度、角色标签等。

#### 2.3 答案级状态条件（QA Level）
- **目标**：防止答案模型将不同状态的证据混淆。
- **迹转标签**：将每个检索条目的元数据（status, relation, transition type 等）映射为简短标签：cur、hist、tran、link、raw。
- **状态显式证据提示**：序列化证据时，附加状态角色标签和可选时间字段，然后通过提示规则 R_qa 约束生成器仅使用检索记忆，并根据查询状态选择正确状态。
- **公式**：最终答案 â = G_H(P_q)，其中 P_q 是带状态标签的证据提示。

#### 算法流程（文字说明）
1. 银行侧：初始化状态银行，遍历输入流，对每个新记忆收集候选先验记录，通过 Sentry 筛选可疑对，Judge 审计后提交状态标签和链接。
2. 检索侧：分析查询状态视图，利用宿主检索器和关系扩展构建候选池，排序得到最终证据包。
3. QA 侧：将证据序列化为带标签形式，通过提示约束答案模型按请求状态回答。

## 6. 论文的主要结论与发现

1. **显式状态角色可减少幽灵记忆**：在 LTP 上，Graphiti/Zep+ATMA 将冲突准确率从 0.480 提升至 0.720（绝对 +0.240），InsideOut+ATMA 将 Acc 从 0.117 提升至 0.662。

## 7. 优点：方法或实验设计上的亮点

- **结构清晰的问题分解**：将记忆状态协调问题分解为三个独立但连接的水平，便于诊断和优化。
- **非侵入式覆盖层设计**：ATMA 不替换宿主系统，而是添加状态语义，易于集成到现有记忆系统中。
- **显式状态标签**：在 QA 级通过标签（cur/hist/tran/link）强制生成器使用正确状态，避免模型从文本推断，显著提升状态敏感问答。
- **构建 LTP 基准**：专门针对幽灵记忆设计的高冲突基准，具有可控的状态变化和双视图探测（当前/历史），补充了现有长对话基准在状态变化方面的不足。
- **全面的消融和案例**：通过组件消融和定性案例展示了每个模块的具体贡献，验证了解耦设计的必要性。
- **公平的实验设置**：保持检索预算、QA 模型、评估法官一致，控制变量。
