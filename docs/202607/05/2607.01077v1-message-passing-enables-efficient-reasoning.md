## 摘要
尽管推理时扩展（inference-time scaling）提升了大型语言模型（LLMs）的推理能力，但生成长思维链（CoT）的需求却成为计算瓶颈。因此，与CoT等顺序扩展方法不同，最近的并行扩展技术转而使用分叉与合并（FJ）原语，将工作分配到多个LLM线程中。然而，在分叉-合并范式下，线程通常是短暂的，且彼此之间不进行逐点通信，这限制了可扩展性。为解决这一问题，我们引入了消息传递语言模型（MPLM），这是一种LLM推理框架，其中线程通过轻量级的发送和接收原语直接通信。MPLM通过两种关键机制实现高效扩展：（1）减少通信开销，通过避免冗余的上下文共享；（2）抢占（preemption），允许线程基于来自同伴的部分信息提前终止。我们在三类任务上展示了MPLM的前景。首先，在数独谜题中，我们证明MPLM所需的上下文比串行CoT和并行FJ都渐近地更小。然后，我们微调了一个单一模型来解决25×25的数独谜题，这对标准CoT和FJ方法以及不使用工具的前沿推理模型来说仍然具有挑战性。其次，在3-SAT谜题中，抢占能力允许提前终止无前途的分支，从而提高了效率。最后，我们展示了通过适当提示的大型预训练模型遵循MPLM协议，在长上下文问答任务中取得了与流行的分叉-合并方法相竞争的结果。

## Abstract
While inference-time scaling has improved the reasoning abilities of large language models (LLMs), the need to generate long chains-of-thought (CoTs) is a computational bottleneck. Thus, in contrast to sequential scaling methods like CoT, recent parallel scaling techniques instead use fork and join (FJ) primitives to divide work across multiple LLM threads. However, in the fork-join paradigm, threads are typically transient and do not communicate pointwise with one another which limits scalability. To tackle this, we introduce Message Passing Language Models (MPLMs), a framework for LLM reasoning in which threads communicate directly via lightweight send and receive primitives. MPLMs enable efficient scaling through two key mechanisms: (1) reduced communication costs, achieved by avoiding redundant context sharing, and (2) preemption, which allows threads to terminate early based on partial information from their peers.   We demonstrate the promise of MPLMs on 3 classes of tasks. First, on Sudoku puzzles, we show that MPLMs require an asymptotically smaller context than both serial CoT and parallel FJ. We then fine-tune a single model to solve 25 x 25 puzzles that remain challenging for standard CoT and FJ approaches, as well as frontier reasoning models without tools. Second, on 3-SAT puzzles, the capability of preemption allows termination of unpromising branches, which results in improved efficiency. Finally, we show that appropriately prompted large pre-trained models follow the MPLM protocol, achieving competitive results on long-context question answering relative to popular fork-join approaches.

## 1. 核心问题与整体含义（研究动机和背景）

大型语言模型（LLMs）通过推理时扩展（如长思维链 CoT）显著提升了推理能力，但生成长序列 token 的计算开销巨大，且 Transformer 注意力机制的二次复杂度以及位置编码限制使得上下文长度成为瓶颈。现有的并行推理方法（如 Fork-Join 范式）允许分解任务到多个线程并行执行，但线程之间缺乏直接通信，所有信息需通过单一主线程聚合，导致通信瓶颈和隐式串行化，限制了可扩展性。受分布式系统中消息传递接口（MPI）启发，本文提出消息传递语言模型（MPLM），通过轻量级的 send/receive 原语使线程直接通信，并支持基于部分信息的提前终止（preemption），从而更高效地利用测试时计算。

## 2. 方法论

### 核心思想
- **MPLM 框架**：将推理分解为多个持久化、半独立的线程，线程间通过显式的点对点消息传递进行协调。模型自身生成特殊指令（directives）控制执行。
- **四大原语**：
  - `<spawn>`：创建新线程并附带初始 prompt。
  - `<send>`：向指定线程发送消息（payload）。
  - `<recv>`：阻塞等待来自指定线程的消息（支持 wait-for-all 或 wait-for-any 两种实现）。
  - `<stop>`：终止当前线程。
- **推理逻辑**：一个轻量级控制器在批推理引擎（如 vLLM）之上运行，协调所有线程的并行解码。当线程生成指令边界时，控制器解析并执行相应动作（如发送消息、阻塞线程、终止等）。
- **抢占机制**：一旦某个线程获得足够信息（如 SAT 问题中找到一个解），它可向父线程发送消息，父线程可立即取消所有未完成子线程的执行，避免无效计算。
- **重生成（Respawning）**：线程可重新生成自身，附带压缩摘要，从而限制上下文长度增长，允许无限迭代。

### 关键技术细节
- **稀疏通信**：每个线程仅与少量邻居通信（如数独中同行、列、宫的单元格），避免全聚合。
- **上下文效率**：理论分析表明，在稀疏通信假设下，MPLM 的最大上下文复杂度为 O(T k M)，而 Fork-Join 为 O(T N M)，Serial CoT 为 O(T N k M)，其中 T 迭代次数，N 线程数，k 邻居数，M 消息大小。
- **训练数据生成**：通过模拟消息传递过程（如数独用 naked-singles 策略，SAT 用 DPLL 算法）生成 CoT 轨迹，用于监督微调（SFT）。

## 6. 主要结论与发现

1. **MPLM 实现更优的渐近扩展**：在 Sudoku 上，MPLM 的顺序 token 和最大上下文长度的幂律指数（α ≈ 1.1–1.2）显著低于 Serial（≈1.8）和 FJ（≈1.8），表明随问题规模增长计算成本增长更慢。
2. **超越前沿模型能力**：MPLM 在 25×25 数独上达到 72% 准确率，而 DeepSeek-R1 和 GPT-5 Pro 分别仅为 0% 和 20%（未使用工具），且 MPLM 模型参数仅 0.6B。
3. **抢占机制提升效率**：在 3-SAT 上，MPLM 通过抢占平均加速比达 1.7×–3.5×（与 FJ 相比），尤其在搜索树不平衡时优势更大。
4. **提示词即可使用 MPLM 协议**：在 LongBench-v2 上，未经过 SFT 的较大模型（Qwen3-30B 和 Qwen3.6-35B）通过 prompt 遵循 send/recv 指令，在相同或更高准确率下将延迟降低至 FJ 方法的约 1/2（平均从 223.5s 降至 102.2s）。
5. **理论支撑**：证明了 MPLM 的最大上下文复杂度远低于 Serial 和 FJ，解释效率提升根源。

## 7. 优点

- **创新性**：首次将消息传递原语（类似 MPI）集成进 LLM 推理，使得线程间具备直接通信和抢占能力，突破了 fork-join 的局限。
- **理论与实践结合**：除了实验验证，还给出了严格的渐近复杂度证明（定理 4.1）。
- **低资源门槛**：使用 0.6B 小模型通过 SFT 即能在结构任务上超越大型前沿模型，说明框架本身的高效性。
- **多场景验证**：覆盖逻辑谜题（Sudoku、SAT）和实际长上下文 NLP 任务，体现了通用性。
- **系统开销极小**：Profiling 显示控制器开销 <2%，消息 payload token 占比 <8%（Sudoku 中），通信成本可控。
- **灵活的 recv 实现**：支持 wait-for-all 和 wait-for-any，可适应不同任务需求（如 SAT 适合 wait-for-any）。

## 8. 不足与局限

- **通信模式依赖先验知识**：在 Sudoku 和 SAT 中，邻居关系/分支结构已给定，但开放任务中线程间通信模式需模型自行判断，可能复杂且需谨慎调 prompt 或训练。
- **不适用于非结构化任务**：论文未验证 MPLM 在数学推理、代码生成等需复杂序列依赖的任务上的性能。
- **少量训练数据下的长上下文性能**：LongBench 上使用 prompt 而非 SFT，准确率与 RLM 相当但并非大幅超越；且结果仅基于单一 prompt 模板，未探索最优化 prompt。
- **缺乏方差统计**：实验未报告多次运行的标准差或置信区间，无法判断显著性。
- **比较公平性**：Sudoku 上与前沿模型的比较未考虑它们是否经过类似任务微调，可能不公平。
- **可扩展性局限**：Sudoku 中每单元格一个线程，对于 N=5（25×25）已有 625 个线程，若问题更复杂（如更大网格或更多交互），线程数可能过大，带来调度开销。
- **仅支持有限的原语**：当前仅支持 send/recv，未来可引入 broadcast、allgather 等以提高表达能力。
- **仅针对单一模型**：训练和评估主要基于 Qwen 系列，未在其他架构（如 Llama、Mistral）上验证泛化性。
