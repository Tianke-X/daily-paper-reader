## 摘要
现有的检索增强生成（RAG）系统将网页视为平面文本，丢失了HTML中编码的结构和语义信号。我们提出了PolyUQuest，这是一个可验证的、结构感知的网页RAG框架，构建在异构图之上，统一了页面间的超链接拓扑、页面内的DOM层次结构以及跨页面的实体关系知识。一个双层路由器将每个查询分派到与其结构需求匹配的三种检索模式之一，包括直接块检索、跨页面图遍历和多跳实体推理。每个答案都是完全可验证的，因为每个引用的块都携带其源页面、标题路径和实体链接，使用户能够将任何声明追溯至其结构证据。我们在香港理工大学（PolyU）的官方网站上进行评估，该网站包含4,240个页面、31,086个DOM块、29,119个实体和37,680个关系，并配有一个多类型评估基准。PolyUQuest在答案正确性、覆盖率和忠实度方面优于现有RAG系统，同时每个查询消耗的LLM词元显著更少。演示提供了一个交互式界面，用于检查引用的答案、比较各路由模式的检索轨迹，以及探索证据图路径。PolyUQuest正准备部署为面向学生的PolyU问答服务。

## Abstract
Existing retrieval-augmented generation (RAG) systems treat web pages as flat text, losing the structural and semantic signals encoded in HTML. We present PolyUQuest, a verifiable, structure-aware web RAG framework built on a heterogeneous graph that unifies hyperlink topology between pages, DOM hierarchy within pages, and entity-relation knowledge across pages. A two-tier router dispatches each query to one of three retrieval modes matched to its structural need, including direct block retrieval, cross-page graph traversal, and multi-hop entity reasoning. Every answer is fully verifiable, as each cited block carries its source page, heading path, and entity links so that users can trace any claim back to its structural evidence. We evaluate on the official websites of the Hong Kong Polytechnic University (PolyU), comprising 4,240 pages, 31,086 DOM blocks, 29,119 entities, and 37,680 relations, together with a multi-type evaluation benchmark. PolyUQuest outperforms existing RAG systems in answer correctness, coverage, and faithfulness, while consuming significantly fewer LLM tokens per query. The demonstration provides an interactive interface for inspecting cited answers, comparing retrieval traces across routing modes, and exploring evidence graph paths. PolyUQuest is being prepared for deployment as a student-facing QA service at PolyU.

## 1. 核心问题与整体含义（研究动机与背景）

现有检索增强生成（RAG）系统在处理网页知识时，普遍将网页视为平铺的纯文本，丢失了 HTML 中编码的结构信号。网页知识实际包含三个互补层次：页面间超链接拓扑、页面内 DOM 层次、跨页面的命名实体关系。例如查询“计算系哪些教授做 NLP 研究并教授相关课程？”需要跨页面跟踪超链接和实体，而现有的基于文本块或纯向量相似性的 RAG、图 RAG 以及 HTML 结构感知的 RAG 均无法同时利用这三层结构，导致要么遗漏跨页证据，要么消耗大量 token 和延迟。PolyUQuest 旨在通过构建统一的异构图，实现可验证、结构感知、低 token 消耗的网页 RAG。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
将网站建模为一个异构四类节点图：网页节点、证据块节点、实体节点、主题节点，边包含页面超链接、页面-块包含、块-实体提及、实体-主题语义关联。通过两级路由器将用户查询分派到三种结构匹配的检索模式，只检索结构上相关的证据，避免全局上下文注入。

### 2.2 关键技术细节
- **离线索引**：
  - **Web Crawling**：爬取 PolyU 官网页面并记录超链接。
  - **DOM Evidence Construction**：清除脚本/样式/注释等非内容区域，自底向上合并 DOM 兄弟子树直到达到 150 词阈值，形成标题路径感知的块，保留表格、列表、代码为原子块，过滤导航栏/页脚等模板。
  - **Entity Extraction & Resolution**：用 LLM 从每个块提取实体（人员、项目、课程、研究主题等）和关系，并链接回源块。通过归一化、语义字符串相似度合并歧义实体，边界情况用 LLM 决策。
- **在线检索**：
  - **两层级路由器**：第一层基于规则（如“which professors”→模式C, “admission requirements for”→模式B, 比较模式→模式B）；未匹配的查询进入第二层 LLM 分类器，输出三个模式及置信度。
  - **模式A：直接块检索**：适用于单跳事实问题。先通过稠密近似最近邻 (ANN) + BM25 混合检索得到候选块集，然后用交叉编码器重排序，加入源页面标题和标题路径上下文。
  - **模式B：导航检索**：适用于跨页面聚合/比较。LLM 将查询分解为子查询，对每个子查询检索相关页面并沿超链接扩展，汇集所有子查询候选块，再执行全局交叉编码器重排序。
  - **模式C：实体推理**：适用于多跳证据问题。LLM 提取主题关键词，通过两条路径扩展实体：一是从候选实体沿关系扩展，二是从主题到关联实体。最终使用综合评分公式：
    \[
    s(b,q) = \cos(\mathbf{x}_q,\mathbf{x}_b) + \frac{\min(|V_E(b)\cap V_{E,q}|, \kappa)}{\kappa} + \left(1 - \frac{\ell(b)}{\max_{b'\in B_q}\ell(b')}\right)
    \]
    三项分别为：查询-块相似度、限制实体覆盖率（κ=30）、块简洁度，并加入同一页面惩罚以鼓励来源多样性。

## 6. 主要结论与发现

- PolyUQuest 在所有三个质量指标上均优于所有基线：正确性 0.644（次优 LightRAG 0.610）、覆盖率 0.649（次优 LightRAG 0.612）、忠实度 0.921（次优 HtmlRAG 0.804）。
- 查询 token 消耗极低（2,968），比 LightRAG 少约 10 倍，比 HtmlRAG 少 26%，且质量更高。这是因为路由机制避免了全局上下文注入。
- 消融实验表明 DOM 结构感知块是质量优势的主要驱动（移除后正确性下降 8.1 点，覆盖率下降 13.9 点）；跨页模式贡献较小但依然重要。
- LightRAG 忠实度低（0.559）是因为直接注入 LLM 生成的社区摘要而非原始文本，导致 claim 无证据支撑。

## 7. 优点

- **结构统一**：首次将超链接、DOM 层次和跨页实体关系统一到一个异构图，支持灵活的三层结构推理。
- **路由机制高效**：两层级路由器根据查询结构需求选择最小足够的检索模式，避免全局上下文浪费，大幅减少 token 消耗。
- **完全可验证**：每个 citation 均携带源页面 URL、标题路径和实体链接，用户可点击追溯，界面支持对比检索轨迹和图路径探索。
- **完备的离线索引**：包含去模板、原子块保护、实体消歧等工程优化，实用性强。
- **性能突出**：在忠实度上领先 36 个点，同时 token 消耗最低。
