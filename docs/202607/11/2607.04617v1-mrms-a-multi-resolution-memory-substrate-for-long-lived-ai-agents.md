## 摘要
长寿命AI智能体需要在交互间保持连续性，但这种连续性无法仅通过扩展提示窗口实现。智能体必须保留有用的先前经验、选择性检索、区分个人语境与外部证据，并在底层情境变化时修正记忆。我们提出了一种架构化的记忆基底，沿两个正交轴组织：一个表示轴，涵盖结构化记录、向量表示和图关系；另一个时间轴，涵盖短期痕迹、中期抽象和长期语义承诺。其关键设计约束是同步的结构化-向量-图记忆：结构化记录管理资格，向量表示支持回忆，图关系在门控上下文投影前裁定支持、矛盾与替代。核心主张是：可靠的个性化是一个记忆设计问题——有用的记忆是结构化、选择性暴露、持续整合并带有认知标签的，而非存储为无差别的对话历史。除框架外，我们将MRMS实例化为一个轻量级原型，实现了结构化记录、向量检索、时间策略和基于图的修正。该原型通过预生成记忆选择、修正、边界执行和证据归因，在具有明确证据要求的受控长寿命交互场景中，演练了核心基底机制。

## Abstract
Long-lived AI agents require continuity across interactions, but continuity cannot be obtained by simply extending the prompt window. An agent must preserve useful prior experience, retrieve it selectively, distinguish personal context from external evidence, and revise memory when the underlying situation changes. We propose an architectural memory substrate organized along two orthogonal axes: a representational axis spanning structured records, vector representations, and graph relations; and a temporal axis spanning short-term traces, medium-term abstractions, and long-term semantic commitments. Its key design constraint is synchronized structured-vector-graph memory: structured records govern eligibility, vector representations support recall, and graph relations adjudicate support, contradiction, and supersession before gated context projection. Its central claim is that reliable personalization is a memory design problem: useful memory is structured, selectively exposed, continuously consolidated, and epistemically labeled rather than stored as undifferentiated conversation history. Beyond the framework, we instantiate MRMS as a lightweight prototype implementing structured records, vector retrieval, temporal policies, and graph-based revision. The prototype exercises the core substrate mechanisms through pre-generation memory selection, revision, boundary enforcement, and evidence attribution under controlled long-lived interaction scenarios with explicit evidence requirements.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：长寿命 AI 智能体需要在连续交互中保持个性化记忆（如用户偏好、先前决策、未完成任务），但仅靠扩展提示窗口或保留完整对话历史行不通。长上下文成本高、注意力不均衡，且历史记录无法区分临时指令与持久偏好、已验证事实与推测、个人记忆与外部证据。
- **背景**：现有方法（如 RAG、长上下文模型、Agent 反射）侧重于检索质量或提示扩展，但缺乏对记忆影响的事前控制——即在使用记忆之前，系统应知道记忆的来源、是否仍有效、属于哪个边界、是否存在更新的证据。
- **核心主张**：可靠个性化本质是**记忆设计问题**，记忆应被结构化、选择性暴露、持续整合、带有认知标签（如来源、置信度、状态），而非无差别的历史存储。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想：两轴多分辨率记忆基底

- **表示轴**：每个记忆对象同时以三种视图存储：
  - **结构化记录**（控制平面）：存储状态、时间层、作用域、来源类、置信度、修订历史，用于资格判断。
  - **向量表示**（语义寻址）：用于近似语义检索。
  - **图关系**（关系推理）：类型化边（支持、矛盾、替代、派生等），用于裁定冲突和追溯证据。
- **时间轴**：按持久性分为短时痕迹（易写入/丢弃）、中时会话抽象、长时语义承诺（需更强证据才能提升）。
- **同步不变量**：三者通过稳定 ID 关联；结构化状态主宰选择（如已弃用/替代的记忆不能被激活）；图节点必须对应有效结构化记录；高影响边须有证据记录。

### 2.2 关键技术细节

- **记忆对象定义**：包含层、文本声明、向量、源、归属、作用域、置信度、修订历史、状态（原始/临时/活跃/替代/退休）。
- **选择流程**：先通过结构化门控（授权、 scope、状态）得到授权集 A_t → 向量检索 top-k → 图扩展（支持、矛盾等） → 最终上下文包（含本地状态、情景证据、语义声明、关系注释、外部证据、负面语境）。
- **评分函数**：综合考虑语义相关性、置信度、新鲜度、任务效用、冗余代价。
- **整合与修订**：写操作仅记录痕迹，整合将痕迹提升为摘要/事实/关系（需重复支持与效用），修订通过增加置信度/缩小 scope/标记为替代来实现。
- **边界投影**：在生成前根据用户、任务模式、来源类、隐私政策等过滤，防止无关记忆干扰。

## 6. 主要结论与发现

- **Full MRMS 整体准确率达 98.8%**，而向量+结构化+时间策略为 75.0%，仅向量为 21.2%，仅最近上下文为 3.9%。
- **各组件贡献明确**：
  - 结构化门控实现 100% 边界控制。
  - 时间策略实现 100% 过期抑制和弃权。
  - 图解析对矛盾处理（100%）和证据归因（95%）贡献最大。
- **残留错误**主要发生在证据归因中：在 100 个任务中有 10 次漏选了支持线索（但选对了活动声明）。
- **上下文简洁性**：平均选择 1.11 个记忆对象（候选 1.8 个），保持提示紧凑。

## 7. 优点

- **架构创新**：将结构化-向量-图同步作为显式架构不变量，而非实现细节，从而将记忆控制从“检索质量”提升为“影响前控制”。
- **可审计性**：每个记忆对象具有结构化记录，支持追溯来源、状态、修订历史，便于调试和验证。
- **设计分离**：写操作与整合分离，允许快速记录但保守提升，解决稳定性-可塑性困境。
- **评估聚焦**：专注预生成记忆决策（而非下游任务），使评估更直接、可复现，并与具体 LLM 解耦。
- **轻量原型**：无需外部 API 或 LLM 判断，便于社区复现和扩展。

## 8. 不足与局限

- **偏差风险**：
  - 场景模板化可能导致某些机制被高估（如时间策略在严格时间戳下表现完美，但实际中时间边界可能模糊）。
- **应用限制**：
  - 当前实现依赖明确的元数据（源、作用域、图边），在自动提取这些信息的真实系统中易出错。
