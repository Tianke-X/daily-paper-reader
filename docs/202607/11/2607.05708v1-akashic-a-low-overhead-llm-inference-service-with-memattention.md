## 摘要
最近的基于LLM的智能体系统在多轮交互、工具调用和跨会话工作流中持续累积上下文。对每个请求重放完整历史很快变得不切实际：长上下文增加了预填充成本，可能超出上下文限制，并且经常将任务相关证据埋没在无关内容中，降低了服务效率和输出质量。我们提出Akashic，一个围绕MemAttention构建的低开销内存系统，它将上下文组织成有界块并建模跨块的语义关系，无需重复重写完整历史即可保留跨块证据。Akashic进一步应用软硬件协同设计的内存放置，将可能共同检索的块共置，减少了检索碎片和I/O开销。在四个代表性工作负载和三种模型规模上，与先前强大的内存基准相比，Akashic将任务准确率提高了最多10.2个百分点，吞吐量提高了最多1.21倍，可持续请求率提高了最多1.88倍。

## Abstract
Recent LLM-based agent systems continuously accumulate context across multi-turn interactions, tool invocations, and cross-session workflows. Replaying the full history for every request quickly becomes impractical: long contexts increase prefill cost, may exceed context limits, and often bury task-relevant evidence in irrelevant content, degrading both serving efficiency and output quality. We propose Akashic, a low-overhead memory system built around MemAttention, which organizes context into bounded chunks and models semantic relationships across chunks, preserving cross-chunk evidence without repeatedly rewriting the full history. Akashic further applies hardware-software co-designed memory placement to co-locate likely co-retrieved chunks, reducing retrieval fragmentation and I/O overhead. Across four representative workloads and three model sizes, Akashic improves task accuracy by up to 10.2 points, throughput by up to 1.21x, and sustainable request rate by up to 1.88x over strong prior memory baselines.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM 智能体（Agent）在多轮交互、工具调用和跨会话工作流中会不断累积长上下文。重放完整历史导致预填充成本高、可能超出上下文窗口、无关内容淹没关键证据，严重降低服务效率与输出质量。
- **现有方法**：采用外部记忆（如摘要压缩和检索）来控制上下文长度。但存在两类不足：
    - **粒度权衡**：全局式摘要（如 Mem0）更新成本随历史增长，且混合无关话题会引入噪声；分段式摘要（如 MemGAS）虽限制单次成本，但会分割语义关联的证据，削弱多轮推理。
    - **物理放置与语义无关**：检索选出的记忆虽语义相关，但物理存储可能分散在不同页面/块中，导致 I/O 开销大，形成“局部性鸿沟”。
- **研究目标**：设计一种低开销、高吞吐且保持任务准确率的 LLM 推理记忆系统。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：通过 **MemAttention** 将上下文组织为有界块（chunk），在块间建立语义关系，避免重写全历史；并结合硬件-软件协同的内存放置优化，减少检索碎片和 I/O 开销。
- **关键技术细节**：
    - **Chunk 级记忆维护**：以固定大小（默认1024 token）的 chunk 为基本单位，仅对触发阈值的当前活跃块进行压缩，避免全局重写。维护成本为 $O(L_{\text{chunk}})$，与历史总长无关。
    - **交叉块推理（Cross-chunk Inference）**：压缩新块时，利用 LLM 将其与之前已存储块的关键词元数据进行语义匹配，选择 top‑5 相关块进行联合推理，以更新或废弃过期/冲突信息。
    - **模型驱动的语义匹配**：在记忆维护和推理检索中均使用同一机制——将当前查询/块的关键词与候选块元数据送入 LLM，由模型选择最相关的 top‑p 块。
    - **记忆管理器**：
        - 进行 **关联感知重定位**：根据联合推理中模型的推断，将频繁共被访问的块物理上放置到同一新块，减少未来读取时的页面数。
        - 采用 **异地更新** 与后台垃圾回收：无效比率 ≥75% 的块触发压缩，提取存活块并按用户/会话分区，再按模型推断的亲和性分组打包，同时回收空间并优化布局。
- **公式与算法**：
    - 保留比 $\rho = L_{\text{eff}} / L_{\text{raw}}$ 描述上下文信息密度。
    - 块无效比 $\phi(B) = N_{\text{invalid}}(B) / N_{\text{total}}(B)$。
    - 算法1（MemAttention）：增量式块压缩与模型驱动检索流程。
    - 算法2（模型驱动的重定位与块压缩）：在线关联重定位 + 后台块压缩。
