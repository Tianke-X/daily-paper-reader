## 摘要
尽管当前的AI智能体支持越来越长的上下文窗口、工具使用以及面向长时任务的技能执行，但它们仍需记忆系统来有效利用历史经验。现有的记忆框架通常依赖固定的存储、检索和摘要机制，这些机制在不同任务间可能较为僵化，且常需手动调整。为解决此局限，我们提出SelfMem——一种自优化记忆框架。受先前自改进AI工作的启发，我们遵循“授人以渔而非授人以鱼”的原则。SelfMem并非迫使模型遵循预定义的记忆策略或格式，而是提供一个包含记忆工具和反馈信号的环境，使智能体能够探索、评估并优化自身的记忆策略。我们的结果表明，在BEAM基准上，从10万到100万token的对话规模中，SelfMem始终优于检索、压缩和智能体记忆基线。相比最强基线，SelfMem在10万、50万和100万token下的官方得分分别提升48.7%、40.8%和41.9%。进一步的问句类型分析显示其对多样化记忆需求具有广泛鲁棒性，而优化研究表明模型引导的策略精炼能进一步提升性能。

## Abstract
While current AI agents support increasingly long context windows, tool use, and skill execution for long-horizon tasks, they still require memory systems to effectively leverage historical experience. Existing memory frameworks typically rely on fixed storage, retrieval, and summarization mechanisms, which can be rigid across different tasks and often require manual tuning. To address this limitation, we propose SelfMem, a self-optimizing memory framework. Inspired by prior work on self-improving AI, we follow the principle of "teaching an agent to fish rather than giving it a fish." Instead of forcing the model to follow a predefined memory strategy or format, SelfMem provides an environment with memory tools and feedback signals that allow the agent to explore, evaluate, and refine its own memory strategy. Our results show that SelfMem consistently outperforms retrieval, compression, and agent-memory baselines on BEAM across conversation scales from 100K to 1M tokens. Compared with the strongest baseline, SelfMem improves the official score by 48.7%, 40.8%, and 41.9% at 100K, 500K, and 1M, respectively. Further question-type analysis shows broad robustness across diverse memory demands, and our optimization study shows that model-guided strategy refinement further improves performance.

#### 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：现有AI智能体的记忆系统（如RAG、MemGPT、MemoryBank等）大多采用**固定的存储、检索和总结机制**，在不同任务和对话历史中表现僵化，且需要大量手动调优才能适应特定场景。这种“僵硬”的设计限制了智能体在长时任务中的自适应能力和泛化性。
- **研究动机**：受自改进AI（Self-Improving AI）工作的启发（如Self-Refine、Reflexion、OPRO等），作者提出应**“授人以渔而非授人以鱼”**——即让智能体自己学会如何管理记忆，而不是强制使用预设策略。
- **整体含义**：SelfMem旨在将记忆管理从固定管线转变为**智能体可控的优化过程**，通过提供记忆工具和反馈信号，让智能体自主探索、评估并优化其记忆策略，从而在不同任务和规模下获得更好的性能-成本权衡。

#### 2. 方法论：核心思想、关键技术细节
- **核心思想**：将记忆管理视为**模型控制的优化过程**，智能体通过工具调用和反馈信号自主决定存储、更新、压缩和检索哪些信息。
- **关键技术细节**：
  1. **三部分架构**：
     - **原始转录存储（Transcript Store）**：只读的SQLite数据库，保存所有交互历史（不可变）。
     - **记忆工作空间（Memory Workspace）**：模型可读写的结构化记忆（用户画像、偏好、项目状态等）。
     - **动作空间（Action Space）**：包含四类工具：
       - **转录读取工具**（conversation_manifest, conversation_sql_query等）
       - **记忆读取工具**（memory_read）
       - **记忆写入工具**（memory_change：添加、替换、合并、归档等）
       - **记忆审核工具**（memory_review：检查来源、是否过时、矛盾等）
  2. **反馈信号**：包括工具反馈（转录查询结果、诊断信息）、操作信号（token使用量、延迟、缓存命中率、估计成本）。**不提供隐藏的监督信号**（如基准答案、评分细则）。
  3. **记忆构建流程**：智能体执行一个“**检查-写入-审核-修订**”循环，从空工作空间开始，通过工具调用逐步构建记忆，并可根据审核反馈进行修订。
  4. **记忆条件回答**：最终记忆工作空间作为紧凑指南，配合从原始转录中检索到的相关片段，提供给答案模型。
  5. **策略优化**：智能体可以通过迭代方式，利用训练对话的汇总反馈（总分、成本、诊断信息）来改进记忆策略（即**自然语言策略优化**），而不使用验证集答案。
