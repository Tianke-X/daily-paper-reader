## 摘要
长期记忆仍然是AI智能体的结构性弱点。主流方法——检索增强生成（RAG）——依赖于基于嵌入的相似性搜索，这从构建上就是不透明的，难以审计，并且受限于向量表示的理论边界。我们提出了记忆编排语义系统（MOSS），这是一种智能体记忆架构，其中智能体驱动对结构化关系数据库的检索。MOSS是模型无关、存储无关且API无关的：它可以在任何关系引擎上运行，连接到任何LLM提供商（或确定性非LLM流程），并部署在任何基础设施上，无论是本地还是云端。其检索执行是符号化且可复现的（一旦查询被制定，检索循环中不再有LLM参与），并且系统的每一步，从索引到答案制定，都被记录并可检查，使得MOSS从设计上就是可审计的。MOSS并未强加外部本体，而是从语料库本身推导出其概念词汇。我们报告了在智能体记忆文献中独一无二的纵向部署：连续一年在单个学者工作语料库上的生产运行——该对话语料库可追溯到2024年10月（约4400万词，已回溯索引），包含110,183个片段，以及163,494个编录文档、569个归纳推导的概念、322,662个概念注释和总计约500万关系的十一个元数据图——跨越连续四代基础设施。虽然当前案例是针对单个研究者，但该架构绝不仅限于个人：它可以服务于团队、机构或任何随时间积累知识的实体。我们认为，可审计、自主、结构无界记忆是旨在伴随个人或组织多年而非仅限会话的AI智能体的先决条件。

## Abstract
Long-term memory remains a structural weakness of AI agents. The dominant approach, retrieval-augmented generation (RAG), relies on embedding-based similarity search, which is opaque by construction, difficult to audit, and bounded by the theoretical limits of vector representations. We present the Memory-Orchestrated Semantic System (MOSS), an agentic memory architecture in which the agent drives retrieval over a structured relational database. MOSS is model-agnostic, storage-agnostic, and API-agnostic: it runs on any relational engine, connects to any LLM provider (or to deterministic non-LLM processes), and deploys on any infrastructure, local or cloud. Its retrieval execution is symbolic and reproducible (once a query is formulated, no LLM participates in the retrieval loop) and every step of the system, from indexing to answer formulation, is logged and inspectable, making MOSS auditable by construction. Rather than imposing an external ontology, MOSS derives its conceptual vocabulary from the corpus itself. We report on a longitudinal deployment unique in the agentic-memory literature: a year of continuous production over an individual scholar's working corpus--a conversational corpus reaching back to October 2024 (some 44 million tokens, retroactively indexed) comprising 110,183 segments, alongside 163,494 catalogued documents, 569 inductively derived concepts, 322,662 concept annotations, and eleven metadata graphs totaling approximately five million relations--across four successive infrastructure generations. While the present case is that of a single researcher, the architecture is in no way specific to one person: it serves a team, an institution, or any entity that accumulates knowledge over time. We argue that auditable, sovereign, structurally unbounded memory is a precondition for AI agents intended to accompany a person or an organization over years rather than sessions.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有AI智能体的长期记忆存在结构性缺陷。主流方法检索增强生成（RAG）依赖嵌入向量相似搜索，从设计上就是不透明的，难以审计，且受到向量表示的理论边界限制（如单向量检索的“淹没”问题、嵌入维度扩展的不可预测性）。
- **背景**：智能体需要能够跨越多年而非仅单次会话地伴随个人或组织，但当前系统（如RAG、GraphRAG、生产级记忆层）要么是管道而非记忆组织（RAG），要么图表替换了原始文本（GraphRAG），要么是私有/混合向量架构，无一兼顾可审计性、主权性和长期语料规模。
- **论文立场**：提出可审计、主权性、结构无界的记忆架构是长期伴随型AI智能体的先决条件。

## 2. 方法论

### 核心思想
- **“数据库是地图，文档是领地”**：原始语料以普通文件形式完整保留（不切碎、不替换）；关系数据库保存密集的元数据地图（摘要、时间戳、参与者、情感状态、概念注释等），大多数查询仅从地图回答，无需触及原始文本。
- **“查询主权”**：相关性是问题本身的属性，而非存储时预定的索引属性。权重由智能体在查询时动态决定，翻转了RAG在存储时固定嵌入空间几何的假设。
- **“审计一切，隐藏无物”**：每次检索都是显式SQL，可重放、可检查；所有系统调用实时日志；对话档案为纯文本+时间戳+词数；无任何不可解释的几何层。

### 关键技术细节
- **架构组件**：Orchestrator（REST API/MCP端点、LLM连接器、QueryProfiler、确定性SQL检索层） + Memory Structure（关系数据库、归纳本体、元数据图）。
- **三元无关性**：模型无关（任何LLM提供商或确定性进程）、引擎无关（任何关系引擎，如SQLite/PostgreSQL）、存储/基础设施无关（云端、本地、企业对象存储等）。
- **查询生命周期**：
  1. 智能体分析意图（时间、情感、主题等）。
  2. 参数化QueryProfiler，生成SQL（或直接写SQL）。
  3. SQL确定性执行（无LLM参与，结果可复现）。
  4. 智能体评估结果并迭代（最多15轮），然后制定答案。
  5. 非确定性限定在智能体边缘，检索核心为符号化、确定性。
- **记忆构建**：实时索引（滑窗溢出内容被分段、摘要、情感注释、概念注释并归档），以及回溯索引旧存档。
- **归纳本体**：569个概念从语料底部向上推导（种子+自动提取+命名实体），322,662个片段级注释，类似定性研究的编码簿主题分析。不使用外部受控词汇。
- **元数据图**：11个图，约500万关系（共现、时间邻接、情感共鸣、主题凝聚等）。情感共鸣边为创新点。
- **覆盖层（métacalque）**：正在开发的统一覆盖层，允许多个可叠加的、可执行的语义层（如情感层、代码块层、项目层），每个层独立作用于语料，由智能体在检索时选择使用。

### 算法流程（文字说明）
没有显式公式。核心流程：输入自然语言问题 → 智能体意图分析 → 生成带权重的查询请求 → QueryProfiler将其转换为SQL → 用SQL直接查询关系数据库（多表连接、递归SQL遍历图） → 返回元组集合 → 智能体评估满意度，若不满意则调整查询参数重新生成SQL并重试，直到满意或达到上限 → 智能体基于结果制定自然语言答案。

## 7. 优点

- **架构创新**：符号化、确定性检索与智能体引导的查询制定分离，同时保持了可审计性；查询主权翻转了RAG的存储时固定权重假设。
- **长期真实部署**：约一年的连续生产运行，跨四代基础设施，覆盖完整个人工作语料，是该领域首个报告的此类证据。
- **三元无关性**：模型、引擎、存储/基础设施完全解耦，无供应商锁定，可本地或云端部署。
- **归纳本体 + 覆盖层**：从语料自底向上构建概念词典，适应个人特有的知识体系；开发中的métacalque概念将文本分析的描述性传统转化为可执行检索原语。
- **情感索引**：是工程系统中罕见的维度，但在个人记忆场景中效用显著。
- **设计哲学清晰**：“数据库是地图，文档是领地”、可审计性作为建筑特性而非插件，避免使用向量嵌入在检索路径中。
